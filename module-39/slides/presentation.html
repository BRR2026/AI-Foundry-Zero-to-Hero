☁️ The CSP Program Model

Cloud Solution Providers own the customer relationship, billing, and first-line support for Azure services.

🏢
Tier 1 — Direct CSP
Partner transacts directly with Microsoft. Sets pricing, provides full support stack, manages billing via Partner Center.
🤝
Tier 2 — Indirect CSP
Partner works through an indirect provider for billing. Maintains customer relationship and technical management.
🔑
AOBO Access
Admin on Behalf Of grants Owner role on customer subscriptions by default. Simple but overly broad for AI workloads.
🛡️
Azure Lighthouse
Granular delegated access with specific RBAC roles. Auditable, customer-visible, revocable. Preferred for AI Foundry.

📐 Deployment Topology Options

PatternIsolationCostBest For
Dedicated Hub per Customer 🔒 Full 💰 High Regulated (HIPAA, PCI), large enterprise
Shared Hub, Separate Projects 🔐 Moderate 💰 Medium SMB, standardized AI solutions
Fully Shared (Logical) 🔑 Basic 💰 Low Dev/test, non-sensitive workloads

Decision Criteria

  • Regulatory requirements → Dedicated isolation
  • Cost sensitivity → Shared infrastructure with RBAC boundaries
  • Data sovereignty → Regional deployment controls
  • Capacity needs → Dedicated vs. shared PTU allocation

🔭 Azure Lighthouse Deep Dive

The preferred mechanism for cross-tenant AI Foundry management.

Why Lighthouse > AOBO

  • Granular RBAC (not just Owner)
  • Just-in-Time elevation
  • Full audit trail in customer logs
  • Customer can revoke anytime
  • Works across Entra ID tenants

Recommended Role Assignments

  • AI Ops Team: Contributor
  • Model Admins: Cognitive Services Contributor
  • Monitoring: Reader + Monitoring Contributor
  • Security: Security Reader
  • Escalation: Eligible Owner (JIT)

📜 Cross-Tenant Governance

Azure Policy Initiatives for AI Foundry

🏷️
Tag Enforcement
Require csp:customer, csp:tier, csp:managed tags on all AI Foundry resources
🌐
Network Rules
Deny public network access on AI Foundry hubs; require private endpoints
📍
Regional Constraints
Restrict allowed regions based on customer data residency requirements
🔐
Encryption
Enforce CMK for Gold/Platinum tiers; platform keys minimum for all
📊
Diagnostics
Require diagnostic settings forwarding to central Log Analytics workspace
🛡️
Content Safety
Require AI Content Safety filters on all model deployments

🔒 Data Isolation Strategies

Network Isolation

  • Private endpoints per AI Foundry hub
  • NSG rules between customer environments
  • Separate Private DNS zones
  • Hub-Spoke VNet topology for CSP management

Identity Isolation

  • Managed Identity per customer hub
  • Customer-Managed Keys (CMK)
  • Conditional Access on CSP admin access
  • Cross-tenant access policies

Storage Isolation

  • Separate storage accounts per customer
  • Geo-restriction via Azure Policy
  • Data classification tags
  • Purge protection on Key Vaults

Compute Isolation

  • Dedicated PTU for premium tiers
  • Shared PTU with rate limiting for standard
  • Separate agent compute pools
  • Resource quotas per project

💰 Cost Management & Chargeback

AI Foundry Cost Components

  • Model Inference: Per-token or provisioned throughput (PTU)
  • Compute: Agent hosting, evaluation pipelines
  • Storage: Data, logs, model artifacts
  • Networking: Private endpoints, data transfer

Chargeback Strategies

  • Subscription-level: Cleanest; 1 sub per customer
  • Tag-based: csp:customer tags + Cost Management filters
  • Token metering: Custom per-project token tracking
  • Hybrid: Subscription for infra + tokens for inference
# Token usage chargeback query AzureDiagnostics | where ResourceProvider == "MICROSOFT.MACHINELEARNINGSERVICES" | extend Customer = tostring(tags_s["csp:customer"]) | summarize PromptTokens=sum(toint(properties_s["promptTokens"])), CompletionTokens=sum(toint(properties_s["completionTokens"])) by Customer, bin(TimeGenerated, 1d)

🛡️ Security & Compliance

Security Baseline Checklist

  • Private endpoints — no public access
  • Managed identities — no credentials
  • CMK for Gold/Platinum tiers
  • Sentinel with cross-tenant connectors
  • CIS benchmark policies on all subs
  • Content Safety on all deployments
  • Diagnostic settings to central workspace
  • Geo-redundant DR for premium tiers

🎯 Key Takeaways

  • Azure Lighthouse over AOBO for AI Foundry management
  • Match topology to customer compliance & cost needs
  • Tiered SLAs = differentiated managed AI offering
  • Tag-based cost tracking + token metering for chargeback
  • Azure Policy enforces security baseline at scale
  • Compliance is shared — know the responsibility boundaries
  • Automate onboarding with Bicep/Terraform templates
