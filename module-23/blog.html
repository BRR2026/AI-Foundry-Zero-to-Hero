1. Copilot Studio Overview and Capabilities

Microsoft Copilot Studio is a low-code platform for building, customizing, and deploying AI-powered conversational agents (copilots). It sits at the intersection of Microsoft's Power Platform and its enterprise AI investments, enabling organizations to create intelligent bots without deep coding expertise while still allowing pro-code extensibility.

1.1 What is Copilot Studio?

Copilot Studio evolved from Power Virtual Agents and has been reimagined as the primary platform for creating and managing copilots across the Microsoft ecosystem. It provides a visual authoring canvas, built-in natural language understanding, and seamless integration with Microsoft 365, Dynamics 365, and Azure services.

🎨

Visual Authoring

Drag-and-drop topic and conversation flow designer with low-code logic branching.

🧠

Generative AI Answers

Built-in generative answers using GPT models with grounded knowledge sources.

🔌

Plugin Ecosystem

Extend copilots with custom plugins, connectors, and AI Foundry model endpoints.

🌐

Omnichannel Deployment

Publish to Teams, web, Dynamics 365, Slack, Facebook, and custom channels.

🔒

Enterprise Security

Entra ID authentication, DLP policies, environment-level governance, and audit logs.

📊

Built-in Analytics

Session analytics, CSAT tracking, topic performance, and AI-driven conversation insights.

1.3 Where AI Foundry Fits In

While Copilot Studio provides built-in generative answers, enterprises often need custom models, domain-specific RAG pipelines, and specialized agents built on Azure AI Foundry. The integration allows you to:

  • Use custom fine-tuned models deployed on AI Foundry as the intelligence layer behind Copilot Studio conversations
  • Leverage RAG agents with proprietary data indexes for accurate, grounded responses
  • Chain multiple AI Foundry endpoints together within a single copilot conversation flow
  • Apply enterprise governance from both platforms — Copilot Studio DLP + AI Foundry content safety
💡 Key Insight
Think of Copilot Studio as the conversational front-end and Azure AI Foundry as the AI back-end. Together, they let you deliver sophisticated AI experiences through familiar chat interfaces without rebuilding everything from scratch.

2. Publishing AI Foundry Models as Copilot Plugins

Copilot Studio's plugin framework enables you to expose any REST API — including AI Foundry model endpoints — as a callable action within your copilot. This section walks through the process of creating a custom connector plugin for an AI Foundry deployment.

2.1 Understanding the Plugin Architecture

Copilot Studio plugins are defined using OpenAPI specifications (Swagger) that describe the API contract. When a user's intent matches a plugin's description, Copilot Studio's orchestrator invokes the API and uses the response to formulate a reply.

Plugin Component Purpose AI Foundry Mapping
OpenAPI Spec Describes API endpoints and parameters AI Foundry model inference endpoint
Authentication Secures API calls API key or Entra ID managed identity
Plugin Description Tells the AI when to invoke this plugin Model / agent purpose description
Input Schema Defines required/optional parameters Model input format (prompt, context)
Output Schema Structures the response Model output format (completion, citations)

2.2 Step-by-Step: Create the OpenAPI Specification

First, create an OpenAPI 3.0 specification file that wraps your AI Foundry model endpoint. This tells Copilot Studio how to call your model:

YAML — OpenAPI Spec for AI Foundry Plugin
openapi: "3.0.1"
info:
  title: "AI Foundry RAG Agent Plugin"
  description: "Queries a RAG agent deployed on Azure AI Foundry for
    grounded answers from enterprise knowledge bases."
  version: "1.0.0"
servers:
  - url: "https://{project-name}.{region}.inference.ml.azure.com"
    description: "AI Foundry Inference Endpoint"
paths:
  /score:
    post:
      operationId: "queryRAGAgent"
      summary: "Query the enterprise RAG agent"
      description: "Send a user question to the RAG agent which
        searches the enterprise knowledge base and returns a grounded
        answer with source citations."
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              properties:
                question:
                  type: string
                  description: "The user's question"
                chat_history:
                  type: array
                  description: "Previous conversation turns"
                  items:
                    type: object
              required:
                - question
      responses:
        "200":
          description: "Successful RAG response"
          content:
            application/json:
              schema:
                type: object
                properties:
                  answer:
                    type: string
                  citations:
                    type: array
                    items:
                      type: object

2.3 Registering the Plugin in Copilot Studio

1

Open Copilot Studio and Navigate to Plugins

In your copilot project, go to Settings → Plugins → Add a plugin. Select "Custom connector" to register your AI Foundry endpoint as a plugin.

2

Upload the OpenAPI Specification

Upload your YAML or JSON OpenAPI spec file. Copilot Studio will parse the spec, extract endpoint definitions, parameter schemas, and generate a plugin configuration preview.

3

Configure Authentication

Set the authentication method — either API Key (inserted as a header) or OAuth 2.0 with Entra ID for managed identity. For production, Entra ID is strongly recommended.

4

Test and Publish the Plugin

Use the built-in test panel to send sample queries to the AI Foundry endpoint through Copilot Studio. Verify the response format and then enable the plugin for your copilot.

⚠️ Important: Plugin Descriptions Matter
The description field in your OpenAPI spec is critically important. Copilot Studio's AI orchestrator uses it to decide when to invoke your plugin. Write clear, specific descriptions that explain exactly what your AI Foundry model does and what types of questions it can answer.

3. Building Conversational Experiences in Copilot Studio

With your AI Foundry plugin registered, you can now build sophisticated conversational flows that combine Copilot Studio's visual authoring with AI Foundry's inference power.

3.1 Designing Conversation Topics

Copilot Studio organizes conversations around topics — self-contained conversational units with trigger phrases, logic nodes, and responses. When integrating AI Foundry, you'll typically create topics that:

  • Collect context from the user before calling the AI Foundry endpoint
  • Invoke the plugin action with structured input
  • Format the response including citations and follow-up suggestions
  • Handle errors gracefully when the AI Foundry endpoint is unavailable

3.2 Example: Knowledge Base Q&A Topic

Below is a Power Fx expression used within a Copilot Studio topic node to call the AI Foundry plugin and process the response:

Power Fx — Plugin Invocation in Topic
// Invoke the AI Foundry RAG agent plugin
Set(ragResponse,
    'AI Foundry RAG Agent Plugin'.queryRAGAgent({
        question: System.Activity.Text,
        chat_history: Conversation.History
    })
);

// Format the response with citations
If(
    IsBlank(ragResponse.answer),
    "I couldn't find an answer. Let me connect you to a specialist.",
    Concatenate(
        ragResponse.answer,
        Char(10) & Char(10) & "📚 Sources:",
        Concat(
            ragResponse.citations,
            Concatenate(Char(10) & "• ", ThisRecord.title)
        )
    )
);

3.4 Generative Orchestration vs. Classic Topics

Feature Classic Topics Generative Orchestration
Routing Trigger phrases (keyword-based) AI-powered intent classification
Plugin invocation Explicit action nodes Automatic based on description
Context management Manual variable passing Automatic conversation memory
Best for AI Foundry Deterministic workflows Open-ended Q&A scenarios
Fallback behavior Configured escalation Generative answer or escalation
💡 Pro Tip: Generative Orchestration + Plugins
When using generative orchestration, Copilot Studio's AI engine reads your plugin's OpenAPI description to determine when to invoke it automatically. This means a well-described AI Foundry plugin can be invoked intelligently without any manual topic configuration — the AI decides when to call it based on the user's intent.

4. Authentication and Data Flow Between Services

Security is paramount when connecting Copilot Studio to AI Foundry. This section covers the authentication patterns and data flow architecture that ensure secure, compliant operation.

4.1 Authentication Patterns

🔑

API Key Authentication

Simplest approach — store the AI Foundry endpoint key in Copilot Studio's secure configuration. Suitable for development and testing environments.

🛡️

Entra ID (OAuth 2.0)

Production-recommended. Uses managed identity with client credentials flow. No secrets stored in Copilot Studio — tokens are acquired dynamically.

👤

User-Delegated Auth

Passes the signed-in user's identity through to AI Foundry — enabling per-user content filtering and row-level security on data.

4.2 Configuring Entra ID Authentication

The recommended production pattern uses an Entra ID app registration to secure the connection between Copilot Studio and AI Foundry:

BASH — Register the App and Configure Permissions
# Step 1: Create an Entra ID app registration for the plugin
az ad app create \
  --display-name "Copilot-Studio-AIFoundry-Plugin" \
  --sign-in-audience "AzureADMyOrg"

# Step 2: Create a service principal
az ad sp create --id <app-id>

# Step 3: Assign the Cognitive Services User role
# This allows the plugin to call AI Foundry endpoints
az role assignment create \
  --assignee <service-principal-id> \
  --role "Cognitive Services User" \
  --scope "/subscriptions/<sub-id>/resourceGroups/<rg>/providers/Microsoft.CognitiveServices/accounts/<account>"

# Step 4: Generate a client secret (store securely)
az ad app credential reset \
  --id <app-id> \
  --display-name "copilot-studio-secret"

4.3 End-to-End Data Flow

Secure Data Flow: Copilot Studio ↔ AI Foundry

End User
Teams / Web
→
Copilot Studio
Entra ID Auth
→
Custom Connector
OAuth 2.0 Token
→
AI Foundry
RBAC Validated

The data flow follows these security checkpoints:

  1. User authenticates to Copilot Studio via Entra ID (SSO through Teams or web)
  2. Copilot Studio acquires a token using the plugin's app registration (client credentials or on-behalf-of flow)
  3. The token is sent in the Authorization header to the AI Foundry inference endpoint
  4. AI Foundry validates the token against its RBAC policies, checking role assignments
  5. The response traverses back through the same secure channel, with content safety applied at the AI Foundry layer

4.4 Data Residency and Compliance

🔐 Compliance Considerations
  • Data residency: Ensure both your Copilot Studio environment and AI Foundry project are deployed in the same Azure region to maintain data residency requirements
  • DLP policies: Configure Power Platform DLP policies to control which connectors can be used together — place your AI Foundry connector in the "Business" group
  • Audit logging: Both platforms generate audit logs — combine them in Microsoft Purview for unified compliance monitoring
  • Content safety: AI Foundry's content safety filters remain active even when called via Copilot Studio, providing defense-in-depth

6. Advanced Integration Patterns

6.1 Multi-Agent Routing with AI Foundry

For complex enterprise scenarios, you can register multiple AI Foundry agents as separate plugins in Copilot Studio. The generative orchestrator will route each user query to the most appropriate agent:

# Plugin 1: HR Knowledge Agent
info:
  title: "HR Knowledge Agent"

    benefits, leave policies, and employee handbook topics."

6.3 Error Handling and Fallback Patterns

Robust conversational experiences need graceful error handling. Design your Copilot Studio topics to handle AI Foundry failures:

Power Fx — Error Handling Pattern
// Attempt to call AI Foundry with error handling
Set(pluginResult,
    IfError(
        'AI Foundry RAG Agent Plugin'.queryRAGAgent({
            question: System.Activity.Text
        }),
        // Fallback: Use built-in generative answers
        {
            answer: "I'm experiencing a temporary issue with my knowledge base.
                     Let me try answering with general knowledge.",
            citations: Blank(),
            fallback: true
        }
    )
);

// Log errors for monitoring
If(pluginResult.fallback,
    Trace("AI Foundry plugin unavailable — used fallback",
          TraceSeverity.Warning)
);
