Creating the OpenAPI Spec

openapi: "3.0.1"
info:
  title: "AI Foundry RAG Agent Plugin"
  description: "Queries a RAG agent deployed on Azure
    AI Foundry for grounded answers from the enterprise
    knowledge base with source citations."
  version: "1.0.0"
servers:
  - url: "https://{project}.{region}.inference.ml.azure.com"
paths:
  /score:
    post:
      operationId: queryRAGAgent
      summary: "Query the enterprise RAG agent"
      requestBody:
        required: true
        content:
          application/json:
            schema:
              type: object
              properties:
                question: { type: string }
                chat_history: { type: array }
              required: [question]
      responses:
        "200":
          description: "RAG response with citations"

Plugin Registration Steps

①

Navigate to Plugins

Settings → Plugins → Add Plugin → Custom Connector

②

Upload OpenAPI Spec

YAML or JSON format. Studio parses endpoints and schemas automatically.

③

Configure Authentication

API Key (dev) or Entra ID OAuth 2.0 (production recommended).

④

Test & Publish

Send sample queries in test panel, verify responses, then enable for copilot.

✅ Pro Tip: Write plugin descriptions as if explaining to a colleague what your AI agent does — be specific about domains and query types.
