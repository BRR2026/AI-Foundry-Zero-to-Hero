Microsoft's Six Responsible AI Principles

⚖️
Fairness
AI should treat all people equitably without reinforcing biases
🛡️
Reliability & Safety
AI should perform reliably and safely under all conditions
🔒
Privacy & Security
AI should respect privacy and be resilient to attacks
🌍
Inclusiveness
AI should empower and engage the broadest range of people
🔍
Transparency
AI should be understandable with clear documentation
📋
Accountability
People must be accountable for how AI systems operate

Key Insight: These principles are interdependent — like six pillars holding up a building. Remove any one and the structure becomes unstable.

When AI Goes Wrong — Real-World Failures

📄

Biased Hiring Tool (2018)

AI trained on historical data penalized resumes containing "women's" — amplified existing gender bias in tech hiring

📸

Facial Recognition Disparities

Error rates up to 34.7% for darker-skinned women vs. 0.8% for lighter-skinned men — unrepresentative training data

🤖

Chatbot Manipulation (2016)

Microsoft Tay generated offensive content within 16 hours — no content safety filters or adversarial testing

🏥

Healthcare Bias (2019)

Algorithm underestimated health needs of Black patients by using spending as a proxy for health — systemic bias in proxy variables

Every one of these failures could have been mitigated with proper Responsible AI practices and the tools we'll explore today.

Category Description Examples
🔴 Violence Physical harm, threats, weapons Graphic violence, threat instructions
🟠 Hate Identity-based attacks, discrimination Slurs, stereotyping, dehumanization
🟡 Sexual Sexually explicit or suggestive content Explicit descriptions, solicitation

Severity Levels (0-7)

Safe (0-1)
Low (2-3)
Medium (4-5)
High (6-7)

Default threshold: Medium and above are blocked. Each category is independently configurable.

Groundedness Detection

The Hallucination Problem

LLMs generate plausible-sounding text without verifying factual accuracy. Groundedness detection checks if responses are supported by provided source documents.

⚠️

Key Distinction

Grounded ≠ Accurate. A response can be grounded (supported by sources) but wrong if sources are incorrect.

How It Works

1. Source Documents

Provide context via RAG retrieval

2. Response Analysis

Each claim checked against sources

3. Scoring

1-5 scale: 5 = fully grounded, 1 = fully hallucinated

Result

Ungrounded responses can be blocked or flagged for review

AI Foundry's Defense

🛡️

Multi-Layer Detection

Pattern analysis, intent classification, and semantic understanding work together to detect manipulation attempts.

🔄

Defense in Depth

Even if an attack bypasses input detection, output content safety filters provide a second line of defense.

Additional Safety Tools

🔏 Protected Material Detection

Text Mode

Detects known copyrighted text — books, articles, song lyrics, and other protected content

Code Mode

Identifies licensed source code from known repositories — GPL, MIT, Apache, etc.

Why it matters: Protects your organization from IP infringement and legal exposure

📋 Custom Blocklists

Domain-Specific Blocking

Block terms, phrases, or regex patterns specific to your use case

Common Uses:

  • Brand protection — competitor names, brand variations
  • Regulatory — restricted terms in finance/healthcare
  • PII protection — SSN, phone number patterns
  • Policy enforcement — organization-specific rules

Configuring Content Filters

Configuration Steps

  1. Navigate to Safety + Security → Content Filters
  2. Click + Create content filter
  3. Set input thresholds for each category
  4. Set output thresholds for each category
  5. Enable additional features (jailbreak, protected material, groundedness)
  6. Add custom blocklists if needed
  7. Associate filter with your deployment

Best Practices

✅ Environment-Specific Filters

Dev: Annotate-only mode for testing
Staging: Production-like for validation
Prod: Strict, tuned for your use case

⚠️ Never Disable All Filters

Even in development, maintain minimum safety thresholds. Use annotate-only mode instead of disabling.

Evaluation Metrics for Safety

Metric What It Measures Target
Safety Score Overall adherence to safety guidelines > 90%
Groundedness Score Claims supported by source documents (1-5) > 4.0
Content Harm Defect Rate % of responses with harmful content < 2%
Jailbreak Resistance % of jailbreak attempts blocked > 95%
Protected Material Hit Rate Frequency of copyrighted content < 1%

Run evaluations regularly — not just at deployment time. Model behavior can drift, user patterns change, and new attack vectors emerge. Use AI Foundry's built-in evaluation tools for continuous monitoring.

Regulatory Landscape

🇪🇺

EU AI Act

World's first comprehensive AI law. Risk-based framework: Unacceptable → High → Limited → Minimal. Fines up to €35M or 7% revenue.

🇺🇸

NIST AI RMF

Voluntary US framework. Four functions: Govern, Map, Measure, Manage. Practical guidance for risk management.

🌐

OECD Principles

International AI principles adopted by 40+ countries. Foundation for national AI policies worldwide.

Industry-Specific Requirements

🏥 Healthcare

HIPAA compliance, FDA AI/ML guidance, clinical validation requirements

🏦 Finance

Fair lending laws, model explainability requirements, SEC AI guidance

Decision Framework

Before deploying AI, ask:

  • Is there a clear problem to solve?
  • Do we have quality data?
  • Can we explain decisions?
  • Have we assessed all 6 principles?
  • Do we have monitoring in place?
  • Is there human oversight?

When NOT to Use AI

No quality data • Can't explain decisions • No human oversight • Task requires moral judgment
