Why Evaluation Matters

Building an AI application is only half the battle. How do you know it's working well?

  • Models can hallucinate — generate plausible but false information
  • Prompt changes can cause regressions in unexpected areas
  • Users will try to misuse your system
  • Quality can drift over time as data and usage patterns change
💡 Evaluation isn't overhead — it's insurance. Organizations with structured evaluation reduce production incidents by up to 70%.

Each metric uses a 1–5 scale. Use multiple metrics together for a complete quality picture.

Groundedness — The #1 RAG Metric

Definition: Can every claim in the response be traced back to the provided context?

High Groundedness (5):

  • All facts supported by context
  • No fabricated details
  • Appropriate "I don't know" when context lacks info

Low Groundedness (1–2):

  • Facts not found in context
  • Invented statistics or dates
  • Plausible-sounding hallucinations
⚠️ A response can be fluent, coherent, and relevant while still being ungrounded. Always check groundedness!

Content Safety — Four Categories

CategoryWhat It Detects
🚫 Hate & FairnessSlurs, stereotypes, discrimination against identity groups
🔞 SexualSexually explicit or suggestive content
⚔️ ViolenceGraphic violence, threats, glorification of harm
💔 Self-HarmInstructions, encouragement, or glorification of self-harm

Each category has four severity levels: Safe (0) → Low (2) → Medium (4) → High (6)

You set the threshold per category per deployment. Content at or above the threshold is blocked.

Content Filters & Custom Blocklists

Content Filters are configured per deployment in AI Foundry:

  • Apply to both input (user prompts) and output (model completions)
  • Set independent severity thresholds for each category
  • Multiple filter configurations can coexist for different deployments

Custom Blocklists:

  • Block domain-specific terms (competitor names, internal codenames, regulated terms)
  • Managed via SDK or REST API
  • Support regex patterns for flexible matching

🛡️ Prompt Shield — Jailbreak Detection

Prompt Shield defends against two attack vectors:

Direct Jailbreaks:

  • "Ignore all previous instructions…"
  • Role-play attacks
  • Encoding tricks (Base64, leetspeak)

Indirect Prompt Injection:

  • Malicious instructions in documents
  • Hidden commands in retrieved data
  • Cross-domain injection via RAG
⚠️ Prompt injection is a top security risk for AI applications. Always enable Prompt Shield on production deployments.

🔴 Red-Teaming Your AI Application

Red-teaming = structured adversarial testing to find vulnerabilities before your users do.

PhaseActivities
1. ScopeDefine attack surfaces and threat models
2. ProbeExecute jailbreaks, bias probes, edge cases
3. AnalyzeCategorize findings by severity
4. RemediateUpdate prompts, filters, and guardrails
5. RetestVerify mitigations, check for regressions
🌍 Assemble a diverse red team — different backgrounds discover different vulnerabilities.

Automated Evaluation Pipelines

Manual evaluation doesn't scale. Build pipelines that run continuously:

  1. Test dataset management — versioned JSONL with queries, responses, context
  2. Multi-metric scoring — quality + safety evaluators in parallel
  3. Threshold gating — minimum acceptable scores per metric
  4. CI/CD integration — fail the pipeline if thresholds aren't met
  5. Trend monitoring — track scores over time to catch drift
Run evaluations on every model/prompt change AND on a regular schedule against production traffic samples.

Python SDK — azure-ai-evaluation

from azure.ai.evaluation import (
    evaluate,
    GroundednessEvaluator,
    RelevanceEvaluator,
    CoherenceEvaluator,
    FluencyEvaluator,
)

results = evaluate(
    data="test_dataset.jsonl",
    evaluators={
        "groundedness": GroundednessEvaluator(model_config=cfg),
        "relevance":    RelevanceEvaluator(model_config=cfg),
        "coherence":    CoherenceEvaluator(model_config=cfg),
        "fluency":      FluencyEvaluator(model_config=cfg),
    },
)

print(results["metrics"])
# {"groundedness": 4.2, "relevance": 4.5, ...}

🛠️ Mini Hack: Evaluate Your Fine-Tuned Model

  1. Prepare a test dataset (≥ 20 rows, JSONL) with normal + adversarial inputs
  2. Run quality evaluators — groundedness, relevance, coherence, fluency
  3. Run safety evaluators — violence, sexual, self-harm, hate
  4. Check thresholds — quality ≥ 4.0, safety scores = 0
  5. Configure content filter in AI Foundry portal
  6. Red-team test — 5 adversarial prompts, document results
✅ Success: Quality avg ≥ 4.0 · Zero safety violations · Filter blocks adversarial inputs · Red-team findings documented

✅ Key Takeaways

  • Five quality metrics: groundedness, relevance, coherence, fluency, similarity
  • Four safety categories with configurable severity thresholds
  • Prompt Shield defends against jailbreaks and prompt injection
  • Custom blocklists extend built-in safety for your domain
  • Red-teaming is essential — automate what you can, but human creativity is irreplaceable
  • Automated pipelines with threshold gates keep quality high continuously
  • The azure-ai-evaluation SDK is your primary tool
