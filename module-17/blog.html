Overview

AI Agents in Azure AI Foundry are intelligent assistants powered by large language models that can reason, plan, and take actions using a set of configurable tools. Unlike simple chat completions, agents maintain state through threads, can invoke tools autonomously, and handle multi-step workflows.

In this module, you'll learn to create your first agent from the ground up — from crafting effective instructions to integrating built-in tools, and from playground experimentation to production-ready SDK implementations.

Prerequisites

You need an Azure AI Foundry project with a deployed GPT-4o or GPT-4.1 model. Module 16 covers project setup if you haven't completed it yet.

Understanding Agent Concepts

Before building an agent, it's essential to understand the core primitives of the Azure AI Foundry Agent Service:

🤖

Agent

An entity configured with instructions, a model, and tools to perform tasks.

🧵

Thread

A conversation session storing message history between user and agent.

💬

Message

A single unit of communication — text, image, or file — within a thread.

▶️

Run

An execution of the agent on a thread, invoking the model and tools.

How Agents Process Requests

  1. A user sends a message to a thread
  2. A run is created, invoking the agent on the thread
  3. The agent reads the thread, follows its instructions, and decides which tools to call
  4. Tool outputs are appended to the thread, and the model generates a final response
  5. The response is added to the thread as an assistant message

Configuring Instructions & Persona

Agent instructions are the system prompt that defines your agent's behavior, personality, and capabilities. Well-crafted instructions are the single most important factor in agent quality.

Anatomy of Effective Instructions

Instructions
# Agent: Financial Data Analyst

## Role
You are a senior financial data analyst specializing in
market trends and portfolio analysis.

## Capabilities
- Analyze CSV and Excel files using Code Interpreter
- Search for current market data using Bing Search
- Generate visualizations (charts, graphs, tables)
- Provide data-driven investment insights

## Behavior Guidelines
- Always cite data sources and explain methodology
- Use clear, professional language
- When uncertain, state assumptions explicitly
- Present numeric results with appropriate precision
- Proactively suggest follow-up analyses

## Constraints
- Never provide specific buy/sell recommendations
- Always include disclaimers for financial advice
- Limit responses to factual, data-driven insights
Pro Tip: Instruction Engineering

Structure instructions with clear sections using Markdown headers. Agents perform best when instructions are specific, organized, and include both capabilities and constraints.

Key Principles for Agent Instructions

Principle Good Example Avoid
Be Specific "Analyze sales data and produce a bar chart comparing Q1 vs Q2" "Look at the data"
Define Boundaries "Only answer questions about Azure services" No constraints defined
Set Tone "Respond in a professional, concise manner" Ambiguous persona
Handle Errors "If data is missing, ask the user for clarification" No error handling guidance

Built-in Tools

Azure AI Foundry agents come with three powerful built-in tools that extend agent capabilities beyond text generation:

🖥️ Code Interpreter

Code Interpreter enables your agent to write and execute Python code in a sandboxed environment. It can process uploaded files, perform calculations, generate charts, and transform data.

Python
from azure.ai.projects import AIProjectClient
from azure.ai.projects.models import CodeInterpreterTool
from azure.identity import DefaultAzureCredential

# Create client
client = AIProjectClient(
    endpoint="https://<your-project>.services.ai.azure.com",
    credential=DefaultAzureCredential()
)

# Create agent with Code Interpreter
agent = client.agents.create_agent(
    model="gpt-4o",
    name="data-analyst",
    instructions="You are a data analyst. Analyze files and create visualizations.",
    tools=[CodeInterpreterTool()]
)
print(f"Agent created: {agent.id}")

🔍 File Search

File Search provides built-in RAG (Retrieval Augmented Generation) capability. Upload documents to a vector store and the agent automatically searches them to ground its responses.

Python
from azure.ai.projects.models import FileSearchTool

# Upload files and create vector store
uploaded_file = client.agents.upload_file_and_poll(
    file_path="product_docs.pdf",
    purpose="agents"
)
vector_store = client.agents.create_vector_store_and_poll(
    name="product-knowledge",
    file_ids=[uploaded_file.id]
)

# Create agent with File Search
agent = client.agents.create_agent(
    model="gpt-4o",
    name="support-agent",
    instructions="Answer questions using the product documentation.",
    tools=[FileSearchTool()],
    tool_resources={"file_search": {"vector_store_ids": [vector_store.id]}}
)

🌐 Bing Search (Grounding)

Bing Search enables agents to access real-time information from the internet, grounding responses with current data.

Python
from azure.ai.projects.models import BingGroundingTool

# Get your Bing Grounding connection
bing_connection = client.connections.get("bing-grounding")

# Create agent with Bing Search
agent = client.agents.create_agent(
    model="gpt-4o",
    name="research-agent",
    instructions="Search the web for the latest information to answer questions.",
    tools=[BingGroundingTool(connection_id=bing_connection.id)]
)
Bing Search Requires a Connection

To use Bing Grounding, you must first create a Grounding with Bing Search resource in the Azure portal and link it as a connection in your AI Foundry project.

Tool Comparison

Feature Code Interpreter File Search Bing Search
Primary Use Data analysis, code execution Document Q&A (RAG) Real-time web data
Input Files (CSV, Excel, JSON) PDFs, DOCX, TXT, MD Natural language queries
Output Text, charts, transformed files Grounded text answers Text with web citations
State Sandboxed per session Persistent vector store Stateless (per query)
Extra Cost Per session time Per GB stored + queries Per Bing API call

Testing Agents in the Playground

The Azure AI Foundry portal includes an interactive Agent Playground that lets you create, configure, and test agents without writing any code.

Using the Playground

  1. Navigate to the Agent Playground

    Open your Azure AI Foundry project in the portal. Select Agents from the left navigation menu, then click + New Agent.

  2. Configure the Agent

    Enter the agent name, select a model deployment (e.g., gpt-4o), and write your system instructions in the Instructions field.

  3. Enable Tools

    Toggle on the desired tools — Code Interpreter, File Search, or Bing Search. For File Search, upload files to create a vector store.

  4. Test in the Chat Panel

    Use the right-side chat panel to send test messages. The playground displays tool invocations, code execution, and intermediate steps in real time.

  5. Iterate and Refine

    Adjust instructions based on test results. You can modify instructions on the fly without recreating the agent.

Playground Tip

Enable "Show tool calls" in the playground to see exactly which tools the agent calls and how it processes data. This transparency helps you debug instruction issues.

Agent SDK — Python

The azure-ai-projects Python package provides a comprehensive SDK for creating and managing agents programmatically.

Installation

Bash
pip install azure-ai-projects azure-identity

Complete Agent Lifecycle

Python
from azure.ai.projects import AIProjectClient
from azure.ai.projects.models import CodeInterpreterTool
from azure.identity import DefaultAzureCredential

# 1. Initialize the client
client = AIProjectClient(
    endpoint="https://<your-project>.services.ai.azure.com",
    credential=DefaultAzureCredential()
)

# 2. Create an agent
agent = client.agents.create_agent(
    model="gpt-4o",
    name="data-analyst",
    instructions="""You are a data analyst. When given data files:
    1. Load and inspect the data
    2. Provide summary statistics
    3. Create relevant visualizations
    4. Highlight key insights""",
    tools=[CodeInterpreterTool()]
)

# 3. Create a thread
thread = client.agents.create_thread()

# 4. Upload a file and add message
file = client.agents.upload_file_and_poll(
    file_path="sales_data.csv",
    purpose="agents"
)
message = client.agents.create_message(
    thread_id=thread.id,
    role="user",
    content="Analyze this sales data and create a trend chart.",
    attachments=[{"file_id": file.id, "tools": [{"type": "code_interpreter"}]}]
)

# 5. Run the agent and stream results
with client.agents.create_stream(
    thread_id=thread.id,
    agent_id=agent.id
) as stream:
    for event in stream:
        if event.type == "thread.message.delta":
            print(event.data.delta.content[0].text.value, end="")

# 6. Clean up
client.agents.delete_agent(agent.id)
print("\nAgent deleted.")

Complete Agent Lifecycle in C#

C#
using Azure.AI.Projects;
using Azure.Identity;

// 1. Initialize the client
var connectionString = "<your-foundry-project-connection-string>";
var client = new AgentsClient(connectionString, new DefaultAzureCredential());

// 2. Create an agent with Code Interpreter
var agent = await client.CreateAgentAsync(
    model: "gpt-4o",
    name: "data-analyst-dotnet",
    instructions: "Analyze data files and provide insights with charts.",
    tools: [new CodeInterpreterToolDefinition()]
);

// 3. Create a thread and add a message
var thread = await client.CreateThreadAsync();
await client.CreateMessageAsync(
    thread.Value.Id,
    MessageRole.User,
    "Analyze the uploaded sales data and identify trends."
);

// 4. Run the agent
var run = await client.CreateRunAsync(thread.Value.Id, agent.Value.Id);

// 5. Poll for completion
do
{
    await Task.Delay(1000);
    run = await client.GetRunAsync(thread.Value.Id, run.Value.Id);
} while (run.Value.Status == RunStatus.InProgress
    || run.Value.Status == RunStatus.Queued);

// 6. Retrieve messages
var messages = await client.GetMessagesAsync(thread.Value.Id);
foreach (var msg in messages.Value.Data)
{
    Console.WriteLine($"[{msg.Role}]: {msg.Content[0]}");
}

// 7. Clean up
await client.DeleteAgentAsync(agent.Value.Id);

🧪 Mini Hack: Data Analyst Agent

Put everything together! Build a data analyst agent with Code Interpreter that can analyze CSV data, compute statistics, and generate visualizations.

Challenge Objectives

Create an agent that can: (1) Load a CSV file, (2) Generate summary statistics, (3) Create a bar chart of top categories, (4) Identify outliers and trends. Target completion: 30 minutes.

Step-by-Step Implementation

Python
import os
from azure.ai.projects import AIProjectClient
from azure.ai.projects.models import CodeInterpreterTool, FilePurpose
from azure.identity import DefaultAzureCredential

# ── Configuration ──
ENDPOINT = os.environ["AZURE_AI_FOUNDRY_ENDPOINT"]
MODEL = "gpt-4o"

INSTRUCTIONS = """You are an expert data analyst. When given a dataset:

1. **Inspect** — Load the file, show shape, columns, dtypes, and first 5 rows.
2. **Summarize** — Compute mean, median, std dev, min, max for numeric columns.
3. **Visualize** — Create a bar chart of the top 10 categories by total value.
   Use matplotlib with a clean style, labeled axes, and a title.
4. **Insights** — Identify the top 3 key trends, outliers, or patterns.

Always explain your methodology and present results clearly."""

# ── Build the Agent ──
def main():
    client = AIProjectClient(
        endpoint=ENDPOINT,
        credential=DefaultAzureCredential()
    )

    # Create agent
    agent = client.agents.create_agent(
        model=MODEL,
        name="mini-hack-data-analyst",
        instructions=INSTRUCTIONS,
        tools=[CodeInterpreterTool()]
    )
    print(f"✅ Agent created: {agent.id}")

    # Upload data file
    data_file = client.agents.upload_file_and_poll(
        file_path="sample_sales.csv",
        purpose=FilePurpose.AGENTS
    )
    print(f"📁 File uploaded: {data_file.id}")

    # Create thread with the file
    thread = client.agents.create_thread()
    client.agents.create_message(
        thread_id=thread.id,
        role="user",
        content="Analyze this sales dataset. Show summary stats, "
                "create a chart of top products by revenue, "
                "and share the 3 most important insights.",
        attachments=[{
            "file_id": data_file.id,
            "tools": [{"type": "code_interpreter"}]
        }]
    )

    # Run and stream the response
    print("\n🔄 Running agent...\n")
    with client.agents.create_stream(
        thread_id=thread.id,
        agent_id=agent.id
    ) as stream:
        for event in stream:
            if hasattr(event, "data") and hasattr(event.data, "delta"):
                for content in event.data.delta.content:
                    if hasattr(content, "text"):
                        print(content.text.value, end="")

    # Retrieve generated images
    messages = client.agents.list_messages(thread_id=thread.id)
    for msg in messages.data:
        for content in msg.content:
            if hasattr(content, "image_file"):
                file_id = content.image_file.file_id
                file_content = client.agents.get_file_content(file_id)
                with open("chart_output.png", "wb") as f:
                    for chunk in file_content:
                        f.write(chunk)
                print("\n📊 Chart saved to chart_output.png")

    # Cleanup
    client.agents.delete_agent(agent.id)
    print("🧹 Agent cleaned up.")

if __name__ == "__main__":
    main()

Best Practices

Agent Design Patterns

  • Single Responsibility: Each agent should have one clear purpose. Compose multiple specialized agents rather than one omniscient agent.
  • Fail Gracefully: Include error-handling guidance in instructions so agents communicate issues clearly instead of hallucinating.
  • Version Instructions: Store instructions in files or config and version-control them alongside your code.
  • Minimize Tool Scope: Only enable tools the agent actually needs. This reduces cost and improves response accuracy.

Security Considerations

  • Use Managed Identity (DefaultAzureCredential) instead of API keys in production
  • Enable content safety filters on your model deployment
  • Review and limit the data uploaded to Code Interpreter sessions
  • Use Azure RBAC to control who can create and manage agents
Cost Optimization

Code Interpreter sessions are billed per session-minute. Design your agents to complete tasks efficiently and always call delete_agent() when done to avoid orphaned resources.
