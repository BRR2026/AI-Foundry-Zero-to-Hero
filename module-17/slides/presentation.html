What Is an AI Agent?

An intelligent entity that can reason, plan, and act using tools — going beyond simple chat completions.

Instructions
+
Model (GPT-4o)
+
Tools
=
Agent 🤖
Key Difference: Unlike chat completions, agents maintain state through threads, can invoke tools autonomously, and handle multi-step workflows.

Agent Primitives

🤖

Agent

Entity configured with instructions, a model, and tools to perform tasks.

🧵

Thread

Conversation session that stores message history between user and agent.

💬

Message

A unit of communication — text, image, or file — within a thread.

▶️

Run

An execution of the agent on a thread, invoking the model and tools.

How Agents Process Requests

1️⃣ User sends a message to a thread
2️⃣ A run is created → agent reads the thread
3️⃣ Agent decides which tools to invoke
4️⃣ Tool outputs appended → model generates response
5️⃣ Response added as assistant message

Crafting Agent Instructions

Instructions are the system prompt — the most important factor in agent quality.

# Agent: Financial Data Analyst

## Role
You are a senior data analyst specializing in market trends.

## Capabilities
- Analyze CSV/Excel files using Code Interpreter
- Generate visualizations (charts, graphs)
- Provide data-driven insights

## Constraints
- Never provide buy/sell recommendations
- Always cite data sources

Built-in Tools

🖥️

Code Interpreter

Execute Python in a sandbox. Analyze data, create charts, transform files.

🔍

File Search

Built-in RAG over uploaded documents. Vector store backed.

🌐

Bing Search

Real-time web search for grounding responses with current data.

Pro Tip: You can combine multiple tools on a single agent. The model automatically decides which tool to invoke for each query.

Code Interpreter

Sandboxed Python execution with pre-installed data science libraries.

from azure.ai.projects import AIProjectClient
from azure.ai.projects.models import CodeInterpreterTool
from azure.identity import DefaultAzureCredential

client = AIProjectClient(
    endpoint="https://<project>.services.ai.azure.com",
    credential=DefaultAzureCredential()
)

agent = client.agents.create_agent(
    model="gpt-4o",
    name="data-analyst",
    instructions="Analyze data and create visualizations.",
    tools=[CodeInterpreterTool()]
)

File Search & Bing Search

File Search (RAG)

  • Upload PDFs, DOCX, TXT
  • Automatic chunking & embedding
  • Persistent vector stores
  • Great for knowledge bases

Bing Grounding

  • Real-time web information
  • Requires Bing resource + connection
  • Responses include citations
  • Ideal for current events

Testing in the Playground

  1. Open AI Foundry portal → Agents → + New Agent
  2. Set name, model (gpt-4o), and paste instructions
  3. Toggle on desired tools (Code Interpreter, etc.)
  4. Send test messages in the chat panel
  5. Enable "Show tool calls" for transparency
  6. Iterate instructions based on results
Debugging Tip: If the agent isn't calling the right tool, make your instructions more explicit about when to use each tool.
