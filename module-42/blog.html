Creating a Vector Index with Azure AI Search

Azure AI Search is the backbone of the retrieval component in your RAG pipeline. When you create a vector index in Azure AI Foundry, it automatically provisions and manages an Azure AI Search resource behind the scenes.

Python create_vector_index.py 
from azure.identity import DefaultAzureCredential
from azure.ai.projects import AIProjectClient
from azure.ai.projects.models import (
    ConnectionType,
    IndexSource,
    AzureAISearchSource,
)

# Initialize the AI Foundry project client
credential = DefaultAzureCredential()
project_client = AIProjectClient(
    credential=credential,
    subscription_id="<your-subscription-id>",
    resource_group_name="rg-ai-foundry-capstone",
    project_name="rag-pipeline-project",
)

# Get the Azure AI Search connection from the project
search_connection = project_client.connections.get(
    connection_name="my-search-connection"
)

print(f"Connected to AI Search: {search_connection.name}")

Document Processing & Chunking Strategy

The quality of your RAG pipeline depends heavily on how you process and chunk your documents. Azure AI Foundry supports multiple chunking strategies:

Chunking Strategy Best For Chunk Size
Fixed-size Uniform documents, logs 512–1024 tokens
Sentence-based Natural language content 3–5 sentences
Semantic Complex, mixed documents Variable (model-driven)
Page / Section PDFs, structured reports By page or heading
💡
Best Practice: Overlap Your Chunks

Always configure chunk overlap (typically 10–15% of chunk size) to preserve context across chunk boundaries. This significantly improves retrieval accuracy for questions that span multiple chunks.

Python index_documents.py 
from azure.ai.projects.models import (
    VectorStoreDataSource, VectorStoreDataSourceAssetType,
    VectorStoreStaticChunkingStrategyRequest,
    VectorStoreStaticChunkingStrategyOptions,
)

# Create vector store with optimal chunking
vector_store = project_client.agents.create_vector_store_and_poll(
    name="capstone-knowledge-base",
    data_sources=[data_source],
    chunking_strategy=VectorStoreStaticChunkingStrategyRequest(
        static=VectorStoreStaticChunkingStrategyOptions(
            max_chunk_size_tokens=800,
            chunk_overlap_tokens=100,  # ~12.5% overlap
        )
    ),
)

print(f"Vector store created: {vector_store.id}")
print(f"File count: {vector_store.file_counts}")

⚠️
Token Limits & Rate Planning

For production RAG pipelines, use provisioned throughput deployments to guarantee consistent tokens-per-minute. Standard deployments use shared capacity and may throttle under load. Monitor your deployment's TPM usage in the Azure AI Foundry portal.

Using the Azure AI Foundry Agent with File Search

The simplest way to build a RAG pipeline in Azure AI Foundry is by using the Agents API with the built-in file_search tool. This approach handles retrieval, re-ranking, and context injection automatically.

Python rag_agent_pipeline.py 
from azure.ai.projects import AIProjectClient
from azure.ai.projects.models import (
    FileSearchTool,
    FileSearchToolResource,
    VectorStoreDataSource,
    VectorStoreDataSourceAssetType,
)
from azure.identity import DefaultAzureCredential

# Initialize the project client
project_client = AIProjectClient(
    credential=DefaultAzureCredential(),
    subscription_id="<your-subscription-id>",
    resource_group_name="rg-ai-foundry-capstone",
    project_name="rag-pipeline-project",
)

# Create a vector store from a data asset
ds = VectorStoreDataSource(
    asset_identifier=project_client.scope["data_asset_uri"],
    asset_type=VectorStoreDataSourceAssetType.URI_ASSET,
)
vector_store = project_client.agents.create_vector_store_and_poll(
    data_sources=[ds], name="rag-knowledge-base"
)

# Create the RAG agent with file_search tool
agent = project_client.agents.create_agent(
    model="gpt-4o",
    name="rag-pipeline-agent",
    instructions="""You are a knowledgeable assistant that answers questions
using ONLY the provided knowledge base. Always cite your sources.
If the answer is not found in the knowledge base, say so clearly.""",
    tools=FileSearchTool().definitions,
    tool_resources={
        "file_search": FileSearchToolResource(
            vector_store_ids=[vector_store.id]
        )
    },
)

# Create a thread and send a query
thread = project_client.agents.create_thread()
message = project_client.agents.create_message(
    thread_id=thread.id,
    role="user",
    content="What are the best practices for scaling RAG pipelines?",
)

# Run the agent and get the response
run = project_client.agents.create_and_process_run(
    thread_id=thread.id, agent_id=agent.id
)

# Retrieve and display the response with citations
messages = project_client.agents.list_messages(thread_id=thread.id)
for msg in messages:
    if msg.role == "assistant":
        for content in msg.content:
            print(content.text.value)
            for annotation in content.text.annotations:
                print(f"  📎 Source: {annotation.file_citation}")

Custom RAG with Direct SDK (Advanced)

For more control, build a custom RAG pipeline using direct SDK calls. This gives you fine-grained control over retrieval, re-ranking, and prompt construction.

Python custom_rag_pipeline.py 
from azure.search.documents import SearchClient
from azure.search.documents.models import VectorizableTextQuery

# Step 1: Hybrid search (vector + keyword)
results = search_client.search(
    search_text=user_query,
    vector_queries=[VectorizableTextQuery(
        text=user_query, k_nearest_neighbors=5,
        fields="content_vector")],
    select=["title", "content", "source"], top=5,
)

# Step 2: Assemble context from results
context = "\n---\n".join(
    [f"[Source: {r['source']}]\n{r['content']}" for r in results])

# Step 3: Generate grounded response
response = chat_client.complete(
    model="gpt-4o",
    messages=[
        {"role": "system", "content": f"Answer using ONLY this context:\n{context}"},
        {"role": "user", "content": user_query},
    ],
    temperature=0.3, max_tokens=1000,
)
✅
Hybrid Search FTW

Always use hybrid search (vector + keyword) rather than pure vector search. Research shows hybrid search improves recall by 10–15% compared to vector-only retrieval, especially for queries with specific technical terms or identifiers.

✅ Testing & Validating the Pipeline

A RAG pipeline is only as good as its evaluation. Azure AI Foundry provides built-in evaluation tools to measure retrieval quality, generation accuracy, and pipeline performance.

Key Evaluation Metrics

Metric What It Measures Target Range
Groundedness Are responses grounded in retrieved context? > 4.0 / 5.0
Relevance How relevant is the response to the query? > 4.0 / 5.0
Coherence Is the response well-structured and logical? > 4.0 / 5.0
Retrieval Score Are the retrieved documents relevant? > 3.5 / 5.0
Fluency How natural and readable is the response? > 4.0 / 5.0

Running Evaluations with the AI Foundry SDK

Python evaluate_pipeline.py 
from azure.ai.evaluation import (
    GroundednessEvaluator, RelevanceEvaluator,
    CoherenceEvaluator, FluencyEvaluator, evaluate,
)

# Initialize evaluators with project scope
model_config = project_client.scope
groundedness = GroundednessEvaluator(model_config)
relevance = RelevanceEvaluator(model_config)
coherence = CoherenceEvaluator(model_config)
fluency = FluencyEvaluator(model_config)

# Run batch evaluation
results = evaluate(
    data="test_dataset.jsonl",
    evaluators={
        "groundedness": groundedness,
        "relevance": relevance,
        "coherence": coherence,
        "fluency": fluency,
    },
    evaluator_config={"default": {
        "query": "${data.query}",
        "context": "${data.context}",
        "response": "${data.response}",
    }},
    azure_ai_project=project_client.scope,
)

for metric, score in results["metrics"].items():
    print(f"  {metric}: {score:.2f}")

Evaluation Dataset Format

JSON test_dataset.jsonl 
{"query": "How do I configure autoscaling?", "context": "Autoscaling in Azure allows...", "response": "To configure autoscaling...", "ground_truth": "Configure autoscaling by..."}
{"query": "What are managed identities?", "context": "Managed identities provide...", "response": "Managed identities are...", "ground_truth": "Managed identities enable..."}

📌 Key Takeaways

  1. Data quality drives RAG quality — invest time in chunking strategy, overlap configuration, and document processing before building the pipeline.
  2. Azure AI Foundry Agents API simplifies RAG by handling retrieval, re-ranking, and context injection via the file_search tool.
  3. Hybrid search (vector + keyword) consistently outperforms pure vector search for RAG retrieval accuracy.
  4. Evaluation is not optional — use groundedness, relevance, and coherence evaluators as quality gates before deploying to production.
  5. Model selection matters — match your model to your latency, cost, and quality requirements. Use provisioned throughput for production workloads.
