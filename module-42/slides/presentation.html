✂️ Chunking Strategies

How you chunk determines retrieval quality

StrategyChunk SizeBest ForTrade-off
Fixed-Size512–1024 tokensUniform docs, logsSimple but may split context
Sentence-Based3–5 sentencesProse, natural languageBetter boundaries, variable size
SemanticVariableComplex documentsBest quality, higher compute
Page / SectionBy headingStructured reports, PDFsPreserves structure, large chunks
⚡ Best Practice

Always configure 10–15% chunk overlap to preserve context at boundaries. This is the single most impactful setting for retrieval quality.

🔢 Creating the Vector Index

# Create a vector store with optimal chunking vector_store = project_client.agents.create_vector_store_and_poll( name="capstone-knowledge-base", data_sources=[data_source], chunking_strategy=VectorStoreStaticChunkingStrategyRequest( static=VectorStoreStaticChunkingStrategyOptions( max_chunk_size_tokens=800, chunk_overlap_tokens=100, # ~12.5% overlap ) ), ) print(f"Vector store: {vector_store.id}") print(f"Files indexed: {vector_store.file_counts}")

The SDK handles document cracking, embedding generation, and index creation automatically.

🤖 Agent-Based RAG Pipeline

The simplest path: use the Agents API with file_search

# Create a RAG agent with automatic retrieval agent = project_client.agents.create_agent( model="gpt-4o", name="rag-pipeline-agent", instructions="Answer using ONLY the knowledge base. Cite sources.", tools=FileSearchTool().definitions, tool_resources={ "file_search": FileSearchToolResource( vector_store_ids=[vector_store.id] ) }, ) # Query → auto-retrieval → grounded response thread = project_client.agents.create_thread() project_client.agents.create_message(thread_id=thread.id, role="user", content="What are RAG best practices?") run = project_client.agents.create_and_process_run(thread_id=thread.id, agent_id=agent.id)

The Agents API handles: embedding the query → searching the index → re-ranking results → injecting context → generating a grounded response with citations.

⚙️ Custom RAG Pipeline

For full control over each stage

1. Retrieve

Hybrid search with Azure AI Search: vector similarity + BM25 keyword matching

2. Augment

Build a system prompt with retrieved context, source citations, and grounding instructions

3. Generate

Call GPT-4o with low temperature (0.1–0.3) for factual, grounded responses

Hybrid Search Advantage

Vector + keyword search improves recall by 10–15% compared to pure vector search. Always use hybrid for production RAG.

🔍 Hybrid Search Implementation

# Hybrid search: vector + keyword for best recall results = search_client.search( search_text=user_query, # keyword component vector_queries=[ VectorizableTextQuery( text=user_query, # vector component k_nearest_neighbors=5, fields="content_vector", ) ], select=["title", "content", "source"], top=5, ) # Build context from results context = "\n---\n".join( [f"[{r['source']}] {r['content']}" for r in results] )

Azure AI Search automatically handles re-ranking when using hybrid mode with Reciprocal Rank Fusion (RRF).

🧪 Running Evaluations

from azure.ai.evaluation import evaluate, GroundednessEvaluator, RelevanceEvaluator, CoherenceEvaluator results = evaluate( data="test_dataset.jsonl", evaluators={ "groundedness": GroundednessEvaluator(model_config), "relevance": RelevanceEvaluator(model_config), "coherence": CoherenceEvaluator(model_config), }, evaluator_config={"default": { "query": "${data.query}", "context": "${data.context}", "response": "${data.response}", }}, azure_ai_project=project_client.scope, ) for metric, score in results["metrics"].items(): print(f" {metric}: {score:.2f}")

Results are automatically logged to your Azure AI Foundry project for trend analysis and comparison across runs.

📌 Key Takeaways

  • Data quality drives RAG quality — invest in chunking strategy and overlap configuration
  • Agents API with file_search — simplest path to production RAG in Azure AI Foundry
  • Hybrid search (vector + keyword) — 10–15% better recall than vector-only search
  • Evaluation is mandatory — groundedness, relevance, and coherence are your quality gates
  • Match model to requirements — balance latency, cost, quality, and context window
  • Continuous evaluation — integrate evaluations into CI/CD with automated quality gates
Remember

A RAG pipeline is only as good as its weakest link. Test every stage independently, then test the full pipeline end-to-end.
