Azure AI Foundry agents support a rich ecosystem of tools that extend their capabilities far beyond simple text generation. Each tool type serves a distinct purpose:

The agent's LLM examines the user's message, determines which tool(s) to invoke, calls them, processes the results, and generates a final response. This tool-calling loop can execute multiple tools in sequence or parallel within a single conversation turn.

1. Code Interpreter Tool

The Code Interpreter tool gives your agent the ability to write and execute Python code in a sandboxed environment. It is purpose-built for data analysis, mathematical computation, and visualization — tasks that LLMs can describe but cannot natively execute.

Capabilities

  • Data analysis — Process CSV, Excel, JSON, and Parquet files with pandas, numpy, and scipy
  • Visualization — Generate charts and plots with matplotlib and seaborn, returned as downloadable images
  • Math & statistics — Execute precise calculations, regressions, and statistical tests
  • File generation — Create output files (CSV, XLSX, PDF) the user can download
  • Iterative debugging — Automatically retry failed code with corrected syntax

Enabling Code Interpreter

Python
from azure.ai.projects import AIProjectClient
from azure.ai.projects.models import CodeInterpreterTool
from azure.identity import DefaultAzureCredential

project_client = AIProjectClient(
    credential=DefaultAzureCredential(),
    endpoint="https://your-project.services.ai.azure.com",
)

# Create an agent with Code Interpreter enabled
code_interpreter = CodeInterpreterTool()

agent = project_client.agents.create_agent(
    model="GPT-5.1",
    name="data-analyst",
    instructions="You are a data analyst. Analyze uploaded files and create visualizations.",
    tools=code_interpreter.definitions,
)
print(f"Agent created: {agent.id}")

Uploading Files for Analysis

Python
# Upload a CSV file for the agent to analyze
uploaded_file = project_client.agents.upload_file_and_poll(
    file_path="sales_data.csv",
    purpose="agents",
)

# Create a thread with the file attached
thread = project_client.agents.create_thread()

message = project_client.agents.create_message(
    thread_id=thread.id,
    role="user",
    content="Analyze trends in this sales data and create a bar chart of monthly revenue.",
    attachments=[{
        "file_id": uploaded_file.id,
        "tools": [{"type": "code_interpreter"}],
    }],
)

# Run the agent and wait for completion
run = project_client.agents.create_and_process_run(
    thread_id=thread.id,
    agent_id=agent.id,
)
Sandbox Limits

Code Interpreter runs in a stateless sandbox. Each execution session has limited memory, a timeout of ~3 minutes, and no internet access. Pre-installed libraries include pandas, numpy, matplotlib, seaborn, scipy, and scikit-learn.

2. File Search Tool

The File Search tool enables semantic retrieval over your uploaded documents. It automatically creates vector store embeddings, chunks your files, and performs similarity search to retrieve the most relevant passages — giving your agent access to knowledge that isn't in its training data.

How It Works

  1. Create a Vector Store

    Upload documents (PDF, DOCX, TXT, MD, HTML) to a vector store. Azure AI Foundry automatically chunks and embeds the content.

  2. Attach to Agent

    Associate the vector store with your agent's File Search tool. The agent can now search across all uploaded documents.

  3. Semantic Query

    When a user asks a question, the agent converts it to a semantic query, retrieves relevant chunks, and synthesizes an answer with citations.

Creating a Vector Store & Agent

Python
from azure.ai.projects.models import FileSearchTool

# Upload documentation files
docs = ["product-guide.pdf", "api-reference.md", "faq.docx"]
file_ids = []
for doc in docs:
    f = project_client.agents.upload_file_and_poll(
        file_path=doc, purpose="agents"
    )
    file_ids.append(f.id)

# Create a vector store from the uploaded files
vector_store = project_client.agents.create_vector_store_and_poll(
    name="product-docs",
    file_ids=file_ids,
)
print(f"Vector store created: {vector_store.id} ({vector_store.file_counts})")

# Create agent with File Search tool
file_search = FileSearchTool(
    vector_store_ids=[vector_store.id]
)

agent = project_client.agents.create_agent(
    model="GPT-5.1",
    name="doc-assistant",
    instructions="Answer questions using the product documentation. Always cite sources.",
    tools=file_search.definitions,
    tool_resources=file_search.resources,
)
Supported File Formats

File Search supports .pdf, .docx, .pptx, .txt, .md, .html, .json, .csv, and more. Each vector store can hold up to 10,000 files with a maximum total of 100 GB.

Vector Store Configuration Options

Python
# Advanced: configure chunking strategy
from azure.ai.projects.models import (
    VectorStoreChunkingStrategyRequest,
    VectorStoreStaticChunkingStrategy,
)

vector_store = project_client.agents.create_vector_store_and_poll(
    name="technical-docs",
    file_ids=file_ids,
    chunking_strategy=VectorStoreChunkingStrategyRequest(
        type="static",
        static=VectorStoreStaticChunkingStrategy(
            max_chunk_size_tokens=800,
            chunk_overlap_tokens=400,
        ),
    ),
)

3. Azure Functions as Tools

When your agent needs custom logic — querying a database, processing a payment, sending an email, or any business-specific operation — you can expose Azure Functions as agent tools. This gives your agent access to serverless compute with your own code running behind the scenes.

Creating a Function Tool Definition

First, define the function schema so the agent knows what parameters to pass:

Python
from azure.ai.projects.models import FunctionTool

# Define the function tool schema
functions = FunctionTool(
    definitions=[
        {
            "name": "get_customer_orders",
            "description": "Retrieve recent orders for a customer by their ID.",
            "parameters": {
                "type": "object",
                "properties": {
                    "customer_id": {
                        "type": "string",
                        "description": "The unique customer identifier",
                    },
                    "limit": {
                        "type": "integer",
                        "description": "Max number of orders to return (default: 10)",
                    },
                },
                "required": ["customer_id"],
            },
        }
    ]
)

agent = project_client.agents.create_agent(
    model="GPT-5.1",
    name="customer-support",
    instructions="You are a customer support agent. Use tools to look up order information.",
    tools=functions.definitions,
)

Handling Function Calls

When the agent decides to call your function, you intercept the call, execute your logic, and return the result:

Python
import json
import requests

def handle_tool_call(tool_call):
    """Route agent tool calls to Azure Function endpoints."""
    name = tool_call.function.name
    args = json.loads(tool_call.function.arguments)

    if name == "get_customer_orders":
        response = requests.get(
            "https://my-func-app.azurewebsites.net/api/orders",
            params=args,
            headers={"x-functions-key": FUNCTION_KEY},
        )
        return response.text

    return json.dumps({"error": f"Unknown function: {name}"})

# Process the run and handle requires_action status
run = project_client.agents.create_run(
    thread_id=thread.id,
    agent_id=agent.id,
)

# Poll until run completes or requires action
while run.status in ("queued", "in_progress", "requires_action"):
    if run.status == "requires_action":
        tool_outputs = []
        for call in run.required_action.submit_tool_outputs.tool_calls:
            result = handle_tool_call(call)
            tool_outputs.append({
                "tool_call_id": call.id,
                "output": result,
            })
        run = project_client.agents.submit_tool_outputs_to_run(
            thread_id=thread.id,
            run_id=run.id,
            tool_outputs=tool_outputs,
        )
    run = project_client.agents.get_run(thread_id=thread.id, run_id=run.id)
Managed Identity

In production, use Azure Managed Identity instead of function keys. Configure your Azure Function with Entra ID authentication and assign the agent's managed identity the appropriate role.

4. OpenAPI Tools

OpenAPI tools let your agent connect to any external REST API by providing an OpenAPI (Swagger) specification. The agent reads the schema, understands the available endpoints, and calls them with the correct parameters — no custom handler code required.

Defining an OpenAPI Tool

Python
from azure.ai.projects.models import OpenApiTool, OpenApiAnonymousAuthDetails

# Load your OpenAPI specification
with open("weather-api.json", "r") as f:
    openapi_spec = json.load(f)

# Create the OpenAPI tool with anonymous auth
weather_tool = OpenApiTool(
    name="weather_api",
    description="Get current weather and forecasts for any location.",
    spec=openapi_spec,
    auth=OpenApiAnonymousAuthDetails(),
)

agent = project_client.agents.create_agent(
    model="GPT-5.1",
    name="weather-assistant",
    instructions="Provide weather information using the weather API tool.",
    tools=weather_tool.definitions,
)

Authentication Options

OpenAPI tools support multiple authentication mechanisms for connecting to secured APIs:

Auth Type Class Use Case
Anonymous OpenApiAnonymousAuthDetails Public APIs with no authentication
API Key OpenApiApiKeyAuthDetails APIs requiring an API key in header/query
OAuth 2.0 OpenApiOAuth2AuthDetails APIs using OAuth 2.0 client credentials
Managed Identity OpenApiManagedAuthDetails Azure services with Entra ID integration

OpenAPI Spec Example

JSON
{
  "openapi": "3.0.0",
  "info": {
    "title": "Weather API",
    "version": "1.0.0"
  },
  "servers": [{ "url": "https://api.weather.example.com" }],
  "paths": {
    "/current": {
      "get": {
        "operationId": "getCurrentWeather",
        "summary": "Get current weather for a city",
        "parameters": [{
          "name": "city",
          "in": "query",
          "required": true,
          "schema": { "type": "string" }
        }],
        "responses": {
          "200": {
            "description": "Weather data"
          }
        }
      }
    }
  }
}
Schema Quality Matters

The quality of your OpenAPI spec directly impacts the agent's ability to use the API correctly. Include detailed description fields for every endpoint, parameter, and schema object. Use meaningful operationId values.

5. Bing Grounding Tool

The Bing Grounding tool gives your agent real-time access to web search results, ensuring responses are grounded in current information. This is essential for questions about recent events, up-to-date pricing, live statistics, and anything that changes frequently.

How Bing Grounding Works

  1. Query formulation — The agent converts the user's question into an optimized search query
  2. Web search — Bing retrieves the most relevant web results
  3. Grounded response — The agent synthesizes an answer from the search results, citing sources
  4. Citation links — Responses include inline citations so users can verify the information

Enabling Bing Grounding

Python
from azure.ai.projects.models import BingGroundingTool

# You need a Bing Grounding connection in your AI Foundry project
bing_connection = project_client.connections.get(
    connection_name="bing-grounding",
)

bing_tool = BingGroundingTool(
    connection_id=bing_connection.id,
)

agent = project_client.agents.create_agent(
    model="GPT-5.1",
    name="research-assistant",
    instructions="""You are a research assistant with access to real-time web search.
Always ground your answers in current search results and cite your sources.
If you're unsure about something, search for it before answering.""",
    tools=bing_tool.definitions,
)
Grounding vs. RAG

Use Bing Grounding for real-time web information (news, current events, public data). Use File Search for your own proprietary documents and internal knowledge. For best results, combine both tools on the same agent.

Tool Comparison

Choosing the right tool for each task is critical. Here's a comprehensive comparison of all five agent tools:

Tool Best For Data Source Latency
Code Interpreter Data analysis, math, charts Uploaded files Medium (5-30s)
File Search Document Q&A, knowledge retrieval Vector store Fast (1-3s)
Azure Functions Custom business logic, DB queries Your own APIs Variable
OpenAPI Third-party API integration External services Variable
Bing Grounding Real-time web search, current events Internet Fast (2-5s)

Need computation?
→ Code Interpreter
Have internal docs?
→ File Search
Custom logic needed?

External API?
→ OpenAPI Tools

🧪 Mini Hack

Put everything together! Build a multi-tool agent that combines Code Interpreter, File Search, and a custom Azure Function tool to handle a complex real-world scenario.

Challenge Requirements

Build an agent that uses Code Interpreter to analyze a CSV dataset, File Search to query documentation, and a custom Azure Function tool — all within a single agent.

Challenge Scenario

You're building an intelligent sales assistant that can:

  • 📊 Analyze a CSV of quarterly sales data and generate visualizations (Code Interpreter)
  • 📄 Answer questions about company policies from an employee handbook (File Search)
  • ⚡ Look up real-time inventory from your warehouse system (Azure Function tool)

Step-by-Step Implementation

  1. Set Up the Vector Store

    Upload your employee handbook PDF and product catalog to a vector store for File Search.

  2. Deploy the Azure Function

    Create a simple HTTP-triggered Function that queries your inventory database and returns stock levels.

  3. Build the Multi-Tool Agent

    Create an agent with all three tools enabled and test it in the playground.

  4. Test End-to-End

    Send a message like: "Analyze Q1 sales trends from the CSV, check if our return policy covers electronics, and look up current stock for product SKU-1234."

Starter Code

Python
from azure.ai.projects import AIProjectClient
from azure.ai.projects.models import (
    CodeInterpreterTool,
    FileSearchTool,
    FunctionTool,
)
from azure.identity import DefaultAzureCredential

project_client = AIProjectClient(
    credential=DefaultAzureCredential(),
    endpoint="https://your-project.services.ai.azure.com",
)

# ── Tool 1: Code Interpreter ──
code_interpreter = CodeInterpreterTool()

# ── Tool 2: File Search ──
vector_store = project_client.agents.create_vector_store_and_poll(
    name="company-docs",
    file_ids=[handbook_file.id, catalog_file.id],
)
file_search = FileSearchTool(vector_store_ids=[vector_store.id])

# ── Tool 3: Custom Function (Inventory Lookup) ──
inventory_tool = FunctionTool(
    definitions=[{
        "name": "check_inventory",
        "description": "Check real-time inventory for a product SKU.",
        "parameters": {
            "type": "object",
            "properties": {
                "sku": {
                    "type": "string",
                    "description": "Product SKU identifier",
                }
            },
            "required": ["sku"],
        },
    }]
)

# ── Create the Multi-Tool Agent ──
all_tools = (
    code_interpreter.definitions
    + file_search.definitions
    + inventory_tool.definitions
)

agent = project_client.agents.create_agent(
    model="GPT-5.1",
    name="sales-assistant",
    instructions="""You are an intelligent sales assistant. You can:
1. Analyze CSV data and create charts (Code Interpreter)
2. Answer questions from company docs (File Search)
3. Check real-time inventory levels (check_inventory function)

Always use the appropriate tool for each part of the user's request.""",
    tools=all_tools,
    tool_resources=file_search.resources,
)

print(f"✅ Multi-tool agent created: {agent.id}")

Best Practices

General Guidelines

  • Keep instructions explicit — Tell the agent exactly when and how to use each tool in its system instructions
  • Minimize tool count — Only attach tools the agent actually needs; too many tools can confuse the LLM
  • Test in the Playground first — Validate tool behavior interactively before writing production code
  • Handle errors gracefully — Always return clear error messages from function tools so the agent can retry or inform the user

Performance Tips

  • Code Interpreter — Keep datasets under 50 MB for optimal performance; pre-clean data when possible
  • File Search — Use focused vector stores per domain rather than one massive store; tune chunk sizes for your content type
  • Azure Functions — Use premium plans for low cold-start latency; return concise JSON responses
  • OpenAPI — Minimize the spec to only the endpoints the agent needs; use descriptive operation summaries
  • Bing Grounding — Instruct the agent to search only when the question requires current information
Security Reminder

Never expose sensitive credentials in function tool definitions. Use Azure Key Vault for secrets, Managed Identity for Azure service-to-service auth, and connection objects in your AI Foundry project for API keys.
