What Are AI Agents?

Autonomous systems that reason, plan, use tools, and take actions to accomplish goals.

  • Goal-oriented — work toward an objective, not just answer questions
  • Autonomous — decide next actions without step-by-step user guidance
  • Tool-equipped — search, compute, call APIs, execute code
  • Multi-step — plan, execute, evaluate, iterate
  • Stateful — maintain memory across interactions

The Evolution to Agents

💬

Single Prompts

One prompt → one response. No memory, no tools.

🔄

Chat Apps

Multi-turn conversation. Session memory. Still reactive.

🔗

RAG Apps

Retrieve external knowledge. Grounded responses.

🤖

AI Agents

Autonomous, multi-step, tool-using, goal-driven. We are here.

Agents vs. Chatbots

💬 Chatbot

  • Reactive — waits for prompts
  • Single-turn logic
  • No tools — text only
  • Session memory only
  • User-guided

🤖 AI Agent

  • Proactive — plans & executes
  • Multi-step reasoning
  • Tool-equipped (APIs, code, search)
  • Persistent memory
  • Self-directed

Pillar 1: Instructions

The soul of your agent — defines who it is and how it behaves.

## Role You are a Financial Analysis Agent for Contoso. ## Goals - Analyze financial data uploaded by users - Generate quarterly reports with key metrics ## Constraints - Never fabricate financial numbers - Always include confidence levels ## Tools - Use Code Interpreter for calculations - Use File Search for company policies

Agent Types

📝 Prompt Agent

  • No code — portal config only
  • Built-in tools (File Search, Code Interpreter, Bing)
  • Fully managed runtime
  • Deploy in minutes
  • Ideal for prototyping & simple tasks

🐳 Hosted Agent

  • Custom code — Python + Docker
  • Any tools, APIs, databases
  • Container-based deployment
  • Any framework (SK, LangChain)
  • Ideal for complex workflows

Choosing the Right Type

✅ Start with Prompt Agents → graduate to Hosted Agents

Agent Security Principles

  • Least privilege — only enable tools the agent needs
  • Content filtering — keep Azure Content Safety active
  • Managed Identity — never embed API keys in instructions
  • Input validation — guard against prompt injection
  • Output guardrails — define what the agent must NEVER do
  • Monitoring — enable tracing for all production agents
  • Human-in-the-loop — require approval for high-stakes actions
🧪

Mini Hack: Your First Prompt Agent

  1. Open ai.azure.com → your project → Agents
  2. Click + New agent
  3. Name: Travel Planner
  4. Select GPT-4o model
  5. Write instructions (persona + goals + constraints)
  6. Enable Code Interpreter + Bing Grounding
  7. Test: "Plan a 5-day Japan trip for $5,000"

Key Takeaways

  • Agents are autonomous, goal-driven — not just better chatbots
  • Four pillars: Instructions + Model + Tools + Memory
  • Azure AI Agent Service provides a fully managed platform
  • Prompt agents = no-code, fast; Hosted agents = custom code, powerful
  • Start simple, validate, then scale up
  • Security first — least privilege, filtering, monitoring
