What Are AI Agents?

AI agents represent a paradigm shift. Unlike traditional AI that responds to a single prompt and stops, AI agents are autonomous systems that reason, plan, use tools, maintain state, and take actions to accomplish goals across multiple steps.

Think of an AI agent as a knowledgeable digital colleague. When asked to "analyze Q3 sales data and prepare recommendations," a colleague doesn't just answer — they plan the approach, access tools (spreadsheets, databases), remember context, iterate on results, and deliver actionable output.

AI agents follow this same pattern. Powered by LLMs as their "reasoning engine," agents break down complex tasks, decide which tools to use, execute multi-step workflows, and adapt based on intermediate results — all without step-by-step user instructions.

ℹ️ Key Insight

The fundamental difference is agency — the ability to make decisions and take actions autonomously. A chatbot waits for your next prompt; an agent decides what to do next on its own based on its goal, instructions, and the tools available to it.

The Evolution: From Prompts to Agents

💬
Single Prompts
User sends one prompt, model returns one response. No memory, no tools. Example: "Translate this sentence."
🔄
Chat Applications
Multi-turn conversations with memory of the session. Still reactive — waits for user input. Example: Customer support chat.
🔗
RAG Applications
Retrieves external knowledge to augment responses. Grounded in data. You built these in previous modules!
🤖
AI Agents
Autonomous, goal-driven, multi-step reasoning. Uses tools, maintains state, adapts plans. This is where we are now.

Agents vs. Chatbots: Understanding the Difference

One of the most common misconceptions is that AI agents and chatbots are the same thing. They're not — and understanding the difference is crucial for choosing the right architecture for your AI solutions.

💬 Traditional Chatbot

  • Reactive — responds only when prompted
  • Single-turn logic — each response is independent
  • No tool access — can only generate text
  • No planning — doesn't break down complex tasks
  • Session memory — forgets after conversation ends
  • Human-guided — user must direct each step

🤖 AI Agent

  • Proactive — plans and executes autonomously
  • Multi-step reasoning — chains of thought and actions
  • Tool-equipped — calls APIs, searches, computes
  • Goal-oriented — decomposes complex tasks into subtasks
  • Persistent memory — retains context across sessions
  • Self-directed — decides next actions independently

A Concrete Example

Consider: "Book a meeting room for our standup next Tuesday at 10 AM, send invites, and prepare an agenda from last week's action items."

Aspect Chatbot Response Agent Response
Room booking "I can't book rooms. Try Outlook." Checks room availability → Books Room B
Calendar invites "Here's a template email you can send." Calls Calendar API → Sends invites to team
Agenda "Here's a generic standup template." Retrieves last week's notes → Extracts action items → Creates agenda
Steps taken 1 (text response) 6+ (plan, search, book, compose, send, confirm)
💡 When to Use Each

Use a chatbot when your scenario is primarily Q&A, doesn't require external actions, and the user is available to guide the conversation. Use an agent when you need autonomous task completion, multi-step workflows, tool integration, or complex reasoning across multiple data sources.

Azure AI Agent Service Overview

The Azure AI Agent Service is a fully managed platform within Microsoft Foundry for building, deploying, and scaling AI agents without managing infrastructure. It provides the runtime, tool integration, memory management, and observability to go from prototype to production.

ℹ️ Naming Clarification

You may encounter references to "Azure AI Foundry Agent Service," "AI Agent Service," or just "Agent Service" — these all refer to the same platform. The service is a core component of the broader Microsoft Foundry platform, which provides the unified AI development experience including models, agents, evaluation, and deployment.

Core Capabilities

🧠
Model Flexibility
Use any model from the Azure AI model catalog — GPT-4o, GPT-4.1, Phi-4, Llama, Mistral, and more. Swap models without changing agent code.
🔧
Built-in Tools
File Search, Code Interpreter, Bing Grounding, Azure Functions, Azure Logic Apps, API connections, and OpenAPI tools — all out of the box.
💾
Managed State
Threads and conversation history managed automatically. No need to build your own state store or manage token windows manually.
📊
Observability
Built-in tracing with OpenTelemetry, Application Insights integration, and the Foundry portal's trace explorer for debugging agent behavior.
🔒
Enterprise Security
Azure RBAC, managed identity, VNET integration, content filtering, and data residency controls for compliance-sensitive deployments.
⚡
Scalable Runtime
Auto-scaling agent execution with managed compute. Handle spikes in agent workloads without provisioning infrastructure.

How Agent Service Fits in the Foundry Ecosystem

The Azure AI Agent Service is not a standalone product — it's deeply integrated with the Microsoft Foundry ecosystem you've been learning about throughout this course:

  • Model Catalog — provides the LLM "brain" for your agents
  • AI Search — powers the File Search and knowledge retrieval tools
  • Prompt Flow — enables advanced orchestration and evaluation
  • Content Safety — applies content filtering to agent inputs/outputs
  • Azure Monitor — captures telemetry and traces for production monitoring
  • Evaluation SDK — measures agent quality with specialized metrics

1. Instructions (System Prompt)

Instructions are the soul of your agent — they define who it is, what it should do, how it should behave, and what constraints apply.

Agent Instructions Example 
## Role
You are a Financial Analysis Agent for Contoso Corp.

## Goals
- Analyze financial data uploaded by users
- Generate quarterly reports with key metrics
- Provide actionable recommendations based on trends

## Constraints
- Never fabricate financial numbers — always use provided data
- If data is incomplete, clearly state what's missing
- Format all currency values in USD with 2 decimal places
- Always include a confidence level with recommendations

## Tools Available
- Use Code Interpreter for calculations and charts
- Use File Search to reference company policies
- Use Bing Grounding for market comparisons
💡 Instruction Design Tip

Structure your instructions with clear sections: Role (who is the agent), Goals (what should it accomplish), Constraints (what it must NOT do), and Tool guidance (when to use which tools). This structure helps the model follow instructions reliably. You'll practice advanced prompt engineering for agents in later modules.

2. Model (The Reasoning Engine)

The model is the brain of your agent — it processes instructions, reasons about tasks, decides which tools to use, and generates responses. You can choose from any model in the Azure AI catalog:

Model Best For Key Strengths
GPT-4o General-purpose agents Strong reasoning, multimodal, fast
GPT-4.1 Complex agentic workflows Best instruction following, long context
GPT-4o-mini Cost-sensitive agents Fast, affordable, good for simpler tasks
Phi-4 Lightweight agents Small, efficient, on-device capable
Llama 4 Open-source requirements Flexible licensing, strong performance

3. Tools (The Agent's Hands)

Tools give your agent the ability to interact with the real world. Without tools, an agent is just a chatbot with a fancy system prompt.

📂
File Search
Searches through uploaded files and knowledge bases using Azure AI Search. Supports PDFs, DOCX, TXT, and more.
💻
Code Interpreter
Executes Python code in a sandboxed environment. Performs calculations, generates charts, processes data files.
🌐
Bing Grounding
Searches the web for real-time information. Grounds agent responses in current, factual data from the internet.
⚙️
Azure Functions
Calls custom serverless functions. Extend your agent with any custom business logic, API integrations, or computations.
🔗
OpenAPI Tools
Connect to any REST API via OpenAPI specification. Your agent can interact with CRMs, ERPs, or any third-party service.
🔄
Azure Logic Apps
Trigger automated workflows. Send emails, update databases, integrate with 400+ connectors through Logic Apps.

4. Memory (Threads & State)

Memory is managed through threads — conversation sessions storing the complete interaction history:

  • Message history — all user and agent messages in order
  • File attachments — files uploaded during the conversation
  • Tool execution results — outputs from Code Interpreter, File Search, etc.
  • Run metadata — details about each agent execution step

The Agent Service handles token management automatically — truncating older messages when the context window fills up while retaining the most relevant history.

Python — Creating a Thread and Running an Agent 
from azure.ai.projects import AIProjectClient
from azure.identity import DefaultAzureCredential

# Connect to your Foundry project
client = AIProjectClient(
    credential=DefaultAzureCredential(),
    endpoint="https://<your-project>.services.ai.azure.com"
)

# Create a new thread (conversation session)
thread = client.agents.create_thread()

# Add a user message to the thread
client.agents.create_message(
    thread_id=thread.id,
    role="user",
    content="Analyze the Q3 revenue trends from the uploaded data."
)

# Run the agent on the thread
run = client.agents.create_and_process_run(
    thread_id=thread.id,
    agent_id="<your-agent-id>"
)

# Retrieve the agent's response
messages = client.agents.list_messages(thread_id=thread.id)
for msg in messages:
    if msg.role == "assistant":
        print(msg.content)

Agent Types: Prompt Agents vs. Hosted Agents

Azure AI Foundry offers two primary agent types for different complexity levels and deployment scenarios.

📝 Prompt Agent

  • No code required — configure entirely in the portal
  • Powered by a model + instructions + built-in tools
  • Ideal for getting started quickly
  • Managed by the platform — no infrastructure
  • Use built-in tools: File Search, Code Interpreter, Bing
  • Great for Q&A, data analysis, document processing

🐳 Hosted Agent

  • Custom code — bring your own Python/container
  • Full control over agent logic and orchestration
  • Supports any framework (Semantic Kernel, LangChain, etc.)
  • Containerized deployment with managed hosting
  • Integrate any external service, database, or API
  • Ideal for complex, multi-agent workflows

Prompt Agents Deep Dive

Prompt agents are the fastest path to your first AI agent. No code required — define the agent entirely through the portal or API. The platform handles execution, tool orchestration, and state management.

What Makes a Prompt Agent

A prompt agent is defined by a configuration that includes:

  • Name & Description — identifies the agent in your project
  • Model deployment — which LLM to use (e.g., GPT-4o)
  • Instructions — the system prompt that defines behavior
  • Tools — which built-in tools the agent can access
  • Knowledge sources — files or data the agent can search
  • Parameters — temperature, top-p, and response format settings
Python SDK — Creating a Prompt Agent 
from azure.ai.projects import AIProjectClient
from azure.ai.projects.models import (
    FileSearchTool,
    CodeInterpreterTool,
    BingGroundingTool
)
from azure.identity import DefaultAzureCredential

client = AIProjectClient(
    credential=DefaultAzureCredential(),
    endpoint="https://<your-project>.services.ai.azure.com"
)

# Define the tools this agent can use
tools = [
    FileSearchTool(),
    CodeInterpreterTool(),
    BingGroundingTool(connection_name="bing-connection")
]

# Create the prompt agent
agent = client.agents.create_agent(
    model="gpt-4o",
    name="Financial Analyst",
    instructions="""You are a financial analyst agent for Contoso Corp.
    Analyze uploaded financial documents, compute key metrics,
    and provide data-driven recommendations.
    Always show your calculations using Code Interpreter.
    Cross-reference company policies using File Search.""",
    tools=tools,
    temperature=0.3,
    top_p=0.9
)

print(f"Agent created: {agent.id}")

When to Choose Prompt Agents

  • You want to prototype quickly without writing code
  • Your use case primarily uses built-in tools (File Search, Code Interpreter, Bing)
  • You need no custom business logic beyond what the model provides
  • You're building a single-agent solution (not multi-agent orchestration)
  • You want the simplest possible deployment — fully managed by the platform

Hosted Agents Deep Dive

Hosted agents give you full control over your agent's logic while benefiting from Foundry's managed infrastructure. Write custom Python code, package it as a container, and deploy to the Agent Service.

Architecture of a Hosted Agent

A hosted agent consists of:

  • Custom code — your Python application with agent logic
  • agent.yaml — configuration file defining the agent's metadata and entry point
  • Container image — Docker image pushed to Azure Container Registry (ACR)
  • Managed hosting — the Agent Service runs your container with auto-scaling
agent.yaml — Hosted Agent Configuration 
name: order-processing-agent
description: Processes customer orders and manages inventory
model: gpt-4o
instructions: |
  You are an order processing agent. Validate orders,
  check inventory, compute totals, and update records.
tools:
  - type: function
    function:
      name: check_inventory
      description: Check product availability
      parameters:
        type: object
        properties:
          product_id:
            type: string
            description: The product SKU
entry_point: app.main:agent_handler

When to Choose Hosted Agents

  • You need custom business logic that goes beyond built-in tools
  • You're building multi-agent systems with coordination between agents
  • You need to integrate with proprietary databases, internal APIs, or legacy systems
  • You want to use a specific framework like Semantic Kernel, LangChain, or AutoGen
  • You require fine-grained control over the agent's reasoning loop

Choosing the Right Agent Type

Use this decision framework to guide your choice:

Criteria Prompt Agent ✅ Hosted Agent ✅
Development speed Minutes to deploy Hours to days
Code required None Python + Docker
Customization Instructions + built-in tools Unlimited — any code
Tool integration Built-in tools only Any API, service, or SDK
Multi-agent Single agent Multi-agent orchestration
Framework support Agent Service native Semantic Kernel, LangChain, etc.
Ideal for Prototyping, simple tasks Production, complex workflows
Infrastructure Fully managed Container-based (managed hosting)
✅ Recommended Approach

Start with prompt agents to validate your use case quickly, then graduate to hosted agents when you need custom logic, external integrations, or multi-agent patterns. Many production systems start as prompt agents and evolve into hosted agents as requirements grow.

🧪 Mini Hack: Create Your First Prompt Agent

Time to get hands-on! In this Mini Hack, you'll create your first prompt agent using the Azure AI Foundry portal. No code required — just the portal and your creativity.

ℹ️ Prerequisites

You need an Azure AI Foundry project with a deployed GPT-4o model (or any chat model from the catalog). If you completed Modules 3–5, you already have this set up. If not, refer to Module 4 for model deployment instructions.

Step-by-Step Instructions

  1. Navigate to the Agent Builder Go to ai.azure.com, open your project, and click "Agents" in the left navigation menu. Click "+ New agent" to start.
  2. Configure the Agent Basics Set the agent name to Travel Planner. Select your deployed GPT-4o model from the model dropdown. Leave the default parameters for now.
  3. Write the Instructions In the Instructions field, enter:

    "You are a helpful travel planning agent. Help users plan trips by suggesting destinations, creating itineraries, estimating budgets, and providing travel tips. Always ask clarifying questions about travel dates, budget range, and preferences before making recommendations. Format itineraries with day-by-day breakdowns."
  4. Enable Tools Toggle on Code Interpreter (for budget calculations) and Bing Grounding (for current travel information). Leave File Search off for now.
  5. Test in the Playground Click "Try in Playground" and send a message: "I want to plan a 5-day trip to Japan for two people in November with a $5,000 budget." Watch how the agent asks follow-up questions, uses Bing for current info, and provides structured recommendations.
  6. Iterate on Instructions Try adding a constraint like "Always include at least one off-the-beaten-path recommendation" and test again. Observe how the agent's behavior changes with updated instructions.
✅ What You Just Built

Congratulations! You've created your first AI agent. Unlike the chatbots you've built before, this agent can use tools autonomously — it decides when to search the web and when to run calculations without you telling it to. That's the power of agents.

Security Considerations for Agents

Because agents take autonomous actions, security is critical. Key principles:

⚠️ Agent Security is Non-Negotiable

An agent with unrestricted tool access and poor instructions can cause real harm — deleting data, exposing sensitive information, or executing unintended operations. Always apply the principle of least privilege: give agents only the tools and permissions they need for their specific task.

  • Least Privilege — Only enable the tools your agent actually needs. Don't enable Code Interpreter if the agent only does Q&A.
  • Content Filtering — Always keep Azure Content Safety filters active on agent inputs and outputs.
  • Managed Identity — Use Azure Managed Identity for service-to-service authentication. Never embed API keys in instructions.
  • Input Validation — Guard against prompt injection attempts by including defensive instructions.
  • Output Guardrails — Define what the agent should never do in its instructions (e.g., "Never share personal data" or "Never execute destructive operations").
  • Monitoring — Enable tracing and logging to audit all agent actions in production.
  • Human-in-the-Loop — For high-stakes decisions, require human approval before the agent takes action.

🎯 Key Takeaways

  • AI agents are autonomous, goal-driven systems that use LLMs to reason, plan, and take actions — fundamentally different from reactive chatbots.
  • The four pillars of agent architecture are Instructions (persona/goals), Model (reasoning engine), Tools (actions), and Memory (threads/state).
  • Azure AI Agent Service in Microsoft Foundry provides a fully managed platform for building, deploying, and scaling agents.
  • Prompt agents are configured without code (portal/API) and use built-in tools — ideal for rapid prototyping and simpler use cases.
  • Hosted agents use custom code in containers — required for complex business logic, custom integrations, and multi-agent systems.
  • Start with prompt agents, validate your use case, then graduate to hosted agents as complexity grows.
  • Security is paramount — always apply least privilege, content filtering, and monitoring for autonomous agents.
