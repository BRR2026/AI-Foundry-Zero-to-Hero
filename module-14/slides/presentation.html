Beyond Basic RAG

Basic RAG: retrieve chunks → stuff into prompt → generate.
What goes wrong? Context loss, hallucinations, poor relevance, missed information.
  • Naive chunking splits content mid-sentence or mid-thought
  • Single retrieval pass misses nuanced queries
  • No re-scoring means irrelevant results pollute the context
  • Single index can't handle heterogeneous data
Advanced RAG = smarter chunking + better retrieval + reranking + orchestration

Chunking Strategies

Strategy How It Works Best For
Fixed-Size Split at every N tokens with overlap Uniform content, logs
Semantic Split at topic/sentence boundaries Articles, documentation
Sliding Window Overlapping windows for context Narrative text, transcripts
Recursive Hierarchical: sections → paragraphs → sentences Structured documents
💡 Rule of thumb: Use semantic chunking for docs, recursive for structured content, fixed-size for uniform data.

Embedding Model Selection

Model Dimensions Performance Cost
text-embedding-3-small 1536 Good Lower
text-embedding-3-large 3072 Best Higher
  • Higher dimensions = better semantic capture, but more storage & compute
  • Azure OpenAI supports dimension reduction — use 1024 dims with the large model for a good balance
  • Always use the same model for indexing and querying

Reranking & Relevance Tuning

Initial Retrieval
(top 50) → 🎯 Cross-Encoder
Reranker → Re-scored
(top 5)
  • Bi-encoder (embeddings): fast but approximate similarity
  • Cross-encoder (reranker): slower but much more accurate relevance
  • Azure AI Search Semantic Ranker provides built-in cross-encoder reranking
  • Typical improvement: 15–35% better top-result relevance
Reranking is the highest-ROI optimization you can add to a RAG pipeline.

Hybrid Search & Score Fusion

Keyword Search (BM25)

  • Exact term matching
  • Great for proper nouns, codes
  • Misses semantic similarity

Vector Search (Embeddings)

  • Semantic understanding
  • Handles paraphrases
  • Can miss exact matches
BM25 Results + Vector Results → RRF Fusion → Unified Ranking

Multi-Index RAG

📚 Product Docs Index 🎫 Support Tickets Index 📋 FAQ Index

↓
🔀 Query Router → 📊 Result Merger → 🤖 LLM
  • Different indexes for different data types and schemas
  • Query routing: classify intent → select relevant index(es)
  • Score normalization: essential when merging across indexes
  • In Azure AI Foundry: connect multiple data sources to your project
9 / 14

Handling Tables & Structured Data

  • Azure Document Intelligence extracts tables, key-value pairs, and layout
  • Convert tables to multiple formats for better retrieval:
    • Markdown table representation
    • Natural language summaries of table contents
    • Row-level chunks with column headers as context
  • Hybrid approach: SQL queries for structured data + vector search for text
💡 Don't just flatten tables to text — preserve structure and add column context to each row chunk.

Agentic RAG Patterns

🤖 AI Agent

🔍 Search Tool 📊 SQL Tool 🌐 Web Tool 📁 File Tool
  • Query decomposition: agent breaks complex questions into sub-queries
  • Iterative retrieval: retrieve → evaluate → retrieve more if needed
  • Tool selection: agent chooses the right retrieval method per sub-query
  • Self-reflection: agent validates retrieved context before answering
Agentic RAG = the agent decides what to retrieve, when, and how — not a fixed pipeline.

Performance Optimization

Latency Reduction

  • Cache frequent queries & embeddings
  • Use smaller embedding dimensions
  • Limit top-k to what the LLM needs
  • Async parallel retrieval

Quality Monitoring

  • Track retrieval precision & recall
  • Monitor reranker score distributions
  • A/B test chunking strategies
  • Use AI Foundry evaluations
Optimize for the right balance of latency, cost, and quality for your use case.

Key Takeaways

  • Chunking is the foundation — semantic > fixed-size for most document types
  • Reranking is highest-ROI — 15-35% improvement with minimal code
  • Hybrid search combines precision of keywords + understanding of vectors
  • Multi-index unlocks cross-source retrieval for complex scenarios
  • Agentic RAG is the future — agents that orchestrate their own retrieval
