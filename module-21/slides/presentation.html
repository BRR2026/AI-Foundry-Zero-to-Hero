What is Prompt Flow?

A development tool and runtime engine for building LLM applications using directed acyclic graphs (DAGs)

📊

Visual Designer

Drag-and-drop canvas to design node-based workflows with typed connections

🧪

Integrated Testing

Interactive & batch testing with variant comparison and execution traces

🚀

One-Click Deploy

Deploy flows as managed online endpoints with auto-scaling and monitoring

💡 Prompt Flow is not Azure ML pipelines. It's purpose-built for LLM orchestration within Azure AI Foundry.

When to Use Prompt Flow

✅ Ideal For

  • →Multi-step LLM pipelines with branching
  • →Rapid prototyping of prompt strategies
  • →Visual debugging of intermediate outputs
  • →Batch evaluations across datasets
  • →Team collaboration on AI logic

⚠️ Consider Alternatives

  • →Simple single-call apps → SDK
  • →Complex agent systems → Semantic Kernel
  • →Data processing → Data pipelines
  • →Model training → Azure ML
  • →Real-time streaming → Custom code

Flow Types

Flow Type Purpose Use Cases
Standard General-purpose LLM app development Q&A, RAG, classification, extraction
Chat Conversation-oriented with chat history Chatbots, multi-turn agents, support
Evaluation Quality assessment of other flows Groundedness, relevance, coherence
💡 Tip: Start with Standard Flow for most use cases. Switch to Chat Flow only when you need built-in conversation history management.

LLM Nodes — Calling Models

Use Jinja2 templates with variable substitution to build prompts

Jinja2 Prompt Template

system: You are a helpful assistant. Answer based on provided context. user: Context: {{context}} Question: {{question}} Provide a clear answer.

Python Nodes — Custom Logic

Run custom code with the @tool decorator for data processing, routing, and validation

from promptflow import tool from typing import List @tool def validate_response(llm_output: str, context: str) -> str: """Validate LLM response against source context.""" if not llm_output or len(llm_output.strip()) < 10: return "low_confidence" # Check context overlap for grounding overlap = len(set(context.lower().split()) & set(llm_output.lower().split())) return "high" if overlap > 20 else "medium"
⚡ Key rules: Always use @tool decorator · Type hints on all params · One return value · Never hardcode secrets

✅ Pro Tip: Create variants of LLM nodes to test different prompts side-by-side. Run batch evaluations to statistically compare performance.

Debugging Common Issues

Issue Symptom Fix
Missing connection Red node outline Configure in project settings
Type mismatch Serialization error Match output → input types
Token limit Timeout / API error Truncate context or raise limit
Import error ModuleNotFoundError Add to requirements.txt
Empty response Blank LLM output Check template variable mapping
⚠️ Most common mistake: Forgetting to start a compute session before testing. Check the status indicator in the top-right corner.

Best Practices

📐

Keep Flows Focused

One task per flow. Split flows with 10+ nodes into sub-flows.

🔄

Use Variants

Test different prompts and models. Use batch runs to compare quantitatively.

🛡️

Validate Everything

Add validation nodes after LLM calls. Fail gracefully with fallbacks.

🛑 Anti-patterns to avoid: Monolith flows (20+ nodes) · Hardcoded secrets · Skipping batch tests · Ignoring latency budgets · No error handling
