Why Prompt Engineering Matters

The Prompt IS the Program

You don't write algorithms — you write instructions. Output quality tracks prompt quality.

Cost & Latency

Precise prompts use fewer tokens, cut retries, and reduce API costs by up to 40%.

Consistency at Scale

Production systems need repeatable results. Structure turns stochastic models into reliable tools.

No Fine-Tuning Needed

~80% of use cases are solvable with the right prompt, no custom training required.

Zero-Shot Prompting

No examples provided — the model infers the task from instructions alone.

Best For

  • Simple classification
  • Straightforward Q&A
  • Summarisation
  • Translation

Example

System:
Classify the sentiment as
positive, negative, or neutral.
Reply with one word only.

User:
"The battery life is incredible,
I can go two days without charging!"

→ Assistant: "positive"

Few-Shot Prompting

Provide 2–5 examples of input → output pairs so the model learns the pattern.

Why It Works

  • Establishes output format by example
  • Reduces ambiguity dramatically
  • Guides tone, length, and structure

Tip: Choose diverse examples that cover edge cases.

Example

User: "Packaging was damaged."
Assistant: {"sentiment": "negative"}

User: "Arrived on time, works great."
Assistant: {"sentiment": "positive"}

User: "It's okay, nothing special."
Assistant: {"sentiment": "neutral"}

User: "Love the new design!"
Assistant: ???

Chain-of-Thought (CoT)

Instruct the model to show its reasoning step by step before giving a final answer.

When to Use

  • Math & logic problems
  • Multi-step reasoning
  • Ambiguous classification
  • Complex decision-making

Without CoT

Q: If a shirt costs $25 and is
   on 20% sale, what's the price?
A: $20  ← correct but no reasoning

With CoT

Q: ... Let's think step by step.
A: 1. Original price: $25
   2. Discount: 25 × 0.20 = $5
   3. Sale price: 25 − 5 = $20
   Answer: $20 ← verifiable!

Prompt Templates

Separate logic from data using placeholders.

SYSTEM PROMPT TEMPLATE:

You are a {{role}} assistant.
Summarise the following article in {{num_points}} bullet points.
Use {{language}} language. Be concise.

---
ARTICLE:
{{article_text}}
---

Respond in this JSON format:
{
  "summary": ["bullet 1", "bullet 2", ...],
  "word_count": <number>
}

In AI Foundry: Save templates in Prompt Flow for versioning, A/B testing, and team collaboration.

Parameters Deep Dive

ParameterRangeLowHigh
temperature0 – 2Deterministic, focusedCreative, varied
top_p0 – 1Narrow token poolBroad token pool
max_tokens1 – limitShort replyLong reply
frequency_penalty-2 – 2Allows repetitionPenalises repetition
presence_penalty-2 – 2Stays on topicEncourages new topics

Rule of thumb: Tune either temperature or top_p — not both at the same time.

Parameter Presets by Use Case

Classification / Extraction

temperature: 0.0
top_p: 1.0
max_tokens: 200

Deterministic. Same input → same output.

Summarisation / Q&A

temperature: 0.3
top_p: 0.95
max_tokens: 500

Slightly creative but mostly factual.

Creative Writing

temperature: 0.9
top_p: 0.95
max_tokens: 2000

High variety. Great for brainstorming.

Code Generation

temperature: 0.2
top_p: 0.95
max_tokens: 4000

Low creativity. Precise, functional output.

Common Anti-Patterns

❌ Vague Instructions

"Do something with this data."

Fix: Be specific about task, format, and constraints.

❌ No Output Format

"Analyze this feedback."

Fix: Specify JSON, markdown, bullet points, etc.

❌ Prompt Stuffing

"You are an expert in everything.
Consider all possibilities..."

Fix: Focus on one task per prompt. Be concise.

❌ Ignoring Edge Cases

No handling for empty input,
wrong language, or off-topic.

Fix: Add explicit guardrails and fallback instructions.

Key Takeaways

  • System prompts are the foundation — invest time crafting them
  • Zero-shot works for simple tasks; few-shot adds consistency
  • Chain-of-thought dramatically improves reasoning accuracy
  • Templates separate logic from data for reusable, testable prompts
  • Temperature 0.0 for deterministic tasks; 0.7–0.9 for creative tasks
  • Always specify output format and handle edge cases explicitly
  • The AI Foundry Playground is your laboratory — experiment freely
