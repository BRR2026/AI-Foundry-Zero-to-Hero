📊 Resource Hierarchy

Azure Subscription └── Resource Group └── AI Foundry Hub ├── Project A │ ├── Deployments │ ├── Connections │ └── Prompt Flows └── Project B ├── Deployments ├── Connections └── Evaluations

🏢 Four Levels

  • Subscription — Billing boundary
  • Resource Group — Lifecycle container
  • Hub — Shared infrastructure
  • Project — Team workspace

💡 Under the hood: Hub = ML Workspace (kind: Hub), Project = ML Workspace (kind: Project)

🏗️ Hub vs Project

🏢 Hub (AI Foundry Account)

Shared infrastructure & governance

  • Storage Account
  • Key Vault
  • Application Insights
  • Container Registry
  • Networking configuration
  • Managed Identity
  • Shared connections

📁 Project

Team workspace for development

  • Model deployments
  • Project connections
  • Prompt flows
  • Evaluations
  • Data assets & indexes
  • Fine-tuning jobs
  • Playground access

🏠 Analogy: Hub = Office building (shared lobby, security, utilities) · Project = Individual office suite (team's workspace)

🌐 Networking Architecture

🌍 Public Access

Dev/Test

  • Internet accessible
  • No VNet needed
  • Easiest setup
  • Use IP restrictions

🔒 Private Endpoints

Staging

  • Traffic on backbone
  • VNet required
  • DNS configuration
  • Hybrid access

🛡️ Managed VNet

Production

  • Fully private
  • No public exposure
  • Maximum security
  • Complex setup

⚠️ Each underlying resource (Storage, Key Vault, OpenAI) needs its own Private Endpoint when using private networking.

🔑 Authentication & Identity

Authentication Methods

  • Managed Identity ✅ Preferred
  • Service Principal — Automation/CI/CD
  • User Identity — Interactive/Portal
  • API Keys — Legacy, avoid

💡 Rule of thumb: Managed Identity > Service Principal > API Keys

☁️ Azure OpenAI Deployment Types

Global Standard

Lowest Cost

  • Best-effort routing
  • Data may leave region
  • Highest availability
  • Pay-per-token

→ Prototyping, dev/test

Standard

Balanced

  • Regional deployment
  • Data stays in region
  • Moderate rate limits
  • Pay-per-token

→ Production w/ compliance

Provisioned (PTU)

Guaranteed

  • Reserved capacity
  • Consistent latency
  • Highest throughput
  • Monthly commitment

→ High-volume production

⚠️ Data residency: Global Standard may process data in any Azure region. Use Standard or Provisioned for compliance requirements.

Hub-Level Connections

Shared across all projects. Use for central services (OpenAI, AI Search).

Isolated to one project. Use for team-specific data (CosmosDB, custom APIs).

💡 Credentials are stored securely in Key Vault

⚙️ Compute Resources & SDK

What Runs Where

  • Inference → Managed Online Endpoints, Serverless API, Azure OpenAI
  • Fine-tuning → Compute Clusters (GPU VMs)
  • Evaluation → Serverless or Managed Compute
  • Development → Compute Instances (Notebooks)
  • Prompt Flow → Managed Endpoints or Serverless

⚠️ Set auto-shutdown on Compute Instances to avoid 24/7 charges!

💰 Cost Structure

✅ Free

  • AI Foundry Hub resource
  • AI Foundry Project resource
  • AI Foundry Portal (web UI)
  • App Insights (first 5 GB/mo)

📊 Prototype (~$30/mo)

  • Storage: ~$2
  • Key Vault: ~$1
  • GPT-5.1 (1M tokens): ~$8
  • Compute (40 hrs): ~$15
  • Container Registry: ~$5

❌ Paid

  • Storage Account (per GB)
  • Key Vault (per operation)
  • Azure OpenAI (per token/PTU)
  • Compute Instances (per hour)
  • Managed Endpoints (per hour)
  • Container Registry (per tier)
  • Private Endpoints (~$7/mo each)

🏭 Production (~$5,700/mo)

  • PTU (50 units): ~$3,300
  • Managed Endpoints: ~$2,200
  • App Insights (50 GB): ~$115
  • Infrastructure: ~$100

📋 Key Takeaways

1️⃣ Hierarchy: Sub → RG → Hub → Project

2️⃣ Hub = Shared infra & governance

3️⃣ Project = Team workspace

4️⃣ Networking: Public → PE → Managed VNet

5️⃣ Identity: MI > SP > API Keys

6️⃣ Deployments: Global → Standard → PTU

7️⃣ Connections bridge projects to data

8️⃣ Cost: Hub/Project free; pay for compute + tokens
