Deployment Types

Three strategies. One platform. Choose wisely.

Three Deployment Types

🖥️ Managed Compute

Dedicated VMs. Full control. Custom & fine-tuned models. Pay per hour.

☁️ Serverless API

Zero infra. Pay-per-token. Instant provisioning. Best for experimentation.

⚡ Provisioned Throughput

Reserved PTUs. Guaranteed latency. Monthly commitment. Best at scale.

💡 How to Choose

  • Experimenting → Serverless
  • Custom model → Managed
  • Production at scale → Provisioned

Deploy GPT-5.1 in AI Foundry

  1. Open ai.azure.com → select your project
  2. Navigate to Model catalog → search GPT-5.1
  3. Click Deploy → choose Serverless API
  4. Name it gpt-5.1-multimodal
  5. Configure content filters & API version
  6. Click Deploy → copy endpoint + key
  7. Test in the Playground

⏱️ Total time: under 60 seconds for serverless

Multimodal with GPT-5.1

One model. Text + Vision + Audio. One API call.

The Multimodal API Pattern

All modalities use the same Chat Completions endpoint:

{
  "messages": [{
    "role": "user",
    "content": [
      { "type": "text", "text": "Describe this image" },
      {
        "type": "image_url",
        "image_url": {
          "url": "https://example.com/photo.jpg",
          "detail": "high"
        }
      }
    ]
  }]
}

💡 detail: "high" for complex images, "low" to save tokens

Working with Images

🔗 URL Reference

  • Pass a public URL
  • Small request payload
  • Requires accessible URL
"url": "https://...jpg"

📦 Base64 Encoding

  • Embed image in request
  • Works with private files
  • ~33% larger payload
"url": "data:image/jpeg;
  base64,/9j/4AAQ..."

✅ Both methods support: JPEG, PNG, GIF, WebP

Model Benchmarking & A/B Testing

  1. Build eval dataset — 50–200 representative queries
  2. Deploy candidates — GPT-5.1, GPT-5-mini, Llama 3, etc.
  3. Run batch evaluation — AI Foundry evaluation framework
  4. Compare metrics — Groundedness, relevance, coherence, fluency
  5. Measure cost & latency — Tokens/request, P50/P95/P99

⚠️ Always benchmark with YOUR data. Public leaderboards don't reflect your workload.

Key Takeaways

  • 3 deployment types — Managed, Serverless, Provisioned — each for different needs
  • GPT-5.1 is natively multimodal: text + vision + audio in one call
  • Images via URL or base64; use detail to balance quality vs cost
  • Audio pipelines: Whisper → GPT-5.1 → TTS
  • Benchmark with your data — not public leaderboards
  • Cost varies dramatically — model your traffic
