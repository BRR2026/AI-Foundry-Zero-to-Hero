💻 CODE

Dashboard-as-Code Configuration

Programmatically create and version-control your RAI dashboard setup:

# Configure RAI Dashboard in Azure AI Foundry from azure.ai.evaluation import RAIDashboardConfig, EvaluatorConfig dashboard_config = RAIDashboardConfig( display_name="Enterprise RAI Dashboard", evaluators=[ EvaluatorConfig(name="content_safety", severity_threshold=2, categories=["hate", "sexual", "violence", "self_harm"], custom_blocklist_ids=["financial-compliance"]), EvaluatorConfig(name="fairness", protected_attributes=["gender", "age_group"], disparity_threshold=0.1), ], schedule="0 */4 * * *", # Every 4 hours alert_recipients=["rai-team@contoso.com"] ) dashboard = project_client.rai_dashboards.create(config=dashboard_config)

Bias & Fairness Monitoring

MetricWhat It MeasuresThreshold
Demographic ParityEqual positive outcome probabilityRatio ≥ 0.8
Equalized OddsEqual TPR & FPR across groupsDiff ≤ 0.1
Quality DisparityOutput quality gap by demographicsGap ≤ 0.15

⚠️ Key principle: Never collect demographic data from end users. Use controlled evaluation datasets with known attributes. Monitor production via aggregate statistical tests.

🔔 ALERTING

Azure Monitor Integration for RAI

Alert Pipeline

  • 1️⃣AI Foundry evaluations emit OpenTelemetry metrics
  • 2️⃣Metrics flow to Azure Monitor / App Insights
  • 3️⃣KQL alert rules detect threshold breaches
  • 4️⃣Action Groups notify teams via email, SMS, webhook
  • 5️⃣Incident response playbook activates
// KQL: Fairness threshold breach customMetrics | where name == "ai.foundry.fairness .quality_disparity" | where timestamp > ago(1h) | summarize avg_disp = avg(value), breaches = countif(value > 0.15) by bin(timestamp, 5m) | where breaches > 0

📋 PLAYBOOK

Incident Response Playbook

Phase 1 — 0–15 min

🔍 Detect & Triage

Validate alert, assign severity, designate incident commander

Phase 2 — 15–60 min

🛑 Contain

Emergency blocklist, reduce temperature, notify stakeholders

Phase 3 — 1–24 hrs

🔬 Investigate

Root cause analysis via tracing, reproduce in sandbox

Phase 4 — 1–7 days

🔧 Remediate

Permanent fix, add to red-team regression dataset

Phase 5 — Post-Incident

📋 Learn

Blameless retro, update playbooks, share learnings

Critical: Test Your Response

  • 📅Quarterly tabletop exercises
  • 🎭Simulate jailbreak scenarios
  • ⏱️Measure response time vs SLAs
  • 📝Document & update playbooks
  • 🔄Iterate on containment strategies
