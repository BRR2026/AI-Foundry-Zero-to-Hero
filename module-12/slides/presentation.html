Core Concept

What is Azure AI Search?

A cloud search service with built-in AI capabilities for full-text, semantic, and vector search.

📝

Full-Text Search

BM25 scoring, language analyzers, fuzzy matching, autocomplete

🧠

Semantic Ranking

Transformer-based re-ranking with captions and answers

🔢

Vector Search

HNSW nearest-neighbor search on embeddings

Fundamentals

Understanding Embeddings

  • Text → high-dimensional vector (e.g., 1536 dims)
  • Semantically similar text = nearby vectors
  • Generated by models like text-embedding-3-large
  • Similarity measured with cosine similarity
"Azure AI Search rocks" ↓ Embedding Model [0.023, -0.041, 0.087, ..., 0.012] ↓ 1536 dimensions Store in vector field ↓ Compare with query vector ↓ 🎯 Nearest Neighbors
Hands-On

Creating Vector Indexes

# Define a vector field in your index schema SearchField( name="content_vector", type=SearchFieldDataType.Collection( SearchFieldDataType.Single ), searchable=True, vector_search_dimensions=1536, vector_search_profile_name="my-hnsw-profile", ) # Configure HNSW algorithm HnswAlgorithmConfiguration( name="hnsw-config", parameters=HnswParameters(m=4, ef_construction=400) )

Index Schema Design

Field TypePurposeExample

✅ Do

Separate text and vector fields · Use language analyzers · Set filterable on category fields

❌ Don't

Make all fields searchable · Skip semantic config · Use oversized dimensions

Hybrid Search in Action

Feature

Integrated Vectorization

Let Azure AI Search generate embeddings for you — at both index time and query time.

🔧 Manual Pipeline

  • You call embedding API
  • You manage vectors
  • You embed queries yourself
  • More control, more code

⚡ Integrated Vectorization

  • Search calls embedding API
  • Vectors auto-generated
  • Queries auto-embedded
  • Less code, simpler arch

Performance & Scaling

Replicas & Partitions

  • Replicas → more query throughput
  • Partitions → more storage capacity
  • ≥ 3 replicas for HA SLA
  • ≥ 2 replicas for read HA

Optimization Tips

  • Use filters to narrow search scope
  • Choose appropriate vector dimensions
  • Monitor with Azure Monitor
  • Cache frequent queries
