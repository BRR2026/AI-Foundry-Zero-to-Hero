### Question 1

**What is the primary purpose of Azure AI Search in an AI application?**

- A) Training machine learning models on large datasets
- B) Providing intelligent information retrieval including full-text, semantic, and vector search
- C) Hosting large language models for inference
- D) Managing virtual machines for compute workloads

<details>
<summary>Answer</summary>

**B) Providing intelligent information retrieval including full-text, semantic, and vector search**

Azure AI Search is a dedicated search service that supports full-text search, semantic ranking, and vector search. It acts as the retrieval backbone for AI applications, particularly in RAG architectures.

</details>

### Question 2

**What are embeddings in the context of vector search?**

- A) Compressed versions of original documents stored as ZIP files
- B) Numerical vector representations of text that capture semantic meaning
- C) Database indexes that speed up keyword lookups
- D) Encryption keys used to secure search queries

<details>
<summary>Answer</summary>

**B) Numerical vector representations of text that capture semantic meaning**

Embeddings are high-dimensional numerical vectors generated by models like `text-embedding-ada-002`. They capture semantic meaning, allowing similar concepts to be near each other in vector space.

</details>

### Question 3

**Which algorithm does Azure AI Search use by default for approximate nearest neighbor (ANN) vector search?**

- A) Inverted File Index (IVF)
- B) Locality-Sensitive Hashing (LSH)
- C) Hierarchical Navigable Small World (HNSW)
- D) Random Projection Trees

<details>
<summary>Answer</summary>

**C) Hierarchical Navigable Small World (HNSW)**

Azure AI Search uses HNSW as its default ANN algorithm. HNSW provides an excellent balance between search speed and recall accuracy for high-dimensional vector data.

</details>

### Question 4

**What does hybrid search in Azure AI Search combine?**

- A) SQL queries and NoSQL queries
- B) Keyword-based BM25 scoring and vector similarity search
- C) Real-time and batch processing
- D) Structured and unstructured data storage

<details>
<summary>Answer</summary>

**B) Keyword-based BM25 scoring and vector similarity search**

Hybrid search combines traditional keyword-based BM25 scoring with vector similarity search. Results from both methods are merged using Reciprocal Rank Fusion (RRF). Semantic ranking can be added as a third layer.

</details>

### Question 5

**What is Reciprocal Rank Fusion (RRF) used for in Azure AI Search?**

- A) Encrypting search results before returning them to the client
- B) Merging and re-ranking results from multiple search methods (keyword + vector)
- C) Splitting documents into smaller chunks for indexing
- D) Compressing vector embeddings to save storage space

<details>
<summary>Answer</summary>

**B) Merging and re-ranking results from multiple search methods (keyword + vector)**

RRF is a rank-fusion algorithm that combines ranked lists from different retrieval methods into a single unified ranking. It ensures that documents scoring well across multiple methods are prioritized.

</details>

### Question 6

**Which field type in Azure AI Search is used to store vector embeddings?**

- A) `Edm.String`
- B) `Edm.Int32`
- C) `Collection(Edm.Single)`
- D) `Edm.Binary`

<details>
<summary>Answer</summary>

**C) `Collection(Edm.Single)`**

Vector embeddings are stored as `Collection(Edm.Single)`, which represents an array of single-precision floating-point numbers. The dimension count is specified in the field's vector search configuration.

</details>

### Question 7

**What is integrated vectorization in Azure AI Search?**

- A) A feature that automatically generates embeddings at both indexing and query time
- B) A method to combine multiple search indexes into one
- C) A process for converting images to text before indexing
- D) A tool for monitoring search performance metrics

<details>
<summary>Answer</summary>

**A) A feature that automatically generates embeddings at both indexing and query time**

Integrated vectorization allows Azure AI Search to call an embedding model automatically when documents are indexed and when queries are received. This eliminates the need for a separate embedding pipeline in your application code.

</details>

### Question 8

**When designing an index schema for hybrid search, which combination of field attributes is recommended for a primary content field?**

- A) `SimpleField` with `filterable=True` only
- B) `SearchableField` with a language analyzer, paired with a separate vector field
- C) `ComplexField` with nested sub-fields
- D) `SimpleField` with `sortable=True` and `facetable=True`

<details>
<summary>Answer</summary>

**B) `SearchableField` with a language analyzer, paired with a separate vector field**

For hybrid search, you need a `SearchableField` for keyword/BM25 search (with a language analyzer like `en.microsoft`) and a separate vector field (`Collection(Edm.Single)`) for vector search. This enables both retrieval methods on the same content.

</details>

### Question 9

**Which similarity metric is most commonly used for comparing text embeddings in Azure AI Search?**

- A) Jaccard similarity
- B) Hamming distance
- C) Cosine similarity
- D) Manhattan distance

<details>
<summary>Answer</summary>

**C) Cosine similarity**

Cosine similarity measures the angle between two vectors, making it effective for comparing text embeddings where the direction (semantic meaning) matters more than magnitude. It is the default metric for HNSW in Azure AI Search.

</details>

### Question 10

**How do you scale Azure AI Search for higher query throughput and larger indexes?**

- A) By adding more vector fields to the index schema
- B) By increasing replicas for throughput and partitions for storage capacity
- C) By switching from HNSW to exhaustive KNN for all queries
- D) By reducing the embedding dimensions to 64

<details>
<summary>Answer</summary>

**B) By increasing replicas for throughput and partitions for storage capacity**

Replicas add copies of the index to handle more concurrent queries, while partitions divide the index across multiple storage units for larger datasets. For high availability, Microsoft recommends at least 3 replicas.

</details>
