M365 Copilot Extensibility Model

Three primary ways to extend Copilot with custom intelligence:

Extension TypeDescriptionBest ForComplexity
Declarative Agents Manifest-defined agents with instructions & knowledge Scoped Q&A, domain assistants ⭐ Low–Medium
API Plugins OpenAPI REST endpoints invoked by Copilot Real-time data, transactions ⭐⭐ Medium
Message Extensions Teams search/action extensions in compose Teams-centric search, cards ⭐⭐⭐ Medium–High

💡 Graph Connectors complement these by ingesting external data into Microsoft Graph — making it natively available to Copilot without a plugin call.

Declarative Agents Deep Dive

The fastest path to a custom M365 Copilot experience:

What You Define

  • Instructions — Custom system prompt
  • Knowledge — SharePoint, OneDrive, Graph
  • Actions — API plugin references
  • Starters — Suggested conversation prompts
  • Capabilities — Code, web search, files
// declarativeAgent.json { "name": "KB Assistant", "instructions": "Answer using the knowledge base. Cite sources.", "capabilities": [{ "name": "OneDriveAndSharePoint" }], "actions": [{ "$ref": "ai-plugin.json" }] }

API Plugins — OpenAPI Integration

Expose your AI Foundry endpoints as OpenAPI-described services for Copilot:

How It Works

  • Copilot reads your OpenAPI spec
  • Matches user intent to operations
  • Extracts parameters from conversation
  • Calls your endpoint with structured data
  • Formats response for the user

🔑 Key: Write descriptive operationId and parameter description fields — Copilot uses these for intent matching!

# OpenAPI spec paths: /api/knowledge/search: post: operationId: searchKnowledgeBase summary: Search the enterprise knowledge base requestBody: content: application/json: schema: properties: query: type: string description: User's search query

Security & Authentication

Authentication Layers

  • Azure AD SSO — User authenticates via M365 identity
  • OBO Flow — Service acts on behalf of the user
  • Managed Identity — No secrets for service-to-service
  • Private Endpoints — No public internet traversal
  • Key Vault — Centralized secret management

Best Practices

✅ Do

  • Write descriptive operation IDs & parameter descriptions
  • Return structured JSON with citations
  • Use managed identity for backend calls
  • Implement OBO for user-scoped data
  • Keep responses under 10 seconds
  • Cache frequent queries with Redis

❌ Don't

  • Use vague descriptions that confuse routing
  • Hardcode API keys in code or manifests
  • Return unstructured text blobs
  • Skip admin consent configuration
  • Expose endpoints without authentication
  • Ignore sensitivity labels on documents

Key Takeaways

  • Three extensibility patterns: Declarative Agents (simplest), API Plugins (most flexible), Message Extensions (Teams-rich)
  • AI Foundry is the intelligence layer: RAG, agents, and prompt flows power the plugin backends
  • Copilot Studio orchestrates: Multi-agent routing for complex enterprise scenarios
  • Compliance is non-negotiable: Match regions, implement OBO, honor sensitivity labels
  • Descriptions drive Copilot: Well-written operation IDs and parameters are critical for intent matching
  • Teams Toolkit accelerates: Scaffold, debug, and deploy Copilot extensions end-to-end
