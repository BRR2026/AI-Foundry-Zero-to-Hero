Foundation

Why Governance & Security Matter

🚨 Risk

AI systems without governance are enterprise liabilities — data breaches, compliance violations, brand damage.

✅ Trust

Stakeholders require proof of security controls before approving production deployments of AI solutions.

📋 Compliance

GDPR, HIPAA, SOC 2, and industry regulations demand auditable access controls and data protection.

Security is not a feature — it's a prerequisite.

Identity

RBAC Assignment Model

Security Principal + Role Definition + Scope = Role Assignment ───────────────── ──────────────── ───── ─────────────── • User • Reader • Subscription → Allow/Deny • Group • Contributor • Resource Group actions • Service Principal • Azure AI Developer • Resource at scope • Managed Identity • Custom Role • AI Project
  • Always assign at the narrowest scope possible
  • Use Entra ID security groups — never assign to individuals
  • Avoid Owner role on AI resources

Identity

Managed Identity Patterns

🔐 System-Assigned

Created with the resource. 1:1 relationship. Deleted when resource is deleted.

Best for: Single-purpose workloads

🪪 User-Assigned

Standalone resource. Shared across multiple services. Independent lifecycle.

Best for: Multi-service architectures

# Python — credential-free auth everywhere from azure.identity import DefaultAzureCredential credential = DefaultAzureCredential() # Works in Azure (Managed ID) and locally (CLI / VS Code)
Anti-Pattern

🚫 Never Hardcode Secrets

❌ Don't Do This

# DANGER: secrets in code api_key = "sk-abc123..." client = OpenAI(api_key=api_key)

✅ Do This Instead

# SAFE: managed identity credential = DefaultAzureCredential() client = AIProjectClient( credential=credential)
  • Hardcoded keys are the #1 cause of production security incidents
  • Use Key Vault references for any external (non-Azure) secrets
Observability

Monitoring Dashboards

📊 API Latency

Track P50 / P95 / P99 response times per deployment. Alert on SLA breaches.

⚠️ Error Rates

Monitor 4xx/5xx rates. Watch for 429 throttling spikes indicating capacity issues.

🔢 Token Usage

Prompt + completion tokens per deployment. Essential for cost management.

💰 Cost Tracking

Budget alerts at 50%, 80%, 100%. Break down costs by team and deployment.

KQL

Key Monitoring Queries

// API latency percentiles — last 24 hours AzureDiagnostics | where ResourceProvider == "MICROSOFT.COGNITIVESERVICES" | where TimeGenerated > ago(24h) | summarize P50 = percentile(DurationMs, 50), P95 = percentile(DurationMs, 95), P99 = percentile(DurationMs, 99) by bin(TimeGenerated, 1h) // Error rate breakdown AzureDiagnostics | summarize Errors4xx = countif(httpStatusCode_d >= 400 and httpStatusCode_d < 500), Errors5xx = countif(httpStatusCode_d >= 500) by bin(TimeGenerated, 15m)

Network

Network Isolation with Private Endpoints

  • Private endpoints for all AI resources
  • Public network access disabled
  • Private DNS zones for resolution
  • NSGs restrict to required ports only
  • TLS 1.2+ enforced everywhere

🔒 Zero Trust

No resource is accessible from the public internet. All traffic flows through the VNet via private endpoints with DNS-based resolution.

Managed VNet (Preview): AI Foundry can auto-provision private endpoints for all dependencies.

Hardening

Security Hardening Checklist

Identity & Access

  • Managed Identity — no API keys
  • RBAC at minimum scope
  • MFA via Conditional Access
  • PIM for elevated roles
  • 90-day credential rotation

Data & Network

  • Encryption at rest (CMK)
  • TLS 1.2+ in transit
  • Key Vault for all secrets
  • Private endpoints enabled
  • Defender for Cloud active

Summary

Key Takeaways

  • RBAC is non-negotiable — minimum role, narrowest scope, group-based assignments
  • Managed Identity eliminates secrets — DefaultAzureCredential everywhere
  • Monitor everything — latency, errors, tokens, cost in a single dashboard
  • Content safety is multi-layered — input filters + output filters + blocklists
  • Network isolation is essential — private endpoints + disabled public access
  • Compliance requires automation — Azure Policy + Purview + audit logs
  • Security is continuous — build reviews into CI/CD and sprint cadence
