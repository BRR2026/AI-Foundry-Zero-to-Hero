📊 AI Foundry Pricing Models

🔒

Provisioned Throughput Units (PTU)

  • Reserved capacity — guaranteed throughput
  • Fixed monthly cost, predictable billing
  • Ideal for steady, high-volume workloads
  • Committed SLA for latency & availability
  • 1-month or 1-year reservation terms
Best for: Production steady-state
⚡

Pay-as-You-Go (Token-Based)

  • Per-token pricing — pay only for what you use
  • No upfront commitment required
  • Ideal for variable, bursty workloads
  • Instant access to all available models
  • Subject to shared capacity & throttling
Best for: Dev/test & variable loads

🔒 Provisioned Throughput Units (PTU)

  • Each PTU provides a fixed amount of model processing capacity measured in tokens-per-minute
  • Capacity is dedicated — not shared with other customers, ensuring consistent performance
  • Includes higher SLA guarantees (99.9%+) and lower p95 latency compared to PAYG
  • Reservations are scoped to a specific model and version within a region
  • Use the AI Foundry capacity calculator to estimate PTU requirements from your TPM needs

🔧 Token Usage Optimization

📦

Prompt Compression

  • Remove redundant instructions and examples
  • Use concise system prompts with clear roles
  • Compress few-shot examples to minimal tokens
  • Leverage structured output (JSON mode) to reduce retries
Saves 15-30%
📏

Response Management

  • Set max_tokens to cap output length
  • Use temperature: 0 for deterministic tasks
  • Request only needed fields with structured outputs
  • Stream responses and stop early when sufficient
Saves 10-25%
💾

Semantic Caching

  • Cache identical or semantically similar prompts
  • Use Azure Redis or Cosmos DB for cache store
  • Set TTL based on content freshness needs
  • Implement embedding-based similarity matching
Saves 20-50%

📋 Quota Management

📈

Tokens-Per-Minute (TPM)

  • Controls total throughput to a deployment
  • Shared across all requests hitting the endpoint
  • Default varies by model and subscription tier
  • Can be split across multiple deployments
🔄

Requests-Per-Minute (RPM)

  • Caps the number of API calls per minute
  • Automatically derived from TPM allocation
  • Prevents small-request flooding
  • Monitor via AI Foundry metrics pane
🌍

Regional Quotas

  • Quotas are region- and model-specific
  • Spread workloads across regions for higher aggregate capacity
  • Check availability per region in the portal
📬

Quota Increase Process

  • Request via Azure Portal → Quotas page
  • Provide business justification & usage patterns
  • Typical turnaround: 1–3 business days

📐 Capacity Planning Framework

  1. Profile your workload — measure avg. tokens per request (input + output) during pilot phase
  2. Estimate peak RPM — identify busiest hour, add 30% headroom for spikes
  3. Calculate TPM need — multiply avg tokens × peak RPM to get required tokens-per-minute
  4. Choose pricing model — if sustained TPM > breakeven threshold, PTU is more cost-effective
  5. Plan multi-region — distribute deployments if single-region quota is insufficient
  6. Set up autoscaling — configure dynamic deployment scaling policies
# Capacity calculation example

avg_input_tokens = 800 # per request
avg_output_tokens = 400 # per request
tokens_per_req = 1200 # total

peak_rpm = 500 # requests/min
headroom = 1.3 # 30% buffer

# Required TPM
required_tpm = tokens_per_req × peak_rpm × headroom

# = 780,000 TPM

🛡️ Budget Alerts & Cost Governance

Setting Up Budget Alerts

  1. Navigate to Azure Cost Management → Budgets
  2. Create a budget scoped to your AI Foundry resource group
  3. Set thresholds at 50%, 75%, 90%, and 100% of monthly budget
  4. Configure alert recipients — email, Teams webhook, or Action Group
  5. Add an action group to auto-disable deployments at 100%

Tagging Strategy

# Required tags for cost allocation
project: "customer-chatbot"
team: "platform-ai"
env: "production"
cost-center: "CC-4200"
model-tier: "standard"
⚠️ Governance Policy: Use Azure Policy to enforce mandatory tags on all AI Foundry resources — block deployment if tags are missing.
