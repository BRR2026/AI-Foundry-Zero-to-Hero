🔢 Model Versioning & Registry

Azure AI Foundry's Model Registry serves as the single source of truth for all models in your organization. Every model — whether fine-tuned, custom-built, or sourced from the model catalog — is registered with immutable version metadata, enabling complete lifecycle tracking.

Understanding the Model Registry

The Model Registry in Azure AI Foundry provides a centralized catalog where teams register, version, and manage models across multiple projects. Each model version captures the model artifacts, metadata, tags, and lineage information necessary for governance.

📦

Immutable Versions

Each registered version is immutable — once created, the artifacts and metadata cannot be altered, ensuring audit integrity.

🏷️

Semantic Tagging

Apply tags like "production", "staging", "candidate" to track model lifecycle stages within the registry.

🔗

Lineage Tracking

Automatically capture training data references, hyperparameters, and evaluation metrics for each version.

🌐

Cross-Project Sharing

Share registered models across Foundry projects while maintaining centralized governance policies.

Registering Models with the Azure AI Foundry SDK

Python
from azure.ai.ml import MLClient
from azure.ai.ml.entities import Model
from azure.ai.ml.constants import AssetTypes
from azure.identity import DefaultAzureCredential

# Connect to your Azure AI Foundry project
client = MLClient(
    credential=DefaultAzureCredential(),
    subscription_id="your-subscription-id",
    resource_group_name="rg-ai-governance",
    workspace_name="foundry-project-prod"
)

# Register a new model version with governance metadata
model = Model(
    name="customer-sentiment-classifier",
    version="3",
    path="./model-artifacts/sentiment-v3",
    type=AssetTypes.CUSTOM_MODEL,
    description="Sentiment classifier v3 — retrained with Q1 2026 data",
    tags={
        "team": "nlp-engineering",
        "stage": "candidate",
        "framework": "transformers",
        "training_dataset": "reviews-q1-2026",
        "accuracy": "0.947",
        "approval_status": "pending_review",
        "risk_tier": "high",
        "data_classification": "confidential"
    },
    properties={
        "training_compute": "Standard_NC24ads_A100_v4",
        "training_duration_hours": "4.2",
        "base_model": "bert-base-uncased",
        "evaluation_dataset": "reviews-eval-q1-2026"
    }
)

registered_model = client.models.create_or_update(model)
print(f"Registered: {registered_model.name} v{registered_model.version}")

Best Practice: Semantic Version Tagging

Use tags to indicate model lifecycle stages rather than encoding stage names in version numbers. This allows the same version number to progress through stages (candidate → staging → production) while maintaining immutable version identifiers.

Querying Audit Logs with Azure Monitor

KQL
// Query model deployment events from Azure Activity Log
AzureActivity
| where ResourceProvider == "Microsoft.MachineLearningServices"
| where OperationNameValue has_any (
    "deployments/write",
    "models/write",
    "models/delete"
  )
| where TimeGenerated > ago(30d)
| project
    TimeGenerated,
    Caller,
    OperationNameValue,
    ResourceGroup,
    ActivityStatusValue,
    Properties_d
| order by TimeGenerated desc
KQL
// Track endpoint access patterns and anomalies
AMLOnlineEndpointConsoleLog
| where TimeGenerated > ago(7d)
| summarize
    RequestCount = count(),
    AvgLatencyMs = avg(DurationMs),
    ErrorCount = countif(ResultType != "Success")
    by bin(TimeGenerated, 1h), EndpointName
| order by TimeGenerated desc

Implementing Approval Gates with GitHub Actions

YAML — .github/workflows/model-promotion.yml
name: Model Promotion Pipeline
on:
  workflow_dispatch:
    inputs:
      model_name:
        description: "Model name in registry"
        required: true
      model_version:
        description: "Version to promote"
        required: true
      target_environment:
        description: "Target: staging | production"
        required: true
        type: choice
        options:
          - staging
          - production

jobs:
  validate-model:
    runs-on: ubuntu-latest
    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Validate model governance metadata
        run: |
          # Verify the model has required governance tags
          python scripts/validate_governance.py \
            --model-name ${{ github.event.inputs.model_name }} \
            --model-version ${{ github.event.inputs.model_version }} \
            --required-tags "risk_tier,approval_status,data_classification"

      - name: Run automated evaluation suite
        run: |
          python scripts/run_evaluation.py \
            --model-name ${{ github.event.inputs.model_name }} \
            --model-version ${{ github.event.inputs.model_version }} \
            --eval-dataset "governance-eval-suite"

  approval-gate:
    needs: validate-model
    runs-on: ubuntu-latest
    environment: ${{ github.event.inputs.target_environment }}
    # ↑ GitHub Environment with required reviewers configured
    steps:
      - name: Approval received — proceed with deployment
        run: echo "Approval granted for ${{ github.event.inputs.model_name }} v${{ github.event.inputs.model_version }}"

  deploy-model:
    needs: approval-gate
    runs-on: ubuntu-latest
    steps:
      - name: Deploy to Azure AI Foundry
        uses: azure/login@v2
        with:
          creds: ${{ secrets.AZURE_CREDENTIALS }}

      - name: Promote model and deploy endpoint
        run: |
          python scripts/deploy_model.py \
            --model-name ${{ github.event.inputs.model_name }} \
            --model-version ${{ github.event.inputs.model_version }} \
            --environment ${{ github.event.inputs.target_environment }}

Governance Validation Script

Python — scripts/validate_governance.py
import argparse
from azure.ai.ml import MLClient
from azure.identity import DefaultAzureCredential

def validate_governance(model_name, model_version, required_tags):
    client = MLClient(
        credential=DefaultAzureCredential(),
        subscription_id="your-subscription-id",
        resource_group_name="rg-ai-governance",
        workspace_name="foundry-project-prod"
    )

    model = client.models.get(name=model_name, version=model_version)
    missing_tags = [t for t in required_tags if t not in model.tags]

    if missing_tags:
        raise ValueError(
            f"Model {model_name} v{model_version} missing required "
            f"governance tags: {missing_tags}"
        )

    # Verify risk tier has been assessed
    if model.tags.get("risk_tier") == "high":
        if model.tags.get("approval_status") != "approved":
            raise ValueError(
                "High-risk models require explicit approval before promotion"
            )

    print(f"✅ Governance validation passed for {model_name} v{model_version}")
    return True

if __name__ == "__main__":
    parser = argparse.ArgumentParser()
    parser.add_argument("--model-name", required=True)
    parser.add_argument("--model-version", required=True)
    parser.add_argument("--required-tags", required=True)
    args = parser.parse_args()
    validate_governance(
        args.model_name,
        args.model_version,
        args.required_tags.split(",")
    )

Tip: Separate Environments for Risk Tiers

Configure separate GitHub Environments (or Azure DevOps Environments) for different risk tiers. High-risk models should require multiple reviewers, including a compliance officer, while low-risk models may only need a single ML lead approval.

📄 Model Documentation & Model Cards

Model cards are standardized documents that describe a model's intended use, performance characteristics, limitations, and ethical considerations. In Azure AI Foundry, model cards serve as the primary documentation artifact for governance, ensuring every deployed model has transparent, accessible documentation for stakeholders.

Model Card Framework

A comprehensive model card should cover the following sections, aligning with both the original Google Model Cards paper and regulatory requirements like the EU AI Act:

🎯

Intended Use

Primary use cases, intended users, and out-of-scope applications that should be avoided.

📊

Performance Metrics

Accuracy, precision, recall, F1, and disaggregated performance across subgroups.

⚠️

Limitations

Known failure modes, bias assessments, and environmental conditions where performance degrades.

🔍

Training Data

Data sources, preprocessing steps, representation analysis, and data quality assessments.

⚖️

Ethical Considerations

Fairness evaluations, potential harms, mitigation strategies, and human oversight requirements.

📋

Governance Metadata

Owner, approval history, risk classification, deployment history, and compliance mappings.

Creating Model Cards in Azure AI Foundry

Python
import json
from datetime import datetime

def generate_model_card(model_name, version, metrics, config):
    """Generate a structured model card for governance compliance."""
    model_card = {
        "schema_version": "1.0",
        "model_details": {
            "name": model_name,
            "version": version,
            "type": config["model_type"],
            "framework": config["framework"],
            "owner": config["owner"],
            "created_date": datetime.utcnow().isoformat(),
            "last_updated": datetime.utcnow().isoformat()
        },
        "intended_use": {
            "primary_uses": config["primary_uses"],
            "primary_users": config["primary_users"],
            "out_of_scope": config["out_of_scope"]
        },
        "performance": {
            "metrics": metrics,
            "evaluation_data": config["eval_dataset"],
            "disaggregated_results": config.get("disaggregated", {})
        },
        "limitations": config.get("limitations", []),
        "ethical_considerations": {
            "fairness_assessment": config.get("fairness", "pending"),
            "bias_mitigations": config.get("mitigations", []),
            "human_oversight": config.get("oversight", "required")
        },
        "governance": {
            "risk_tier": config["risk_tier"],
            "data_classification": config["data_classification"],
            "approval_status": "pending_review",
            "regulatory_mappings": config.get("regulations", [])
        }
    }
    return model_card

# Example usage

    },
    config={
        "model_type": "text-classification",
        "framework": "transformers",
        "owner": "nlp-engineering@contoso.com",
        "primary_uses": ["Customer feedback sentiment analysis"],
        "primary_users": ["Customer Experience team"],
        "out_of_scope": ["Medical or legal sentiment analysis"],
        "eval_dataset": "reviews-eval-q1-2026",
        "risk_tier": "high",
        "data_classification": "confidential",
        "limitations": [
            "Performance degrades on texts shorter than 10 words",
            "Not validated for languages other than English"
        ],
        "regulations": ["EU AI Act Art. 13", "ISO 42001"]
    }
)

3 Implement the Approval Gate 
def check_approval_gate(client, model_name, version, governance_config):
    """Enforce governance approval gate before model promotion."""
    model = client.models.get(name=model_name, version=version)
    risk_tier = model.tags.get("risk_tier", "unclassified")
    rules = governance_config["risk_tiers"].get(risk_tier, {})

    # Check required governance tags
    for tag in governance_config["required_tags"]:
        if tag not in model.tags:
            return False, f"Missing required tag: {tag}"

    # Enforce model card requirement
    if rules.get("require_model_card") and model.tags.get("has_model_card") != "true":
        return False, "Model card is required for this risk tier"

    # Enforce fairness evaluation
    if rules.get("require_fairness_eval") and model.tags.get("fairness_evaluated") != "true":
        return False, "Fairness evaluation required for this risk tier"

    return True, "All governance checks passed"

# Run the approval gate
passed, message = check_approval_gate(
    client, "fraud-detection-model", "2", governance
)
print(f"Gate result: {message}")

if passed:
    # Update approval status
    model = client.models.get(name="fraud-detection-model", version="2")
    model.tags["approval_status"] = "approved"
    model.tags["approved_by"] = "mlops-lead@contoso.com"
    model.tags["approved_at"] = datetime.utcnow().isoformat()
    client.models.create_or_update(model)
