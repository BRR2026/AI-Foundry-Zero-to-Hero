🔢 Registering Models with Governance Tags

from azure.ai.ml import MLClient
from azure.ai.ml.entities import Model
from azure.ai.ml.constants import AssetTypes

model = Model(
    name="customer-sentiment-classifier",
    version="3",
    path="./model-artifacts/sentiment-v3",
    type=AssetTypes.CUSTOM_MODEL,
    tags={
        "risk_tier": "high",
        "approval_status": "pending_review",
        "data_classification": "confidential",
        "owner": "nlp-engineering@contoso.com"
    }
)

registered = client.models.create_or_update(model)

🔍 Querying Audit Logs with KQL

// All model lifecycle events in the last 30 days
AzureActivity
| where ResourceProvider == "Microsoft.MachineLearningServices"
| where OperationNameValue has_any (
    "deployments/write", "models/write", "models/delete"
  )
| where TimeGenerated > ago(30d)
| project TimeGenerated, Caller, OperationNameValue,
          ActivityStatusValue
| order by TimeGenerated desc

💡 Set log retention to 365+ days for EU AI Act compliance. Use Azure Monitor archival tiers for cost-effective storage.

⚙️ Approval Gates with GitHub Actions

jobs:
  validate-model:
    runs-on: ubuntu-latest
    steps:
      - name: Validate governance metadata
        run: python scripts/validate_governance.py \
          --model-name ${{ inputs.model_name }} \
          --required-tags "risk_tier,approval_status"

  approval-gate:
    needs: validate-model
    environment: production   # ← Required reviewers
    steps:
      - run: echo "Approval granted"

  deploy-model:
    needs: approval-gate
    steps:
      - uses: azure/login@v2
      - run: python scripts/deploy_model.py \
          --environment production
