🔑 DefaultAzureCredential

The Python SDK uses DefaultAzureCredential which automatically picks up your az login session — no API keys needed during development!

⚠️ Common Pitfall

Make sure you select the subscription that has your AI Foundry quota. Wrong subscription = "Resource not found" errors.

Deploying GPT-4o-mini

Why GPT-4o-mini?

The perfect model for learning — high capability at a fraction of the cost.

Model Input / 1M tokens Output / 1M tokens
GPT-4o-mini $0.15 $0.60
GPT-4o $2.50 $10.00
GPT-4 Turbo $10.00 $30.00

💰 Cost Savings: GPT-4o-mini is ~17× cheaper than GPT-4o on input tokens. Perfect for development & learning.

Python Code

Hello World — Python SDK

hello_world.py 
import os
from dotenv import load_dotenv
from azure.ai.inference import ChatCompletionsClient
from azure.ai.inference.models import SystemMessage, UserMessage
from azure.core.credentials import AzureKeyCredential

# Load environment variables from .env file
load_dotenv()

# Create the inference client
client = ChatCompletionsClient(
    endpoint=os.getenv("AZURE_AI_ENDPOINT"),
    credential=AzureKeyCredential(os.getenv("AZURE_OPENAI_API_KEY"))
)

# Send a chat completion request
response = client.complete(
    model="gpt-4o-mini",
    messages=[
        SystemMessage(content="You are a helpful assistant."),
        UserMessage(content="Say 'Hello, World!'")
    ]
)

# Print the response
print(response.choices[0].message.content)

▶️ Run it: python src/hello_world.py  →  Expected output: Hello, World! 👋

Response Anatomy

Understanding the Response

JSON Response 
{
  "id": "chatcmpl-abc123...",
  "model": "gpt-4o-mini",
  "choices": [
    {
      "index": 0,
      "message": {
        "role": "assistant",
        "content": "Hello, World! 👋"
      },
      "finish_reason": "stop"
    }
  ],
  "usage": {
    "prompt_tokens": 25,
    "completion_tokens": 5,
    "total_tokens": 30
  }
}

Key Fields

choices[0].message.content

The actual text response from the model. Always access choices[0] for single responses.

usage

Token consumption breakdown. Track this for cost management. prompt_tokens = input, completion_tokens = output.

finish_reason

"stop" Normal completion ✅
"length" Hit max_tokens limit ⚠️
"content_filter" Content was filtered 🛡️

Capstone Exercise

Building a Simple Chatbot

chatbot.py 
import os
from dotenv import load_dotenv
from azure.ai.inference import ChatCompletionsClient
from azure.ai.inference.models import (
    SystemMessage, UserMessage, AssistantMessage
)
from azure.core.credentials import AzureKeyCredential

load_dotenv()

client = ChatCompletionsClient(
    endpoint=os.getenv("AZURE_AI_ENDPOINT"),
    credential=AzureKeyCredential(
        os.getenv("AZURE_OPENAI_API_KEY")
    )
)

# Conversation history
messages = [
    SystemMessage(content="You are a friendly AI tutor.")
]

print("Chat started! Type 'quit' to exit.\n")

while True:
    user_input = input("You: ")
    if user_input.lower() == "quit":
        break

    messages.append(UserMessage(content=user_input))

    response = client.complete(
        model="gpt-4o-mini",
        messages=messages
    )

    reply = response.choices[0].message.content
    print(f"Bot: {reply}\n")
    messages.append(AssistantMessage(content=reply))

Key Concepts

💬

Conversation History

Messages list grows with each turn. The model sees the entire conversation on every request — this is how it maintains context.

⚡

Streaming (Advanced)

Use client.complete(..., stream=True) to get tokens as they're generated — reduces perceived latency.

📏

Token Management

GPT-4o-mini has a 128K token context window. For long conversations, implement a sliding window or summarization strategy to stay within limits.

🎉 Try it! Run python src/chatbot.py and have a conversation with your bot!
