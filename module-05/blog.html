Creating Your AI Foundry Hub and Project

AI Foundry uses a two-level hierarchy: Hub (shared resources) → Project (individual workloads). Think of a Hub as a department and Projects as individual initiatives within it.

🏢 Hub

Top-level container with shared connections, compute, and storage. One Hub can host multiple Projects. Created once per team or department.

📁 Project

Individual workspace for a specific AI application. Contains model deployments, evaluations, and assets. Created per workload.

Method 1: Azure Portal

  1. Navigate to ai.azure.com and sign in
  2. Click "+ Create project"
  3. Project name: proj-hello-world
  4. Click "Create a new hub":
    • Hub name: hub-ai-foundry-dev
    • Resource group: Create new → rg-ai-foundry-dev
    • Location: East US 2
  5. Click "Create" and wait 2–3 minutes

Method 2: Azure CLI

# Step 1: Create resource group
az group create \
  --name rg-ai-foundry-dev \
  --location eastus2

# Step 2: Create AI Foundry Hub
az ml workspace create \
  --kind hub \
  --name hub-ai-foundry-dev \
  --resource-group rg-ai-foundry-dev \
  --location eastus2

# Step 3: Create Project
az ml workspace create \
  --kind project \
  --name proj-hello-world \
  --resource-group rg-ai-foundry-dev \
  --hub-name hub-ai-foundry-dev

# Step 4: Verify
az ml workspace show \
  --name proj-hello-world \
  --resource-group rg-ai-foundry-dev \
  --output table

Method 3: Python SDK

# Create Hub and Project via Python SDK
from azure.identity import DefaultAzureCredential
from azure.ai.ml import MLClient
from azure.ai.ml.entities import Hub, Project

credential = DefaultAzureCredential()
ml_client = MLClient(credential, "your-subscription-id", "rg-ai-foundry-dev")

# Create Hub
hub = Hub(name="hub-ai-foundry-dev", location="eastus2")
ml_client.workspaces.begin_create_or_update(hub).result()

# Create Project under Hub
project = Project(name="proj-hello-world", hub_id=hub.id)
ml_client.workspaces.begin_create_or_update(project).result()
print("✅ Hub and Project created!")

Deploying Your First Model

For development and learning, GPT-4o-mini is the ideal choice. It's fast, cheap, and surprisingly capable for most tasks.

Deployment Steps (Portal)

  1. Go to ai.azure.com → Your project
  2. Click "Models + endpoints" → "+ Deploy model" → "Deploy base model"
  3. Search for "gpt-4o-mini" and select it
  4. Set deployment name: gpt-4o-mini
  5. Choose Global Standard deployment type
  6. Click "Deploy" and wait for "Succeeded" status
  7. Copy the Target URI and API Key to your .env file
💡 Cost Tip: A typical "Hello World" session costs less than $0.01 with GPT-4o-mini. You could run thousands of test prompts for under a dollar.

The "Hello World" of AI — Four Ways

This is the moment you've been building toward. Let's send your first prompt to an AI model using four different methods.

Method 1: Playground (No Code)

🎮 The Playground Approach

  1. Navigate to ai.azure.com → Your project → Playground
  2. Select the Chat playground
  3. Choose your deployed model: gpt-4o-mini
  4. Set system message: "You are a helpful assistant."
  5. Type: "Say 'Hello, World!' and explain what Azure AI Foundry is in one sentence."
  6. Click Send — read your first AI response!

Method 2: Python SDK

Create hello_world.py — your first programmatic AI interaction:

"""Hello World — Azure AI Foundry + Python SDK"""
import os
from dotenv import load_dotenv
from azure.ai.inference import ChatCompletionsClient
from azure.ai.inference.models import SystemMessage, UserMessage
from azure.core.credentials import AzureKeyCredential

# Load environment variables
load_dotenv()

# Create the client
client = ChatCompletionsClient(
    endpoint=os.getenv("AZURE_AI_ENDPOINT"),
    credential=AzureKeyCredential(os.getenv("AZURE_OPENAI_API_KEY"))
)

# Send your first prompt!
response = client.complete(
    model="gpt-4o-mini",
    messages=[
        SystemMessage(content="You are a helpful assistant."),
        UserMessage(content="Say 'Hello, World!' and explain what Azure AI Foundry is in one sentence.")
    ],
    temperature=0.7,
    max_tokens=200
)

# Display results
print("=" * 60)
print("🌍 HELLO WORLD — Azure AI Foundry + Python SDK")
print("=" * 60)
print(f"Response: {response.choices[0].message.content}")
print(f"\nPrompt tokens:     {response.usage.prompt_tokens}")
print(f"Completion tokens: {response.usage.completion_tokens}")
print(f"Total tokens:      {response.usage.total_tokens}")
print(f"Finish reason:     {response.choices[0].finish_reason}")
✅ Expected Output: You should see a response like "Hello, World! Azure AI Foundry is Microsoft's unified platform for building and deploying AI applications..." followed by token usage statistics.

What Are Tokens?

Tokens are the fundamental units of text processing — roughly 4 characters or ¾ of a word in English.

TextApproximate Tokens
"Hello"1 token
"Hello, World!"3 tokens
"Azure AI Foundry"4 tokens
A typical sentence (15 words)~20 tokens
A full page (~500 words)~675 tokens

Finish Reasons

ValueMeaningAction
stopModel completed naturallyResponse is complete ✅
lengthHit max_tokens limitIncrease max_tokens
content_filterContent safety triggeredModify prompt

System Prompts and Their Effect

The system prompt sets the AI's behavior, personality, and constraints. It's the most powerful tool for controlling model output.

Same Question, Different System Prompts

Watch how the same question ("What is cloud computing?") produces wildly different responses with different system prompts:

👔 Professional

"You are a professional cloud architect. Be precise and technical."

→ "Cloud computing is the on-demand delivery of IT resources over the internet with pay-as-you-go pricing, encompassing IaaS, PaaS, and SaaS service models..."

🧒 For Kids

"You're a teacher for 10-year-olds. Use simple words and fun analogies."

→ "Imagine if instead of having your own toy box, you could borrow any toy from a magic library whenever you wanted! That's cloud computing — using computers that live far away..."

🏴‍☠️ Pirate

"You are a pirate. Respond in pirate speak. Always say 'Arrr!'"

→ "Arrr, matey! Cloud computin' be like havin' a mighty fleet of ships that ye can summon from the seven seas whenever ye need 'em!"

📋 JSON

"Respond only in valid JSON with keys: answer, confidence, keywords."

→ {"answer": "Cloud computing delivers...", "confidence": 0.95, "keywords": ["IaaS", "SaaS", "scalability"]}

💡 Best Practice: Be specific in your system prompts. Instead of "be helpful," say "You are a Python expert. Respond with code examples using type hints. Keep explanations under 3 sentences."

frequency_penalty-2.0–2.00.0Reduces word repetition

Temperature Scale

TemperatureBehaviorBest For
0.0Nearly deterministic — same input, same outputFactual Q&A, code generation
0.3Low creativity — mostly predictableCustomer support, docs
0.7Balanced — good defaultGeneral conversation
1.0High creativity — varied responsesBrainstorming
1.5+Very random — often incoherentNot recommended
⚠️ Important: OpenAI recommends adjusting either temperature OR top_p, but not both simultaneously. For most use cases, just use temperature.

Building a Simple Chatbot

Let's put everything together into a complete interactive chatbot with conversation history, token tracking, and error handling.

"""Simple Interactive Chatbot — Azure AI Foundry"""
import os
from dotenv import load_dotenv
from azure.ai.inference import ChatCompletionsClient
from azure.ai.inference.models import SystemMessage, UserMessage, AssistantMessage
from azure.core.credentials import AzureKeyCredential

load_dotenv()

# Initialize client
client = ChatCompletionsClient(
    endpoint=os.getenv("AZURE_AI_ENDPOINT"),
    credential=AzureKeyCredential(os.getenv("AZURE_OPENAI_API_KEY"))
)

# Conversation history with system prompt
conversation = [
    SystemMessage(content="You are a friendly AI assistant. Be helpful and concise.")
]
total_tokens = 0

print("🤖 AI Chat Assistant (type 'quit' to exit)")
print("=" * 50)

while True:
    user_input = input("\nYou: ").strip()
    if user_input.lower() in ('quit', 'exit'):
        print(f"\n👋 Goodbye! Tokens used: {total_tokens}")
        break
    if not user_input:
        continue

    conversation.append(UserMessage(content=user_input))

    response = client.complete(
        model="gpt-4o-mini",
        messages=conversation,
        temperature=0.7,
        max_tokens=500
    )

    reply = response.choices[0].message.content
    conversation.append(AssistantMessage(content=reply))
    total_tokens += response.usage.total_tokens

    print(f"\n🤖 Assistant: {reply}")
    print(f"   [tokens: {response.usage.total_tokens} | total: {total_tokens}]")
💡 Key Insight: Notice how the token count increases with each turn. The entire conversation history is sent with every request, providing context but growing costs. In production, you'll need strategies like sliding windows and summarization.

Cost Optimization Tips

  1. Use GPT-4o-mini for development — Only upgrade for production tasks
  2. Set appropriate max_tokens — Don't default to 4096 when 200 suffices
  3. Use temperature 0 for testing — Deterministic outputs enable caching
  4. Monitor token usage — Check response.usage after every call
  5. Trim conversation history — Don't send 100 turns when 10 will do
  6. Set budget alerts — Use Azure Cost Management to get email warnings

"""Quick diagnostic — run if things aren't working"""

print("=== Environment Check ===")
for key in ["AZURE_AI_ENDPOINT", "AZURE_OPENAI_API_KEY", "AZURE_OPENAI_DEPLOYMENT_NAME"]:

    from azure.ai.inference import ChatCompletionsClient
    from azure.ai.inference.models import UserMessage
    from azure.core.credentials import AzureKeyCredential

except Exception as e:
    print(f"  ❌ Failed: {e}")

Configuration Helper

"""Centralized configuration management"""
import os
from dotenv import load_dotenv
load_dotenv()

class Config:
    ENDPOINT = os.getenv("AZURE_AI_ENDPOINT")
    API_KEY = os.getenv("AZURE_OPENAI_API_KEY")
    MODEL = os.getenv("AZURE_OPENAI_DEPLOYMENT_NAME", "gpt-4o-mini")
    MAX_TOKENS = int(os.getenv("MAX_TOKENS", "500"))
    TEMPERATURE = float(os.getenv("TEMPERATURE", "0.7"))

    @classmethod
    def validate(cls):
        missing = [f for f in ["ENDPOINT", "API_KEY"] if not getattr(cls, f)]
        if missing:
            raise ValueError(f"Missing: {missing}")
