2.1 Creating an Agent with Azure AI Foundry SDK

Use the azure-ai-projects SDK to create an agent that leverages your existing Azure AI Search index as a knowledge base. The agent automatically retrieves relevant chunks and synthesizes answers with citations. 

Python
from azure.ai.projects import AIProjectClient
from azure.ai.projects.models import (
    AzureAISearchTool,
    AzureAISearchQueryType,
)
from azure.identity import DefaultAzureCredential

# Connect to your AI Foundry project
client = AIProjectClient(
    credential=DefaultAzureCredential(),
    endpoint="https://<your-project>.services.ai.azure.com",
)

# Configure RAG tool with your search index
search_tool = AzureAISearchTool(
    index_connection_id="<your-search-connection-id>",
    index_name="product-knowledge-base",
    query_type=AzureAISearchQueryType.SEMANTIC,
    top_k=5,
)

# Create the agent with grounded knowledge
agent = client.agents.create_agent(
    model="GPT-5.1",
    name="product-support-agent",
    instructions="""You are a product support specialist.
Use the search tool to find relevant information before answering.
Always cite your sources. If unsure, say so honestly.""",
    tools=[search_tool],
)
print(f"Agent created: {agent.id}")

2.2 Adding Custom Function Tools

Beyond RAG retrieval, agents can invoke custom functions for actions like creating tickets, looking up order status, or triggering workflows. Define your functions as tool definitions and handle the invocations in your orchestration loop. 

Python
from azure.ai.projects.models import FunctionTool, ToolSet

# Define custom functions for the agent
def create_support_ticket(title: str, priority: str, description: str) -> dict:
    """Create a support ticket in the ticketing system."""
    # Integration with your ticketing API
    return {"ticket_id": "TKT-2026-4301", "status": "created"}

def check_order_status(order_id: str) -> dict:
    """Look up the current status of a customer order."""
    return {"order_id": order_id, "status": "shipped", "eta": "2026-04-18"}

# Register functions as tools
functions = FunctionTool(functions=[create_support_ticket, check_order_status])
toolset = ToolSet()
toolset.add(functions)
toolset.add(search_tool)

# Create agent with combined toolset
agent = client.agents.create_agent(
    model="GPT-5.1",
    name="full-support-agent",
    instructions="""You are a support agent that can:
1. Search the knowledge base for product information
2. Create support tickets for unresolved issues
3. Check order status for customers""",
    toolset=toolset,
)

2.3 Conversational Agent Loop

The agent loop manages the conversation lifecycle: create a thread, post messages, process runs, handle tool calls, and stream responses back to the user. 

Python
# Create a conversation thread
thread = client.agents.create_thread()

# Post a user message
client.agents.create_message(
    thread_id=thread.id,
    role="user",
    content="My order #ORD-9921 hasn't arrived. What's the status?",
)

# Process with the agent (auto-handles tool calls)
run = client.agents.create_and_process_run(
    thread_id=thread.id,
    agent_id=agent.id,
)

# Retrieve the agent's response
messages = client.agents.list_messages(thread_id=thread.id)
for msg in messages:
    if msg.role == "assistant":
        print(msg.content[0].text.value)
ℹ️ Agent vs. Chat Completion

Unlike a simple chat completion call, agents maintain state across turns, autonomously plan tool usage, and handle complex multi-step reasoning. This makes them ideal for scenarios requiring context continuity, action execution, and grounded responses.

Python — FastAPI Agent Endpoint
from fastapi import FastAPI, HTTPException
from pydantic import BaseModel

app = FastAPI(title="AI Agent API", version="1.0")

class ChatRequest(BaseModel):
    message: str
    session_id: str | None = None

class ChatResponse(BaseModel):
    reply: str
    session_id: str
    citations: list[dict] = []

@app.post("/api/chat", response_model=ChatResponse)
async def chat(request: ChatRequest):
    # Create or retrieve thread
    thread_id = request.session_id or client.agents.create_thread().id

    # Send message and process
    client.agents.create_message(
        thread_id=thread_id, role="user", content=request.message
    )
    run = client.agents.create_and_process_run(
        thread_id=thread_id, agent_id=agent.id
    )

    # Extract response
    msgs = client.agents.list_messages(thread_id=thread_id)
    reply = msgs.data[0].content[0].text.value
    return ChatResponse(reply=reply, session_id=thread_id)

4. Teams & M365 Integration

Deploying your AI agent into Microsoft Teams brings intelligence directly into the workspace where teams collaborate. Users can interact with your agent through chat, adaptive cards, and message extensions — no context switching required.

4.2 Deploying via Copilot Studio to Teams

The simplest path to Teams is through Copilot Studio. Once your agent is connected via a custom connector (Section 3), you can publish directly to Teams:

  1. In Copilot Studio, go to Channels → Microsoft Teams
  2. Configure the bot name, icon, and description
  3. Select the Availability scope (specific users, security group, or entire tenant)
  4. Click Publish — your agent appears in the Teams app catalog
  5. Users install it from the Teams app store and begin chatting

4.3 Teams AI Library Integration

For deeper control, use the Teams AI Library to build a native Teams bot that routes messages to your Azure AI Foundry agent: 

TypeScript — Teams Bot with Foundry Agent
import { Application, TurnState } from '@microsoft/teams-ai';
import { MemoryStorage } from 'botbuilder';

const app = new Application<TurnState>({
    storage: new MemoryStorage(),
});

// Route all messages to the Foundry agent
app.activity('message', async (context, state) => {
    const userMessage = context.activity.text;
    const sessionId = context.activity.conversation.id;

    // Call your Foundry agent API
    const response = await fetch('https://your-agent-api.azurewebsites.net/api/chat', {
        method: 'POST',
        headers: { 'Content-Type': 'application/json' },
        body: JSON.stringify({ message: userMessage, session_id: sessionId }),
    });

    const data = await response.json();
    await context.sendActivity(data.reply);
});
⚠️ Authentication in Teams

When deploying agents to Teams, always use Entra ID SSO for authentication. The Teams platform provides a seamless single sign-on experience. Configure your agent's API to validate the Teams token and map it to your application's identity model. Never expose unauthenticated agent endpoints.

5.2 Azure Functions Agent Wrapper

Wrap your agent invocation in an Azure Function for serverless scaling and easy integration with API Management: 

Python — Azure Function (HTTP Trigger)
import azure.functions as func
import json, logging
from agent_service import invoke_agent

app = func.FunctionApp()

@app.route(route="agent/chat", methods=["POST"], auth_level=func.AuthLevel.FUNCTION)
async def agent_chat(req: func.HttpRequest) -> func.HttpResponse:
    try:
        body = req.get_json()
        message = body.get("message")
        session_id = body.get("session_id")

        if not message:
            return func.HttpResponse(
                json.dumps({"error": "'message' is required"}),
                status_code=400,
            )

        result = await invoke_agent(message, session_id)

        return func.HttpResponse(
            json.dumps(result),
            mimetype="application/json",
        )
    except Exception as e:
        logging.error(f"Agent error: {e}")
        return func.HttpResponse(
            json.dumps({"error": "Internal server error"}),
            status_code=500,
        )

5.3 API Management Configuration

Configure APIM to protect, rate-limit, and version your agent API: 

Bicep — APIM Agent API
resource agentApi 'Microsoft.ApiManagement/service/apis@2023-09-01-preview' = {
  name: 'ai-agent-api'
  parent: apimService
  properties: {
    displayName: 'AI Agent API'
    path: 'agent'
    protocols: ['https']
    subscriptionRequired: true
    serviceUrl: 'https://your-function-app.azurewebsites.net/api'
    apiVersion: 'v1'
  }
}

resource rateLimit 'Microsoft.ApiManagement/service/policies@2023-09-01-preview' = {
  parent: agentApi
  properties: {
    value: '<policies>
  <inbound>
    <rate-limit calls="100" renewal-period="60" />
    <validate-jwt header-name="Authorization">
      <openid-config url="https://login.microsoftonline.com/{tenant}/.well-known/openid-configuration" />
    </validate-jwt>
  </inbound>
</policies>'
  }
}
✅ API Design Best Practices

Versioning: Use URL path versioning (/v1/agent/chat).
Idempotency: Include a request ID header for retry-safe operations.
Pagination: For conversation history endpoints, use cursor-based pagination.
Health check: Expose /health and /ready endpoints for monitoring.

6. End-to-End Testing Strategies

End-to-end (E2E) testing validates that your entire agent pipeline — from user input through agent reasoning, tool calls, RAG retrieval, and response generation — works correctly as an integrated system. This is essential for production confidence.

6.1 Testing Pyramid for AI Agents

🔬

Unit Tests

Test individual functions, tool handlers, and prompt templates in isolation.

🔗

Integration Tests

Validate agent + RAG pipeline, agent + tools, and API endpoint contracts.

🌐

E2E Tests

Full conversation flows through deployed endpoints with real data.

📈

Evaluation

LLM-as-judge quality scoring for groundedness, relevance, and coherence.

6.2 Automated E2E Test Suite

Python — pytest E2E Test Suite
import pytest
import httpx

BASE_URL = "https://your-agent-api.azurewebsites.net"

class TestAgentE2E:
    """End-to-end tests for the deployed AI agent."""

    @pytest.fixture
    def client(self):
        return httpx.Client(base_url=BASE_URL, timeout=30)

    def test_health_check(self, client):
        response = client.get("/health")
        assert response.status_code == 200
        assert response.json()["status"] == "healthy"

    def test_simple_query(self, client):
        response = client.post("/api/chat", json={"message": "What is your return policy?"})
        assert response.status_code == 200
        data = response.json()
        assert "reply" in data
        assert len(data["reply"]) > 20  # Non-trivial response

    def test_session_continuity(self, client):
        # First turn
        r1 = client.post("/api/chat", json={"message": "I need help with product X."})
        session_id = r1.json()["session_id"]

        # Second turn (same session)
        r2 = client.post("/api/chat", json={
            "message": "What about its warranty?",
            "session_id": session_id,
        })
        assert r2.status_code == 200
        # Agent should recall context about product X

    def test_tool_invocation(self, client):
        response = client.post("/api/chat", json={
            "message": "Check order status for ORD-1234"
        })
        data = response.json()
        assert "ORD-1234" in data["reply"]

    def test_error_handling(self, client):
        response = client.post("/api/chat", json={})
        assert response.status_code == 400

6.3 AI Quality Evaluation with Azure AI Foundry

Use the Azure AI Evaluation SDK to score your agent's responses on quality metrics like groundedness, relevance, coherence, and fluency: 

Python — AI Evaluation
from azure.ai.evaluation import (
    GroundednessEvaluator,
    RelevanceEvaluator,
    CoherenceEvaluator,
    evaluate,
)

# Define test dataset
test_data = [
    {
        "query": "What is the return policy?",
        "context": "Our return policy allows returns within 30 days...",
        "response": agent_response,
    },
]

# Run evaluation
results = evaluate(
    data=test_data,
    evaluators={
        "groundedness": GroundednessEvaluator(model_config),
        "relevance": RelevanceEvaluator(model_config),
        "coherence": CoherenceEvaluator(model_config),
    },
)
print(results.to_dict())

🧪 Mini Hack: Deploy Your Integrated Solution

Put everything together! In this capstone mini hack, you will deploy a fully integrated AI agent solution that combines RAG, custom tools, Copilot Studio, Teams, and external API access — all validated with end-to-end tests.

  1. Build the Agent: Create an Azure AI Foundry agent with your RAG index and at least two custom function tools (e.g., ticket creation, status lookup).

  8. Document & Demo: Record a 3-minute video demonstrating your integrated solution across all channels (API, Teams, Copilot Studio).
