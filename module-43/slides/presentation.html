Why AI Agents?

Chat Completions

  • Stateless — no memory across calls
  • Single request/response pattern
  • Manual tool orchestration
  • No planning or reasoning loops

AI Agents

  • Stateful — persistent conversation threads
  • Autonomous multi-step reasoning
  • Automatic tool selection & invocation
  • Planning, reflection, and self-correction
Key Insight: Agents transform your RAG pipeline from a search engine into an intelligent assistant that can reason, act, and learn from conversation context.

Creating an Agent

from azure.ai.projects import AIProjectClient
from azure.ai.projects.models import AzureAISearchTool
from azure.identity import DefaultAzureCredential

client = AIProjectClient(
    credential=DefaultAzureCredential(),
    endpoint="https://my-project.services.ai.azure.com",
)

search_tool = AzureAISearchTool(
    index_connection_id="<connection-id>",
    index_name="product-kb",
    query_type="semantic",
    top_k=5,
)

agent = client.agents.create_agent(
    model="GPT-5.1",
    name="support-agent",
    instructions="You are a product support specialist...",
    tools=[search_tool],
)

Custom Function Tools

Extend your agent beyond retrieval with actionable function tools:

from azure.ai.projects.models import FunctionTool, ToolSet

def create_ticket(title: str, priority: str) -> dict:
    """Create a support ticket."""
    return {"ticket_id": "TKT-001", "status": "created"}

def check_order(order_id: str) -> dict:
    """Look up order status."""
    return {"order_id": order_id, "status": "shipped"}

toolset = ToolSet()
toolset.add(FunctionTool([create_ticket, check_order]))
toolset.add(search_tool)
Tip: Always include detailed docstrings — the agent uses them to decide when and how to call each function.

FastAPI Agent Endpoint

from fastapi import FastAPI
from pydantic import BaseModel

app = FastAPI(title="AI Agent API")

class ChatRequest(BaseModel):
    message: str
    session_id: str | None = None

@app.post("/api/chat")
async def chat(req: ChatRequest):
    thread_id = req.session_id or create_thread()
    result = await invoke_agent(req.message, thread_id)
    return {"reply": result, "session_id": thread_id}

This endpoint serves as the bridge between Copilot Studio, Teams, and external consumers.

End-to-End Testing

Testing Pyramid

🔬 Unit Tests

Individual functions, tool handlers, prompt templates

🔗 Integration Tests

Agent + RAG pipeline, agent + tools, API contracts

🌐 E2E Tests

Full conversation flows through deployed endpoints

📈 Evaluations

LLM-as-judge scoring for quality metrics

Key Test Cases

  • Health check endpoint returns 200
  • Simple query returns non-empty response
  • Multi-turn conversation maintains context
  • Tool invocation produces correct action
  • Invalid input returns 400 error
  • Session ID is returned and reusable
  • Response time under SLA threshold
