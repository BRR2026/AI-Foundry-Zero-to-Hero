Minimum Viable Capstone

Start Small, Iterate Fast

Define the smallest end-to-end slice that demonstrates real value:

  • One agent with system prompt and ≥1 tool integration
  • One data source indexed in Azure AI Search
  • One evaluation run with baseline metrics captured
  • Content safety filter enabled on inputs and outputs
  • One conversation flow demonstrated end-to-end

💡 Lock your MVC scope before Module 42 begins. Additional features go to a "future work" backlog.

Requirements Gathering

Functional & Non-Functional Requirements

Functional

  • User stories (As a… I want… So that…)
  • Core interactions & queries
  • Tool integrations & APIs
  • Multi-turn & session memory
  • Output formats (text, JSON, charts)

Non-Functional

  • Latency: <3s response time
  • Throughput: concurrent users
  • Security: managed identity, RBAC
  • Compliance: PII handling, data residency
  • Cost budget: monthly Azure spend

Module 41: Choose Your Capstone Project 10 / 14

Evaluation Plan

Four-Phase Assessment

Phase 1: Baseline (M42)

Create test dataset (50+ cases), run initial batch evaluation, capture baseline scores.

Phase 2: Iteration (M43)

Optimize prompts, tune retrieval, refine tools. Target ≥15% improvement over baseline.

Phase 3: Final Eval (M44)

Comprehensive evaluation with edge cases, adversarial inputs, and RAI red-teaming.

Phase 4: Presentation

Live demo to peers with Q&A. Rubric: technical depth, UX, RAI, and presentation.

Mini Hack

🧪 Write Your Project Proposal

  1. Choose your project (15 min)
  2. Gather requirements (20 min)
    — 5–8 user stories + NFRs
  3. Design architecture (25 min)
    — Diagram + pattern selection
  4. Define success criteria (15 min)
    — 3–5 SMART goals
  5. Write PROPOSAL.md (15 min)
    — All sections, no placeholders

Deliverables Checklist

  • ☐ Project selected & validated
  • ☐ 5+ user stories written
  • ☐ Architecture diagram drawn
  • ☐ 3+ SMART success criteria
  • ☐ PROPOSAL.md completed
  • ☐ Risk table with 3+ risks
