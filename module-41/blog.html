Module 41: Choose Your Capstone Project

Define, scope, and architect your capstone project — the culminating experience that brings together every AI Foundry skill you've built across 40 modules. From idea to proposal, set yourself up for a masterful build.

📅 April 2026 ⏱ 90 min read 🧩 Module 41 of 45 🎓 Capstone Phase

Introduction

You've spent 40 modules mastering every layer of Azure AI Foundry — from model deployments and prompt engineering to multi-agent orchestration, responsible AI, and enterprise-grade operations. Now it's time to bring it all together in a capstone project that demonstrates your end-to-end mastery.

This module guides you through choosing, scoping, and planning your capstone. You'll select a project that aligns with real-world needs, define success criteria, plan your architecture using AI Foundry services, and produce a formal proposal that will drive your implementation in Modules 42–44.

🎓 Capstone Philosophy

Your capstone should demonstrate breadth across the curriculum — touching at minimum model deployment, prompt engineering, agent orchestration, data integration, responsible AI, and operational monitoring. Projects that tackle a genuine business or community problem will stand out.

2. Requirements Gathering & Architecture Planning

A strong capstone begins with rigorous requirements. Use the structured approach below to capture functional needs, non-functional constraints, and architectural decisions.

Requirements Gathering Framework

Functional Requirements

  • User stories: Who is the user? What do they need to accomplish? What outcome do they expect?
  • Core interactions: What conversations, queries, or workflows does the system support?
  • Tool integrations: What external systems, APIs, or data sources must the agent access?
  • Multi-turn capabilities: Does the system need session memory, context carryover, or thread management?
  • Output formats: Text responses, structured JSON, charts, documents, or actions?

Non-Functional Requirements

  • Latency: Expected response time for user-facing interactions (e.g., <3 seconds)
  • Throughput: Concurrent users or requests the system must handle
  • Security: Authentication, authorization, data encryption requirements
  • Compliance: Data residency, PII handling, industry regulations
  • Cost budget: Target monthly Azure spend for the capstone workload

Key Architectural Decisions

Document these decisions in your proposal — they'll guide your implementation:

  1. Agent topology: Single agent, multi-agent pipeline, or hierarchical agent swarm?
  2. Knowledge strategy: RAG with Azure AI Search, fine-tuned model, or hybrid?
  3. Orchestration pattern: Sequential chain, parallel fan-out, or event-driven?
  4. Deployment model: Serverless (AI Foundry hosted), containerized, or hybrid?
  5. Evaluation approach: Online metrics, batch evaluation, human-in-the-loop, or all three?

3. Success Criteria & Evaluation Plan

Define how you'll know your capstone succeeded. Strong success criteria are measurable, time-bound, and tied to user outcomes.

The SMART Criteria Framework

Dimension Example for Capstone
Specific "The agent correctly answers 90% of test queries from the evaluation dataset"
Measurable "Groundedness score ≥ 4.0/5.0 on the AI Foundry evaluation pipeline"
Achievable "Achievable within 3 module build sprints with available Azure credits"
Relevant "Solves a real workflow bottleneck for the target user persona"
Time-bound "MVP functional by end of Module 43, polished by Module 44"

Evaluation Plan Components

Phase 1 — Baseline Metrics (Module 42)

Establish baseline performance using a curated test dataset of 50+ test cases. Run AI Foundry batch evaluation to capture groundedness, relevance, coherence, and fluency scores.

Phase 2 — Iterative Improvement (Module 43)

Optimize prompts, tune retrieval parameters, and refine agent tools. Track metric improvements per iteration. Target: ≥15% improvement over baseline.

Phase 3 — Final Evaluation (Module 44)

Run comprehensive evaluation including edge cases, adversarial inputs, and responsible AI red-teaming. Produce final evaluation report with comparison to baseline.

Phase 4 — Presentation & Demo

Live demo to peers with Q&A. Evaluation rubric covers technical depth, user experience, responsible AI practices, and presentation quality.

Python evaluation_plan.py — Automated evaluation scaffold 
from azure.ai.evaluation import EvaluatorConfig, BatchEvaluator
from azure.ai.projects import AIProjectClient

# Configure the evaluation pipeline for your capstone
def create_evaluation_plan(project_client, agent_id, dataset_path):
    """Create a structured evaluation plan for capstone assessment."""

    evaluators = {
        "groundedness": EvaluatorConfig(
            id="azureai://groundedness",
            init_params={"threshold": 4.0}
        ),
        "relevance": EvaluatorConfig(
            id="azureai://relevance",
            init_params={"threshold": 4.0}
        ),
        "coherence": EvaluatorConfig(
            id="azureai://coherence",
            init_params={"threshold": 3.5}
        ),
        "content_safety": EvaluatorConfig(
            id="azureai://content-safety",
            init_params={"severity_threshold": "medium"}
        ),
    }

    # Run batch evaluation
    batch_eval = BatchEvaluator(
        project_client=project_client,
        evaluators=evaluators,
        agent_id=agent_id,
        dataset_path=dataset_path,
        output_path="./eval-results/",
    )

    results = batch_eval.run()

    # Print summary
    print("=== Capstone Evaluation Results ===")
    for metric, score in results.metrics.items():
        status = "✅ PASS" if score >= 4.0 else "⚠️ NEEDS WORK"
        print(f"  {metric}: {score:.2f} {status}")

    return results

Starter Agent Code

Python src/agent.py — Capstone agent starter 
import os
from azure.identity import DefaultAzureCredential
from azure.ai.projects import AIProjectClient
from azure.ai.agents.models import (
    AzureAISearchTool,
    FunctionTool,
    CodeInterpreterTool,
)
from src.tools import tool_functions

def create_capstone_agent():
    """Initialize and configure the capstone AI Foundry agent."""

    # Connect to your AI Foundry project
    client = AIProjectClient(
        credential=DefaultAzureCredential(),
        endpoint=os.environ["AZURE_AI_FOUNDRY_ENDPOINT"],
    )

    # Configure tools
    search_tool = AzureAISearchTool(
        index_connection_id=os.environ["SEARCH_CONNECTION_ID"],
        index_name="capstone-index",
    )
    code_tool = CodeInterpreterTool()
    custom_tools = FunctionTool(functions=tool_functions)

    # Create the agent
    agent = client.agents.create_agent(
        model="gpt-4o",
        name="capstone-agent",
        instructions=load_system_prompt(),
        tools=[search_tool, code_tool, custom_tools],
    )

    print(f"✅ Agent created: {agent.id}")
    return client, agent


def load_system_prompt():
    """Load the system prompt from prompts.py or a file."""
    return """
You are an AI assistant for [YOUR DOMAIN].

## Core Responsibilities
- Answer user questions accurately using the knowledge base
- Perform actions through available tools when requested
- Maintain conversation context across multi-turn dialogues
- Flag uncertain or out-of-scope queries for human review

## Guardrails
- Never fabricate information not in the knowledge base
- Always cite sources when providing factual answers
- Decline requests that violate content safety policies
- Escalate sensitive topics to human operators
"""


if __name__ == "__main__":
    client, agent = create_capstone_agent()

5. Capstone Architecture Patterns

Choose the architecture pattern that best matches your project's complexity and requirements. Each pattern builds on skills you've developed throughout the curriculum.

Pattern A: Single-Agent RAG Application

Best for: Knowledge-intensive Q&A, document search, customer support

  • One AI Foundry agent with Azure AI Search tool
  • Grounded responses from indexed documents
  • Content safety layer for input/output filtering
  • Batch evaluation for quality assurance

Pattern B: Multi-Agent Orchestration

Best for: Complex workflows, task decomposition, domain-specific expertise

  • Orchestrator agent that delegates to specialist agents
  • Each specialist handles a specific domain or capability
  • Shared context through thread management
  • Inter-agent evaluation and handoff protocols

Pattern C: Event-Driven AI Pipeline

Best for: Document processing, monitoring, automated workflows

  • Event triggers (file upload, schedule, API call) initiate agent workflows
  • Azure Functions or Container Apps as compute substrate
  • Asynchronous processing with status tracking
  • End-to-end tracing with Azure Monitor
Python Multi-agent orchestration pattern 
async def orchestrate_capstone(user_query: str, client):
    """Route queries to specialized agents based on intent."""

    # Step 1: Classify intent
    intent = await classify_intent(user_query, client)

    # Step 2: Route to specialist agent
    agent_map = {
        "knowledge_query": rag_agent_id,
        "data_analysis":   analytics_agent_id,
        "action_request":  action_agent_id,
    }

    target_agent = agent_map.get(intent, rag_agent_id)

    # Step 3: Execute with target agent
    thread = client.agents.threads.create()
    client.agents.messages.create(
        thread_id=thread.id,
        role="user",
        content=user_query,
    )

    run = client.agents.runs.create_and_process(
        agent_id=target_agent,
        thread_id=thread.id,
    )

    # Step 4: Return result with metadata
    messages = client.agents.messages.list(thread_id=thread.id)
    return {
        "response": messages.data[0].content[0].text.value,
        "agent_used": target_agent,
        "intent": intent,
    }

7. Mini Hack: Write Your Project Proposal & Architecture Diagram

🧪 Mini Hack Challenge

In this hands-on challenge, you'll produce the two foundational artifacts for your capstone: a written proposal and an architecture diagram. These will be reviewed and refined before you begin building in Module 42.

Step 1: Choose Your Project (15 min)

Select from the curated ideas above or define your own. Validate it against the Scope Fitness Test table. Document your choice and rationale.

Step 2: Gather Requirements (20 min)

  • Write 5–8 user stories in "As a [role], I want [goal], so that [benefit]" format
  • List 3+ non-functional requirements (latency, security, cost)
  • Identify 2+ external data sources or APIs your agent will use

Step 3: Design Your Architecture (25 min)

  • Draw an architecture diagram showing all AI Foundry components
  • Label data flows between components with arrows
  • Identify which architecture pattern (A, B, or C) you're following
  • Document 3 key architectural decisions and their rationale

Step 4: Define Success Criteria (15 min)

  • Write 3–5 SMART success criteria for your capstone
  • Define your evaluation dataset strategy (how many test cases, categories)
  • Set target scores for at least 3 AI Foundry evaluation metrics

Step 5: Write the Proposal (15 min)

  • Use the PROPOSAL.md template provided in Section 4
  • Fill in every section — leave no placeholders
  • Include your architecture diagram (embedded or as a link)
  • Add a risk mitigation table with at least 3 identified risks

  • 5+ user stories documented
  • Architecture diagram with all AI Foundry components labeled
  • 3+ SMART success criteria defined

8. Best Practices for Capstone Success

Do's

  • Start with the evaluation dataset — building test cases first clarifies requirements and prevents scope creep
  • Use AI Foundry's built-in evaluators — they provide standardized metrics that demonstrate proficiency
  • Version everything — prompts, agent configs, evaluation results, and infrastructure code belong in source control
  • Build incrementally — get a single end-to-end flow working before adding complexity
  • Document decisions — your architecture decision records (ADRs) are as valuable as the code

Don'ts

  • Don't boil the ocean — scope to what's achievable in 3 modules, not a production startup
  • Don't skip responsible AI — content safety and bias evaluation are required, not optional
  • Don't hardcode secrets — use Azure Key Vault and managed identities from day one
  • Don't ignore costs — set Azure budget alerts and choose model deployments wisely
  • Don't work in isolation — peer feedback during the proposal phase saves rework later
⚠️ Common Pitfall

The #1 reason capstones stall is scope creep. Lock your MVC scope before Module 42 begins. Any feature not in your approved proposal goes to a "future work" backlog — no exceptions.

9. Key Takeaways

  • Your capstone is the culminating demonstration of all 40 modules of AI Foundry mastery
  • Choose a project that demonstrates breadth (multiple services) and depth (real-world complexity)
  • Rigorous requirements gathering and architecture planning prevent costly rework during build sprints
  • SMART success criteria with measurable evaluation metrics ensure objective quality assessment
  • Use the provided templates and starter code to accelerate your start — don't build scaffolding from scratch
  • The evaluation rubric is your guide: technical depth, agent design, evaluation quality, responsible AI, and presentation
  • Start with your Minimum Viable Capstone (MVC) and iterate — perfection is the enemy of completion
