Introduction: When Prompting Isn't Enough

Throughout ARC 2, we've explored how careful prompt engineering can dramatically improve model outputs. We've crafted system messages, used few-shot examples, and tuned parameters. But sometimes — no matter how sophisticated your prompt — you need the model itself to internalize your domain knowledge, terminology, and output patterns.

That's where fine-tuning comes in. Fine-tuning takes a pre-trained foundation model and trains it further on your specific data, adjusting the model's weights so that it naturally produces outputs aligned with your requirements — without needing lengthy system prompts or dozens of in-context examples.

ℹ️ What You'll Learn

By the end of this module, you'll understand when fine-tuning is the right choice, how to prepare JSONL training data, execute the fine-tuning workflow in Azure AI Foundry, evaluate your custom model's performance, and deploy it to production.

Fine-tuning is ideal when you need to:

  • Reduce token usage — eliminate long system prompts and few-shot examples
  • Improve consistency — enforce a specific output format, style, or tone
  • Encode domain expertise — teach the model your company's terminology and processes
  • Decrease latency — shorter prompts mean faster responses
  • Improve reliability — the model "knows" the patterns rather than inferring them from context

The Decision Framework: Prompt Engineering vs. RAG vs. Fine-Tuning

One of the most common questions is: "When should I fine-tune versus improve my prompts or add RAG?" Each approach solves a different problem, and they aren't mutually exclusive — you can combine them.

🎯 Prompt Engineering

  • Quick iterations, no training
  • Adjusting behavior/format
  • Small task variations
  • Zero additional cost
  • Best for: Getting started, prototyping, general tasks

📚 RAG (Retrieval-Augmented)

  • Dynamic/changing data
  • Need source citations
  • Large knowledge bases
  • Data freshness matters
  • Best for: Grounding answers in current facts

🔧 Fine-Tuning

  • Specific style/format/tone
  • Domain-specific expertise
  • Reduce prompt size/latency
  • Consistent, repeatable output
  • Best for: Specialized tasks at scale
💡 Pro Tip

Fine-tuning + RAG is a powerful combination. Fine-tune for output style and domain terminology, then use RAG to ground responses in the latest data. This gives you the best of both worlds: a model that "speaks your language" while staying factually current.

Decision Flowchart

Ask yourself these questions in order:

  1. Can I solve this with a better prompt? → Start with prompt engineering. It's free and fast.
  2. Does the model need access to specific/changing data? → Add RAG for grounded, up-to-date answers.
  3. Do I need a specific output style, format, or domain expertise that prompts can't reliably achieve? → Fine-tune the model.
  4. Do I need all of the above? → Fine-tune for style + RAG for data = production-grade solution.

Preparing Training Data

The quality of your fine-tuned model is directly proportional to the quality of your training data. AI Foundry uses the JSONL (JSON Lines) format, where each line is a self-contained JSON object representing one training example.

JSONL Format for Chat Models

Each training example follows the chat completions format with a list of messages:

training_data.jsonl
{"messages": [{"role": "system", "content": "You are a helpful medical coding assistant that maps clinical descriptions to ICD-10 codes."}, {"role": "user", "content": "Patient presents with acute bronchitis."}, {"role": "assistant", "content": "J20.9 — Acute bronchitis, unspecified"}]}
{"messages": [{"role": "system", "content": "You are a helpful medical coding assistant that maps clinical descriptions to ICD-10 codes."}, {"role": "user", "content": "Diagnosis: Type 2 diabetes mellitus with diabetic chronic kidney disease."}, {"role": "assistant", "content": "E11.22 — Type 2 diabetes mellitus with diabetic chronic kidney disease"}]}
{"messages": [{"role": "system", "content": "You are a helpful medical coding assistant that maps clinical descriptions to ICD-10 codes."}, {"role": "user", "content": "The patient has essential hypertension, stage 2."}, {"role": "assistant", "content": "I10 — Essential (primary) hypertension"}]}

Data Quality Guidelines

✅ Data Quality Checklist
  • Minimum 50 examples — AI Foundry requires at least 10, but 50–100+ gives much better results
  • Consistent format — every example should follow the same structure
  • Representative diversity — cover edge cases, variations, and the full range of expected inputs
  • High-quality outputs — the assistant responses are what the model learns; they must be correct
  • Consistent system messages — use the same system message across all examples (or omit it)
  • Avoid duplicates — remove exact or near-duplicate entries

Recommended Dataset Sizes

Use Case Minimum Recommended Optimal
Style/tone transfer 50 200–500 1,000+
Classification tasks 100 500–1,000 5,000+
Domain Q&A 100 300–800 2,000+
Complex reasoning 200 1,000–3,000 10,000+

Preparing a Validation Set

Always split your data into training and validation sets (typically 80/20 or 90/10). The validation set should be representative of your training data but contain examples the model won't see during training. AI Foundry uses the validation set to compute loss metrics that help you detect overfitting.

⚠️ Common Pitfalls
  • Don't fine-tune with fabricated data — use real examples from your domain
  • Don't include sensitive/PII data — the model may memorize and reproduce it
  • Don't use too few examples — the model will overfit to your small sample
  • Don't mix conflicting instructions — be consistent in what you teach the model

Fine-Tuning Workflow in AI Foundry

Azure AI Foundry provides a streamlined, managed fine-tuning experience that abstracts away the complexities of infrastructure, GPU management, and distributed training. Here's the step-by-step workflow:

1

Open AI Foundry & Select Your Project

Navigate to ai.azure.com and open the project where you want to create your fine-tuned model. Ensure you have a connected Azure OpenAI resource in a region that supports fine-tuning.

2

Navigate to Fine-Tuning

In the left sidebar, click Fine-tuning under the Build & customize section. Click "+ Fine-tune a model" to start a new fine-tuning job.

3

Select the Base Model

Choose from supported models (e.g., GPT-5-mini, GPT-5.1, gpt-4.1-mini). The model catalog shows which models support fine-tuning and in which regions.

4

Upload Training & Validation Data

Upload your JSONL files. AI Foundry validates the format automatically — you'll see a preview of your data and any errors that need fixing. Specify which file is training data and which is validation data.

5

Configure Hyperparameters

Set the number of epochs, learning rate multiplier, and batch size. AI Foundry provides sensible defaults, but you can adjust these based on your dataset size and requirements.

6

Start Training

Click "Submit" to begin the fine-tuning job. Training runs on managed GPU infrastructure — you don't provision or manage any compute. Training time depends on dataset size and model.

7

Monitor & Deploy

Track training progress with real-time loss curves. Once complete, review metrics and deploy your custom model to an endpoint for inference.

Hyperparameter Configuration

Fine-tuning performance depends heavily on hyperparameter selection. AI Foundry provides reasonable defaults, but understanding these parameters helps you optimize results.

n_epochs
Default: auto (typically 3–5)
Number of complete passes through the training dataset. More epochs = more learning, but risk of overfitting.
learning_rate_multiplier
Default: auto (typically 1.0–2.0)
Scales the base learning rate. Higher values learn faster but may be unstable. Lower values are safer for small datasets.
batch_size
Default: auto (typically 1–4)
Number of examples processed together. Larger batches provide more stable gradients but use more memory.
seed
Default: random
Controls reproducibility. Set a fixed seed to get consistent results across training runs for comparison purposes.

Tuning Recommendations

Dataset Size Recommended Epochs Learning Rate Notes
50–100 examples 4–6 0.5–1.0× Watch for overfitting; use validation set
100–500 examples 3–4 1.0× Good balance for most tasks
500–5,000 examples 2–3 1.0–2.0× Reduce epochs as dataset grows
5,000+ examples 1–2 1.0–2.0× Large datasets need fewer passes
💡 Start with Defaults

AI Foundry's auto settings are well-calibrated for most workloads. Start with defaults, evaluate, then adjust one parameter at a time. This controlled approach makes it easier to understand what's improving (or degrading) performance.

Monitoring Training Progress

During training, AI Foundry provides real-time metrics that help you understand how well your model is learning.

Key Metrics to Watch

  • Training Loss — measures how well the model fits the training data. Should decrease steadily and plateau.
  • Validation Loss — measures performance on held-out data. Should track with training loss.
  • Token Accuracy — percentage of tokens correctly predicted. Higher is better, but 100% indicates overfitting.

Reading the Loss Curves

Pattern Training Loss Validation Loss Diagnosis Action
✅ Healthy ↓ Decreasing ↓ Decreasing Model is learning well Continue training
⚠️ Overfitting ↓ Decreasing ↑ Increasing Model memorizing training data Reduce epochs, add more data
⚠️ Underfitting → Flat / High → Flat / High Model not learning enough Increase epochs, raise learning rate
✅ Converged → Plateaued → Plateaued Model fully trained Training complete

Monitoring in AI Foundry

In the AI Foundry portal, navigate to your fine-tuning job to see:

  • Status panel — shows current training step, estimated completion time
  • Metrics charts — interactive loss and accuracy curves updated in real time
  • Training events — log of checkpoints, warnings, and completion events
  • Result files — downloadable training results and model artifacts after completion

Validating Your Fine-Tuned Model

After training completes, it's critical to validate your model before deploying it to production. AI Foundry provides several ways to evaluate performance.

Step 1: Review Training Metrics

Check the final training and validation loss. Low, converging values indicate the model learned effectively from your data. Compare the final validation loss to the initial loss — a significant decrease means the model improved.

Step 2: Playground Testing

Once your model is deployed (even to a test endpoint), use the AI Foundry Playground to run interactive tests:

  • Test with examples from your training data — the model should handle these well
  • Test with new, unseen examples — this reveals true generalization ability
  • Test edge cases and adversarial inputs
  • Compare side-by-side with the base model to see improvements

Step 3: Quantitative Evaluation

For rigorous evaluation, run your fine-tuned model against a held-out test set and compute relevant metrics:

Python — Batch Evaluation
from openai import AzureOpenAI
import json

client = AzureOpenAI(
    azure_endpoint="https://your-resource.openai.azure.com/",
    api_key="your-api-key",
    api_version="2024-12-01-preview"
)

# Load test set
with open("test_data.jsonl", "r") as f:
    test_examples = [json.loads(line) for line in f]

correct = 0
total = len(test_examples)

for example in test_examples:
    messages = example["messages"][:-1]  # All but the expected answer
    expected = example["messages"][-1]["content"]

    response = client.chat.completions.create(
        model="your-fine-tuned-deployment-name",
        messages=messages,
        temperature=0
    )

    predicted = response.choices[0].message.content
    if predicted.strip() == expected.strip():
        correct += 1

accuracy = correct / total * 100
print(f"Accuracy: {accuracy:.1f}% ({correct}/{total})")
✅ Validation Best Practice

Always compare your fine-tuned model against the base model using the same test set and the same evaluation criteria. This gives you a clear picture of what fine-tuning improved and whether the investment was worthwhile.

Deploying Your Custom Model

Once validated, deploy your fine-tuned model directly from AI Foundry to make it available via the API.

Deployment Steps

  1. In your AI Foundry project, go to Deployments (under Manage)
  2. Click "+ Create deployment"
  3. Select your fine-tuned model from the Custom fine-tuned models tab
  4. Choose a deployment name (this becomes the model parameter in API calls)
  5. Set the Tokens Per Minute (TPM) rate limit
  6. Click Create to deploy

Using Your Deployed Model

Python — Calling Your Fine-Tuned Model
from openai import AzureOpenAI

client = AzureOpenAI(
    azure_endpoint="https://your-resource.openai.azure.com/",
    api_key="your-api-key",
    api_version="2024-12-01-preview"
)

response = client.chat.completions.create(
    model="my-custom-medical-coder",  # Your deployment name
    messages=[
        {"role": "user", "content": "Patient diagnosed with acute appendicitis."}
    ],
    temperature=0
)

print(response.choices[0].message.content)
# Output: K35.80 — Unspecified acute appendicitis without abscess
ℹ️ Shorter Prompts

Notice that with a fine-tuned model, you often don't need a detailed system message. The model has already learned the expected behavior from your training data. This reduces token usage and latency in production.

Cost & Performance Considerations

Fine-tuning involves both training costs and ongoing inference costs. Understanding the economics helps you decide when fine-tuning makes financial sense.

Cost Components

Component When Charged Factors
Training One-time per training job Model size × tokens in training data × epochs
Hosting Per hour while deployed Model size, deployment type
Inference Per request (tokens) Input + output tokens per call

When Fine-Tuning Saves Money

  • High-volume workloads — if you're sending long system prompts with every request, fine-tuning can reduce per-request token costs significantly
  • Model downsizing — fine-tuning a smaller model (e.g., GPT-5-mini) to match the quality of a larger model on your specific task
  • Latency reduction — shorter prompts = faster responses = better user experience and lower compute costs

Performance Benefits

  • Reduced latency — up to 50% faster responses by eliminating few-shot examples from prompts
  • Higher accuracy — on domain-specific tasks, fine-tuned models often outperform prompted base models by 10–30%
  • Better consistency — fine-tuned models produce more uniform outputs across similar inputs
  • Token efficiency — 60–80% reduction in prompt tokens when system instructions are "baked in"
💡 Cost Optimization Strategy

Start by fine-tuning GPT-5-mini rather than GPT-5.1. The smaller model is significantly cheaper to train and host, and for many specialized tasks, a fine-tuned mini model can match or exceed the performance of the larger base model. Only upgrade to GPT-5.1 if the mini model doesn't meet your quality bar.

Mini Hack: Fine-Tune GPT-5-mini on a Custom Q&A Dataset

🧪 Hands-On Challenge

In this Mini Hack, you'll fine-tune GPT-5-mini on a custom question-and-answer dataset to create a domain-specific assistant.

Objective

Create a fine-tuned model that answers product support questions for a fictional SaaS company "CloudFlow" with accurate, branded responses.

Steps

  1. Create Training Data — write 50+ Q&A pairs in JSONL chat format covering CloudFlow product features, troubleshooting, pricing, and setup
  2. Create Validation Data — set aside 10–15 examples that aren't in the training set
  3. Upload to AI Foundry — navigate to Fine-tuning, select GPT-5-mini as the base model, and upload both JSONL files
  4. Configure & Train — use default hyperparameters (auto epochs, auto learning rate), submit the job
  5. Monitor Progress — watch the loss curves; training should take approximately 15–30 minutes
  6. Deploy & Test — deploy the fine-tuned model and test in the Playground with new questions
  7. Evaluate — compare responses from your fine-tuned model vs. the base GPT-5-mini with a simple prompt

Sample Training Data

cloudflow_training.jsonl
{"messages": [{"role": "system", "content": "You are CloudFlow's AI support assistant. Answer customer questions accurately and helpfully."}, {"role": "user", "content": "How do I reset my CloudFlow password?"}, {"role": "assistant", "content": "To reset your CloudFlow password:\n1. Go to app.cloudflow.io/login\n2. Click 'Forgot Password?' below the sign-in button\n3. Enter your registered email address\n4. Check your inbox for a reset link (valid for 24 hours)\n5. Click the link and set your new password\n\nIf you don't receive the email, check your spam folder or contact support@cloudflow.io."}]}
{"messages": [{"role": "system", "content": "You are CloudFlow's AI support assistant. Answer customer questions accurately and helpfully."}, {"role": "user", "content": "What's included in the CloudFlow Pro plan?"}, {"role": "assistant", "content": "The CloudFlow Pro plan ($49/month per user) includes:\n• Unlimited workflows and automations\n• Up to 50 team members\n• Priority email and chat support\n• Advanced analytics dashboard\n• Custom integrations via REST API\n• 100 GB cloud storage\n• SSO with SAML 2.0\n• 99.9% uptime SLA\n\nYou can start a 14-day free trial at app.cloudflow.io/pricing."}]}

Success Criteria

  • Training completes with decreasing validation loss
  • Fine-tuned model provides CloudFlow-specific answers without needing a detailed system prompt
  • Responses are more accurate and consistent than the base model with a simple prompt
  • You can articulate what improved and why
ℹ️ Time & Cost Estimate

Training 50 examples on GPT-5-mini typically takes 15–30 minutes and costs under $1 in training charges. Deployment hosting costs apply while the model is active.

Key Takeaways

📝 What You've Learned

  • Fine-tuning customizes a model's weights to internalize your domain knowledge, style, and output patterns
  • Use the decision framework: start with prompts, add RAG for data, fine-tune for behavior and style
  • Training data must be in JSONL chat format with high-quality, diverse, and consistent examples
  • AI Foundry provides a fully managed fine-tuning experience — no GPU management needed
  • Monitor training/validation loss curves to detect overfitting, underfitting, and convergence
  • Always validate against a held-out test set and compare with the base model
  • Fine-tuning reduces prompt tokens, latency, and can improve accuracy by 10–30% on specialized tasks
  • Start with GPT-5-mini — it's cost-effective and often sufficient when fine-tuned
  • Combine fine-tuning + RAG for production systems that need both style consistency and data freshness
  • Deploy directly from AI Foundry and call via the standard Azure OpenAI API
