Azure SQL & Cosmos DB

🗃️

Azure SQL Database

Ideal for structured, relational data. Use SQL queries to extract specific datasets for indexing. Supports row-level filtering and joins for precise data selection.

SELECT title, content
FROM knowledge_articles
WHERE status = 'published'
🌍

Azure Cosmos DB

Best for semi-structured and document data. Native JSON support makes it ideal for flexible schemas. Use change feed for real-time ingestion patterns.

SELECT c.id, c.body
FROM c
WHERE c.type = "article"

📁

SharePoint Online

Connect to Microsoft 365 document libraries. Access enterprise content where teams already collaborate. Automatic metadata extraction from document properties.

  • Site-level or library-level connections
  • Respects M365 permissions
  • Built-in document versioning

Microsoft OneLake

Unified data layer for Microsoft Fabric. One copy of data accessible across all analytics workloads. Ideal for organizations already invested in Fabric.

  • Single data lake for all workloads
  • Shortcuts to external data
  • Built-in governance & security
Patterns

Data Ingestion Patterns

📥

Batch

Process large volumes of data at scheduled intervals. Best for static or infrequently updated content.

⚡

Real-time

Stream data as it arrives using event-driven architectures. Ideal for frequently changing data.

🔄

Incremental

Sync only new or modified data since last update. Balances freshness with processing efficiency.

Recommendation: Start with batch ingestion for your initial index, then add incremental refresh for production to keep data current without reprocessing everything.

Preparation

Data Preparation & Cleaning

  • Text Extraction — Parse PDFs, DOCX, HTML into clean text
  • Chunking — Split documents into 512–2048 token segments
  • Overlap — 10–15% overlap preserves context at boundaries
  • Metadata — Enrich chunks with source, date, author info
  • Deduplication — Remove redundant or near-duplicate content
// Chunking configuration
{
  "chunk_size": 1024,
  "chunk_overlap": 128,
  "strategy": "sentence",
  "metadata": [
    "filename",
    "page_number",
    "file_path"
  ]
}

Summary

Key Takeaways

  • Five data sources — Blob, SQL, Cosmos DB, SharePoint, OneLake
  • Authentication — Use Managed Identity with RBAC for production

  • Metadata enrichment — Always include source, page, and file info in chunks
