Packaging Checklist

Every repeatable AI offering should include:

  • Externalized configuration — All endpoints, model names, knowledge sources are parameterized
  • Tenant-specific values — Branding, domain, data sources are configurable per customer
  • Evaluation datasets — Baseline metrics and pass/fail criteria
  • Documentation — README, onboarding guide, architecture diagram
  • Deployment validation — Smoke tests that run post-deployment
  • Version management — Semantic versioning for templates and agents

✅ If you can't deploy for a new customer in under 30 minutes, your solution isn't productized yet.

Cost & Margin Calculation

Per-conversation pricing example:

Avg tokens per conversation: 2,500 GPT-4o cost per 1K tokens: ~$0.005 (blended) Azure AI Search cost per query: ~$0.001 Infrastructure overhead: ~$0.002 ────────────────────────────────────────────── Total cost per conversation: ~$0.020 Your price per conversation: $0.10 Gross margin: 80%

Licensing reminders

  • Model terms — Comply with Microsoft's usage policies for redistribution
  • Data ownership — Customer data stays in the customer's tenant
  • IP clarity — Define who owns fine-tuned models, evals, and custom prompts
  • Usage transparency — Provide customers with usage dashboards

🧪 Mini Hack: Customer FAQ Bot Template

Build a reusable, parameterized solution template:

  1. Define the config schema — What varies per customer?
  2. Create Bicep templates — Parameterized infrastructure for AI Foundry + Search + Storage
  3. Build agent templates — agent.yaml + system prompt with placeholders
  4. Write deployment automation — Script that reads config and provisions everything
  5. Add smoke tests — Validate agent config, tenant isolation, guardrails
  6. Deploy for two customers — Contoso + Fabrikam with different configs

✅ Success criteria: Both tenants deploy in < 10 minutes, pass all smoke tests, and each agent only answers from its own knowledge base.

14 / 14

Key Takeaways

  • Productize, don't customize — Parameterize once, deploy many times
  • White-label with config files — Branding, knowledge, prompts, and guardrails per customer
  • Isolate tenants at the data layer — Shared infra, separate search indexes and projects
  • Build accelerators, not demos — Include IaC, security, monitoring, eval, and docs
  • Price for margin — Account for all cost components before setting your price
  • Automate everything — Customer config → deployed agent in minutes, not weeks
